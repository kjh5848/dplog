# 타임존 기반 알림/SSE 설계 초안

## 목적
- `nextBillingAt` 기준 청구 예정/실패/해지 등 이벤트를 사용자 타임존에 맞춰 알림으로 제공.
- 실시간 UI 알림은 SSE로, 푸시/이메일은 후속 선택지로 남겨둔다.

## 이벤트 소스
- **청구 예정**: `nextBillingAt` D-3, D-1, D-0 시점.
- **결제 결과**: PortOne 웹훅(PAID/FAILED) → 상태 전이 후 알림.
- **스케줄 상태**: 예약 누락/취소 감지(`SubscriptionScheduleSyncService`).
- **사용자 행위**: 해지/업/다운 신청/적용 시점.

## 시간 계산 규칙
- 기준: `subscription.nextBillingAt` (없으면 `nextBillingDate` + 기본 타임존/기본 시각 09:00).
- 타임존: `User.timezone`(IANA) 우선, 없거나 파싱 실패 시 `payment.timezone.default`.
- 알림 시각 예: D-3, D-1, D-0 09:00 (사용자 선호 청구 시각을 알림에도 동일하게 적용).

## SSE 채널 (안)
- 엔드포인트: `GET /api/v1/notifications/stream` (SSE, Last-Event-ID 지원).
- 이벤트 페이로드 공통 필드:
  - `eventId`: 멱등 키(`alert-<subscriptionId>-<nextBillingAt>-<type>` 등).
  - `type`: UPCOMING_BILLING | PAYMENT_PAID | PAYMENT_FAILED | SCHEDULE_MISSING | CANCELLED | UPGRADE_APPLIED …
  - `subscriptionId`, `userId`, `occurredAt`, `nextBillingAt`(옵션), `message`.
- 멱등: `SubscriptionEventLog`와 동일한 테이블/키 스킴을 재사용하거나 별도 `NotificationLog`로 중복 전송 차단.
- 재전송: 클라이언트가 Last-Event-ID로 재연결 → 서버는 미전달 알림을 재방출.

## 스케줄러/큐 (안)
- 알림 스케줄러가 `nextBillingAt` - {3d, 1d, 0d} 범위에 도달한 건을 조회.
- 조회 시 필터: `status in (ACTIVE, PENDING, PENDING_CANCEL)`, `nextBillingAt` not null.
- 발송 후 로그 기록(eventId, status=PROCESSED/FAILED) → 멱등 보장.
