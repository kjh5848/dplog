# 카카오 로그인 플로우 문서

## 개요
이 문서는 카카오 OAuth 2.0을 통한 소셜 로그인의 전체 플로우와 데이터베이스 저장 과정을 설명합니다.

## 1. 전체 플로우 다이어그램

```
[프론트엔드] → [카카오 서버] → [백엔드] → [데이터베이스]
     ↓             ↓           ↓          ↓
  1. 로그인 요청   2. 인증코드   3. 사용자정보   4. DB 저장
                  반환        처리 & 세션생성
```

## 2. 상세 플로우

### 2.1 프론트엔드에서 카카오 로그인 시작

**사용자 액션:**
- 사용자가 "카카오로 로그인" 버튼 클릭

**프론트엔드 처리:**
```javascript
// 카카오 OAuth 인증 URL로 리다이렉트
const kakaoAuthUrl = `https://kauth.kakao.com/oauth/authorize?client_id=${REST_API_KEY}&redirect_uri=${REDIRECT_URI}&response_type=code&state=${STATE}`;
window.location.href = kakaoAuthUrl;
```

### 2.2 카카오 서버에서 인증

**카카오 인증 과정:**
1. 사용자가 카카오 계정으로 로그인
2. 앱 권한 동의 (닉네임, 프로필 이미지, 이메일 등)
3. 인증 완료 후 `redirect_uri`로 인증 코드 전달

**리다이렉트 URL:**
```
http://localhost:3000/kakao/callback?code={AUTHORIZATION_CODE}&state={STATE}
```

### 2.3 프론트엔드에서 백엔드로 인증 코드 전송

**API 호출:**
```javascript
// POST /v1/auth/kakao/callback
const response = await fetch('/v1/auth/kakao/callback', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Cookie': document.cookie  // 세션 쿠키 포함
  },
  body: JSON.stringify({
    code: authorizationCode,
    state: stateValue
  })
});
```

### 2.4 백엔드에서 카카오 API 호출 및 처리

#### 2.4.1 컨트롤러 (AuthControllerApiV1)
```java
@PostMapping("/kakao/callback")
public HttpEntity<?> kakaoCallback(@RequestBody @Valid ReqKakaoCallbackDTOApiV1 reqDto, HttpSession session) {
    // AuthServiceApiV1.kakaoCallback() 호출
    return authServiceApiV1.kakaoCallback(reqDto, session);
}
```

#### 2.4.2 서비스 레이어 (AuthServiceApiV1)
**주요 처리 과정:**

1. **KakaoOAuthService를 통한 사용자 정보 조회**
   ```java
   KakaoUserResponseDTO kakaoUserInfo = kakaoOAuthService.processOAuthFlow(reqDto.getCode());
   ```

2. **기존 사용자 확인**
   ```java
   String providerId = kakaoUserInfo.getId().toString();
   Optional<UserEntity> existingUserOptional = userRepository
       .findByProviderAndProviderIdAndDeleteDateIsNull("KAKAO", providerId);
   ```

3. **신규 사용자 생성 또는 기존 사용자 업데이트**
4. **사용자 상태 검증**
5. **중복 로그인 확인**
6. **FREE 멤버십 할당 (신규 사용자)**
7. **세션 생성 및 로그인 처리**

#### 2.4.3 카카오 API 통신 (KakaoOAuthService)

**1단계: 액세스 토큰 발급**
```java
// POST https://kauth.kakao.com/oauth/token
KakaoTokenResponseDTO tokenResponse = getAccessToken(authorizationCode);
```

**요청 데이터:**
- `grant_type`: authorization_code
- `client_id`: 카카오 REST API 키
- `redirect_uri`: 설정된 리다이렉트 URI
- `code`: 인증 코드

**응답 데이터:**

## 3. 데이터베이스 저장

### 3.1 사용자 정보 저장 (user_entity 테이블)

#### 신규 사용자 생성 시:
```sql
INSERT INTO user_entity (
    username,           -- 'kakao_123456789' (카카오 ID 기반)
    name,               -- 카카오 닉네임
    email,              -- 카카오 이메일 (중복 확인 후)
    password,           -- BCrypt 해시된 더미 패스워드
    provider,           -- 'KAKAO'
    provider_id,        -- '123456789' (카카오 사용자 ID)
    profile_image,      -- 카카오 프로필 이미지 URL
    status,             -- 'COMPLETION' (즉시 승인)
    balance,            -- 0
    authority,          -- ['USER']
    create_date,        -- 현재 시간
    last_login_date,    -- 현재 시간
    expire_date         -- 기본값 (1년 후)
) VALUES (
    'kakao_123456789',
    '사용자닉네임',
    'user@example.com',
    '$2a$10$...',  -- BCrypt("KAKAO_USER_NO_PASSWORD")
    'KAKAO',
    '123456789',
    'https://profile-image-url',
    'COMPLETION',
    0,
    'USER',
    NOW(),
    NOW(),
    DATE_ADD(NOW(), INTERVAL 1 YEAR)
);
```

#### 기존 사용자 업데이트 시:
```sql
UPDATE user_entity SET
    name = '업데이트된닉네임',                    -- 카카오에서 변경된 경우
    profile_image = '새프로필이미지URL',           -- 카카오에서 변경된 경우
    email = 'new-email@example.com',            -- 기존에 없던 경우만
    last_login_date = NOW()                     -- 마지막 로그인 시간 업데이트
WHERE provider = 'KAKAO' AND provider_id = '123456789';
```

### 3.2 멤버십 할당 (membership_user_entity 테이블)

**신규 사용자에게 FREE 멤버십 자동 할당:**
```sql

SELECT id FROM membership_entity WHERE name = 'FREE';

INSERT INTO membership_user_entity (
    user_entity_id,
    membership_entity_id,

    {FREE멤버십ID},
    'ACTIVATE',
    NOW()
);
```

## 5. 데이터베이스 스키마

### user_entity 테이블 구조:
```sql
CREATE TABLE user_entity (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    name VARCHAR(100),
    email VARCHAR(255),
    password VARCHAR(255) NOT NULL,
    provider VARCHAR(20) DEFAULT 'LOCAL' COMMENT 'OAuth 제공자',
    provider_id VARCHAR(100) COMMENT 'OAuth 제공자 사용자 ID',
    profile_image VARCHAR(500) COMMENT '프로필 이미지 URL',
    status ENUM('WAITING', 'COMPLETION', 'WITHDRAW', 'STOP'),
    balance INT DEFAULT 0,
    authority JSON,
    create_date DATETIME DEFAULT CURRENT_TIMESTAMP,
    last_login_date DATETIME COMMENT '마지막 로그인 시간',
    expire_date DATETIME,
    delete_date DATETIME,
    -- 인덱스
    INDEX idx_provider_provider_id (provider, provider_id),
    INDEX idx_email (email),
    INDEX idx_username (username)
);
```

## 6. 보안 고려사항

### 6.1 데이터 검증
- **이메일 중복 확인**: 기존 사용자와 이메일 충돌 방지
- **Username 충돌 방지**: `kakao_{providerId}` 형식으로 고유성 보장
- **입력 데이터 검증**: `@Valid` 어노테이션으로 DTO 검증

### 6.2 에러 처리
- **카카오 API 호출 실패**: 적절한 에러 메시지 반환
- **데이터베이스 오류**: 트랜잭션 롤백 처리
- **세션 충돌**: 중복 로그인 방지

### 6.3 개인정보 보호
- **패스워드 암호화**: BCrypt를 사용한 더미 패스워드 저장
- **토큰 보안**: 액세스 토큰은 메모리에서만 사용, 저장하지 않음
- **세션 관리**: 안전한 세션 타임아웃 설정
