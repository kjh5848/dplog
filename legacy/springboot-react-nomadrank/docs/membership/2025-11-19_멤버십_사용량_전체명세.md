## 1. 테이블 개요

| 컬럼 | 타입 | 널 | 설명 |
| --- | --- | --- | --- |
| id | BIGINT | N | 사용량 고유 ID |
| membership_user_id | BIGINT | N | MEMBERSHIP_USER FK |
| membership_detail_id | BIGINT | N | MEMBERSHIP_DETAIL FK (서비스 제한 정의) |
| service_sort | VARCHAR(50) | N | 서비스 구분 (ServiceSort) |
| period_type | VARCHAR(20) | N | 집계 단위 (MONTHLY/DAILY) |
| period_key | CHAR(10) | N | 집계 키 (MONTHLY: yyyy-MM, DAILY: yyyy-MM-dd) |
| limit_count | INT | Y | 한도 (NULL=무제한) |
| used_count | INT | N | 현재 사용량 (기본 0) |
| unlimited | TINYINT(1) | N | 무제한 여부 |
| last_used_at | DATETIME | Y | 마지막 사용 시각 |
| created_at | DATETIME | N | 생성일시 |
| updated_at | DATETIME | N | 수정일시 |

- **유니크 키:** `(membership_user_id, membership_detail_id, period_type, period_key)`  
- **인덱스:** `(membership_user_id, service_sort)`  
- **규칙:** 무제한 플랜은 `unlimited=1`, `limit_count=NULL`  
- **기간 규칙:** MONTHLY → `yyyy-MM`, DAILY → `yyyy-MM-dd`

## 2. MySQL DDL

```sql
CREATE TABLE `MEMBERSHIP_USAGE` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '사용량 고유 ID',
  `membership_user_id` BIGINT NOT NULL COMMENT '멤버십 가입자 ID (MEMBERSHIP_USER FK)',
  `membership_detail_id` BIGINT NOT NULL COMMENT '멤버십 상세 ID (MEMBERSHIP_DETAIL FK)',
  `service_sort` VARCHAR(50) NOT NULL COMMENT '서비스 구분 (ServiceSort)',
  `period_type` VARCHAR(20) NOT NULL COMMENT '집계 단위 (MONTHLY/DAILY 등)',
  `period_key` CHAR(10) NOT NULL COMMENT '집계 기간 키 (예: 2025-10, 2025-10-21)',
  `limit_count` INT NULL COMMENT '서비스 사용 한도 (null: 무제한)',
  `used_count` INT NOT NULL DEFAULT 0 COMMENT '현재 사용량',
  `unlimited` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '무제한 여부 (1: 무제한)',
  `last_used_at` DATETIME NULL COMMENT '마지막 사용 시각',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '생성일시',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '수정일시',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_membership_usage_period` (`membership_user_id`, `membership_detail_id`, `period_type`, `period_key`),
  KEY `ix_membership_usage_user_sort` (`membership_user_id`, `service_sort`),
  CONSTRAINT `fk_usage_membership_user`
    FOREIGN KEY (`membership_user_id`) REFERENCES `MEMBERSHIP_USER`(`id`)
    ON DELETE CASCADE ON UPDATE RESTRICT,
  CONSTRAINT `fk_usage_membership_detail`
    FOREIGN KEY (`membership_detail_id`) REFERENCES `MEMBERSHIP_DETAIL`(`id`)
    ON DELETE RESTRICT ON UPDATE RESTRICT
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

## 3. JPA 엔티티

```java
@Entity
@Table(

@Getter @Setter

    @ManyToOne(fetch = FetchType.LAZY, optional = false)

            foreignKey = @ForeignKey(name = "fk_usage_membership_detail"))
    private MembershipDetailEntity membershipDetail;

    @Enumerated(EnumType.STRING)
    @Column(name = "service_sort", nullable = false, length = 50,
            columnDefinition = "VARCHAR(50) NOT NULL COMMENT '서비스 구분 (ServiceSort)'")

    @Enumerated(EnumType.STRING)
    @Column(name = "period_type", nullable = false, length = 20,
            columnDefinition = "VARCHAR(20) NOT NULL COMMENT '집계 단위 (MONTHLY/DAILY 등)'")
    private UsagePeriodType periodType;

    @Column(name = "period_key", nullable = false, length = 10,
            columnDefinition = "CHAR(10) NOT NULL COMMENT '집계 기간 키 (예: 2025-10)'")

    @Column(name = "limit_count",
            columnDefinition = "INT COMMENT '서비스 사용 한도 (null: 무제한)'")
```

## 5. Util (UsageKey)

```java
@Component
public class UsageKey {
    public String key(UsagePeriodType type, LocalDate date) {
        return switch (type) {
            case MONTHLY -> date.format(DateTimeFormatter.ofPattern("yyyy-MM"));
            case DAILY -> date.format(DateTimeFormatter.ISO_LOCAL_DATE);
        };
    }
}
```

## 6. Service (MembershipUsageService)

```java
@Service
@RequiredArgsConstructor
public class MembershipUsageService {
    private final MembershipUsageRepository usageRepo;
    private final UsageKey usageKey;

    @Transactional
    public MembershipUsageEntity ensureRowForToday(MembershipUserEntity membershipUser, ServiceSort serviceSort, UsagePeriodType periodType, Integer limitCountOrNull, boolean unlimited) {
        String periodKey = usageKey.key(periodType, LocalDate.now(ZoneId.of("Asia/Seoul")));
        return usageRepo.findForUpdate(membershipUser.getId(), serviceSort, periodType, periodKey)
            .orElseGet(() -> {
                var e = MembershipUsageEntity.builder()
                        .membershipUser(membershipUser)
                        .serviceSort(serviceSort)
                        .periodType(periodType)
                        .periodKey(periodKey)
                        .limitCount(unlimited ? null : limitCountOrNull)
                        .unlimited(unlimited)
                        .usedCount(0)
                        .lastUsedAt(null)
                        .build();
                return usageRepo.save(e);
            });
    }

    @Transactional
    public void increaseUsageOrThrow(MembershipUserEntity membershipUser, ServiceSort serviceSort, UsagePeriodType periodType, int delta) {
        var plan = /* 현재 요금제 조회 */;
        boolean unlimited = plan.isUnlimited(serviceSort);
        Integer limit = unlimited ? null : plan.limitCount(serviceSort);
        var row = ensureRowForToday(membershipUser, serviceSort, periodType, limit, unlimited);

        if (!row.getUnlimited()) {
            int next = row.getUsedCount() + delta;
            if (row.getLimitCount() != null && next > row.getLimitCount()) {
                throw new IllegalStateException("사용 한도 초과");
            }
            row.setUsedCount(next);
        }
        row.setLastUsedAt(LocalDateTime.now());
    }
}
```

## 8. 테스트 체크리스트

| 항목 | 내용 |
| --- | --- |
| 첫 가입 | FREE 플랜 생성 + 사용량 레코드 정상 생성 |
| 업그레이드 | 기존 ACTIVE → EXPIRED, 새 ACTIVE + 사용량 초기화 |
| 다운그레이드 | 즉시 미적용, 예약 생성 후 다음달 전환 |
| 사용 증가 | 무제한은 제한 없음, 한도 초과 시 예외 발생 |
| 동시성 | 비관적 잠금으로 중복 증가 방지 |
| 웹훅 | PG 멱등성 유지 (paymentId 기준) |
