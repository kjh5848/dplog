# FREE 플랜 자동 가입 구현 계획서

## 1. 현황 분석

### 1.1 기존 시스템 구조
현재 시스템은 **이중 구조**로 되어 있습니다:

#### A. Plan 시스템 (신규)
- `PlanEntity`: 구독 플랜 정의 (FREE, PRO, PREMIUM, AGENCY)
- `PlanDetailEntity`: 플랜별 서비스 제한 정보
- `SubscriptionEntity`: 사용자별 구독 정보

#### B. Membership 시스템 (기존)
- `MembershipEntity`: 멤버십 정의 (FREE, BASIC, PREMIUM 등)
- `MembershipDetailEntity`: 멤버십별 서비스 제한 정보  
- `MembershipUserEntity`: 사용자별 멤버십 정보

### 1.2 회원가입 프로세스 현황
- 현재는 **로그인만** 존재하고 별도 회원가입 API가 없음
- 사용자는 `UserEntity`에 직접 생성되어야 함
- 로그인 시 멤버십이 없으면 "FREE" 멤버십 자동 할당

### 1.3 문제점
1. **회원가입 API 부재**: 신규 사용자 등록 프로세스가 없음
2. **이중 시스템**: Plan과 Membership이 중복 역할 수행
3. **자동 플랜 할당 로직 부재**: 신규 가입 시 FREE 플랜 자동 연결 로직 없음

### 3.2 FREE 플랜 자동 할당 로직

#### 3.2.1 Membership 시스템 활용 (단기)
```java
// AuthServiceApiV1.signup() 메서드 내
private void assignDefaultMembership(UserEntity userEntity) {
    // FREE 멤버십 조회
    MembershipEntity freeMembership = membershipRepository.findByName("FREE");
    
    // 사용자에게 FREE 멤버십 할당
    MembershipUserEntity membershipUser = MembershipUserEntity.builder()
        .userEntity(userEntity)
        .membershipEntity(freeMembership)
        .membershipState(MembershipState.ACTIVATE)
        .build();
    
    membershipUserRepository.save(membershipUser);
}
```

#### 3.2.2 Plan 시스템 연동 (중장기)
```java
// 새로운 서비스: UserPlanService
@Service
public class UserPlanService {
    
    @Transactional
    public void assignFreePlan(UserEntity userEntity) {
        // 1. FREE 플랜 조회
        PlanEntity freePlan = planRepository.findByNameAndDeleteDateIsNull("FREE");
        
        // 2. 구독 엔티티 생성
        SubscriptionEntity subscription = SubscriptionEntity.builder()
            .userId(userEntity.getId())
            .productId("free-plan")
            .productName("FREE 플랜")
            .amount(BigDecimal.ZERO)
            .billingCycle(BillingCycle.MONTHLY)
            .status(SubscriptionStatus.ACTIVE)
            .nextBillingDate(LocalDate.now().plusMonths(1))
            .build();
            
        subscriptionRepository.save(subscription);
    }
}
```

## 4. 구현 우선순위

### 4.1 높은 우선순위 (1주차)
1. **회원가입 API 구현**
   - ReqAuthSignUpDTOApiV1, ResAuthSignUpDTOApiV1 생성
   - AuthControllerApiV1에 signup 메서드 추가
   - AuthServiceApiV1에 signup 로직 구현

2. **FREE 멤버십 자동 할당**
   - 회원가입 시 FREE 멤버십 자동 할당 로직
   - 기본 데이터 확인 (FREE 멤버십, 서비스 제한 등)

3. **기존 사용자 처리**
   - 멤버십이 없는 기존 사용자 대상 FREE 멤버십 할당

### 4.2 중간 우선순위 (2-3주차)  
1. **Plan 시스템 연동**
   - UserPlanService 구현
   - 회원가입 시 FREE 플랜 자동 생성
   - Plan과 Membership 연동 로직

2. **유효성 검사 강화**
   - 중복 가입 방지
   - 이메일 인증 (선택적)
   - 비밀번호 정책

### 4.3 낮은 우선순위 (4주차 이후)
1. **시스템 통합**
   - Plan/Membership 시스템 일원화
   - 레거시 코드 정리
   - 성능 최적화

2. **부가 기능**
   - 소셜 로그인 연동
   - 프로필 설정 기능
   - 가입 통계 및 분석
