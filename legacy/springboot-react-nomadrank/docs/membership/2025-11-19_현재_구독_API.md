# 📋 API 문서: 현재 구독 정보 조회

## 🎯 개요

사용자의 현재 활성화된 멤버십(구독) 정보를 조회하는 API입니다. 기존 `/v1/pricing/subscription/current` 404 오류가 해결되었습니다.

## 🔗 API 엔드포인트

### GET /v1/membership/current

현재 로그인된 사용자의 구독 정보를 조회하거나, 관리자가 특정 사용자의 구독 정보를 조회합니다.

**URL**: `/v1/membership/current`  
**Method**: `GET`  
**Authentication**: 세션 기반 인증 필요  

#### 파라미터

| 파라미터 | 타입 | 필수 | 설명 |
|----------|------|------|------|
| `userId` | Long | ❌ | 조회할 사용자 ID (없으면 본인 조회) |

#### 권한

- **본인 조회**: 파라미터 없이 호출 시 현재 로그인된 사용자의 구독 정보 조회
- **타인 조회**: `userId` 파라미터 사용 시 ADMIN 권한 필요

## 📤 요청 예시

### 1. 본인 구독 조회 (일반적인 케이스)
```javascript
const response = await fetch('/v1/membership/current', {
  method: 'GET',
  credentials: 'include' // 세션 쿠키 포함 필수
});

const result = await response.json();
```

### 2. 특정 사용자 구독 조회 (관리자용)
```javascript
const response = await fetch('/v1/membership/current?userId=123', {
  method: 'GET',
  credentials: 'include'
});

const result = await response.json();
```

## 📥 응답 형식

### ✅ 성공 응답 (활성 구독 있음)
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "membershipId": 1,
    "membershipName": "PREMIUM",
    "startDate": "2024-01-01",
    "endDate": "2024-12-31",
    "membershipState": "ACTIVATE",
    "remainingDays": 120,
    "point": 1000
  }
}
```

### ✅ 성공 응답 (활성 구독 없음)
```json
{
  "code": 0,
  "message": "현재 활성화된 구독이 없습니다.",
  "data": null
}
```

## 📊 응답 필드 설명

| 필드명 | 타입 | 설명 |
|--------|------|------|
| `membershipId` | Long | 멤버십 고유 ID |
| `membershipName` | String | 멤버십 이름 (예: "PREMIUM", "BASIC", "ENTERPRISE") |
| `startDate` | String | 구독 시작일 (YYYY-MM-DD 형식) |
| `endDate` | String | 구독 종료일 (YYYY-MM-DD 형식) |
| `membershipState` | String | 구독 상태<br/>• `ACTIVATE`: 활성화<br/>• `READY`: 준비<br/>• `EXPIRED`: 만료 |
| `remainingDays` | Long | 남은 일수 (0 이하면 만료) |
| `point` | Integer | 멤버십 포인트 |

## ❌ 에러 응답

### 401 Unauthorized - 인증 실패
```json
{
  "code": -1,
  "message": "인증에 실패했습니다."
}
```
**원인**: 세션이 만료되었거나 로그인하지 않은 상태

### 403 Forbidden - 권한 없음
```json
{
  "code": -1,
  "message": "권한이 없습니다."
}
```
**원인**: 다른 사용자의 구독 정보를 조회하려 했으나 ADMIN 권한이 없음

### 404 Not Found - 사용자 없음
```json
{
  "code": -2,
  "message": "존재하지 않는 사용자입니다."
}
```
**원인**: 요청한 `userId`에 해당하는 사용자가 존재하지 않음

## 🔧 프론트엔드 구현

### React Hook 예시
```javascript
import { useState, useEffect } from 'react';

const useCurrentSubscription = (userId = null) => {
  const [subscription, setSubscription] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const fetchCurrentSubscription = async () => {
      try {
        let url = '/v1/membership/current';
        if (userId) {
          url += `?userId=${userId}`;
        }

        const response = await fetch(url, {
          method: 'GET',
          credentials: 'include'
        });
        
        const result = await response.json();
        
        if (result.code === 0) {
          setSubscription(result.data);
        } else {
          setError(result.message);
        }
      } catch (err) {
        setError('구독 정보를 불러오는데 실패했습니다.');
      } finally {
        setLoading(false);
      }
    };

    fetchCurrentSubscription();
  }, [userId]);

  return { subscription, loading, error };
};

export default useCurrentSubscription;
```

### 컴포넌트 사용 예시
```javascript
import useCurrentSubscription from './hooks/useCurrentSubscription';

const SubscriptionInfo = () => {
  const { subscription, loading, error } = useCurrentSubscription();

  if (loading) return <div className="spinner">로딩 중...</div>;
  if (error) return <div className="error">에러: {error}</div>;

  return (
    <div className="subscription-info">
      {subscription ? (
        <div className="active-subscription">
          <h3>현재 구독: {subscription.membershipName}</h3>
          <div className="subscription-details">
            <p><strong>남은 일수:</strong> {subscription.remainingDays}일</p>
            <p><strong>구독 기간:</strong> {subscription.startDate} ~ {subscription.endDate}</p>
            <p><strong>포인트:</strong> {subscription.point}P</p>
            <span className={`status ${subscription.membershipState.toLowerCase()}`}>
              {subscription.membershipState === 'ACTIVATE' ? '활성화' : 
               subscription.membershipState === 'READY' ? '준비' : '만료'}
            </span>
          </div>
        </div>
      ) : (
        <div className="no-subscription">
          <p>현재 활성화된 구독이 없습니다.</p>
          <button className="btn-primary">구독하기</button>
        </div>
      )}
    </div>
  );
};

export default SubscriptionInfo;
```

## 🧪 테스트 방법

### 1. 브라우저 개발자 도구
```javascript
// 본인 구독 조회
fetch('/v1/membership/current', { credentials: 'include' })
  .then(res => res.json())
  .then(data => console.log('내 구독:', data));

// 특정 사용자 조회 (관리자만)
fetch('/v1/membership/current?userId=123', { credentials: 'include' })
  .then(res => res.json())
  .then(data => console.log('사용자 123 구독:', data));
```

### 2. cURL 테스트
```bash
# 본인 구독 조회
curl -X GET "http://localhost:8081/v1/membership/current" \
     -H "Cookie: JSESSIONID=YOUR_SESSION_ID"

# 특정 사용자 조회 (관리자)
curl -X GET "http://localhost:8081/v1/membership/current?userId=123" \
     -H "Cookie: JSESSIONID=YOUR_SESSION_ID"
```

## 📝 주의사항

1. **세션 인증**: 모든 요청에 `credentials: 'include'` 옵션이 필요합니다.
2. **CORS**: 이미 설정되어 있어 별도 설정 불필요합니다.
3. **캐싱**: 구독 정보는 자주 변경되지 않으므로 적절한 캐싱 전략을 고려해보세요.
4. **에러 핸들링**: 네트워크 오류와 API 오류를 구분하여 처리하세요.
