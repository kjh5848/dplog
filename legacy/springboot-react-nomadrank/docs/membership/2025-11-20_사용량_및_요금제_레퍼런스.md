## 2. 요금제별 기능 한도

아래 표는 현재 정책 + 코드/로그로 확인된 수치를 우선 정리한 것이다. `MembershipDetail` 테이블이 단일 출처(Source of Truth)이므로 실시간 수치는 SQL로 조회해 유지한다.

### 2.1 일간 한도 (상점 등록/키워드 추적 제외)

| 기능 \ 요금제 | FREE | 실속 점주 | 성장 오너 | 마스터 |
| --- | --- | --- | --- | --- |
| 실시간 순위 조회 (`NPLACE_RANK_REALTIME`) | **5회/일 (≈150회/월)** | **20회/일 (≈600회/월)** | **60회/일 (≈1,800회/월)** | 무제한 |
| 키워드 조회 (`NPLACE_KEYWORD`) | 5회/일 (≈150회/월) | 20회/일 (≈600회/월) | 50회/일 (≈1,500회/월) | 무제한 |
| 연관 키워드 (`NPLACE_KEYWORD_RELATION`) | 5회/일 (≈150회/월) | 10회/일 (≈300회/월) | 30회/일 (≈900회/월) | 무제한 |
| 블로그 저품질 (`BLOG_QUALITY_CHECK`) | 5회/일 (≈150회/월) | 10회/일 (≈300회/월) | 30회/일 (≈900회/월) | 무제한 |
| 리뷰 답글 (`NPLACE_REVIEW_REPLY`) | 미제공 | 미제공 | 미제공 | 무제한 |

> ⚠️ 일간 한도만 관리한다. 예: 5회/일 이라면 별도 월간 누적 없이 하루 단위로 차단 메시지를 노출한다. 필요한 경우 “당월 예상 사용량=일간 × 30” 정도로 가이드만 제공한다.

### 2.2 월간 유지 항목

| 기능 \ 요금제 | FREE | 실속 점주 | 성장 오너 | 마스터 | 비고 |
| --- | --- | --- | --- | --- | --- |
| 키워드 추적 (`NPLACE_RANK_TRACK`) | 3개 (플레이스/키워드 슬롯) | 10개 | 30개 | 200개 | 월간 계약 성격 유지 |
| 상점 등록 (`NPLACE_SHOP_REGISTER`) | 1개 | 1개 | 3개 | 무제한 | 월간 쿼터 유지 |

### 2.3 한도 데이터 조회 SQL

```sql
SELECT m.name AS membership_name,
       md.service_sort,
       md.limit_count,
       CASE WHEN md.limit_count IS NULL THEN 'UNLIMITED' ELSE CONCAT(md.limit_count, '회') END AS display_limit
FROM MEMBERSHIP m
JOIN MEMBERSHIP_DETAIL md ON m.id = md.membership_id AND md.delete_date IS NULL
WHERE m.delete_date IS NULL
ORDER BY m.sort_order, md.service_sort;
```

- 운영/스테이징 DB 모두 `Asia/Seoul` 기준.  
- 신규 요금제가 나오면 `MembershipDetail`에 먼저 등록한 뒤 이 문서를 갱신한다.  
- `limit_count`가 `NULL`이면 무제한, `0`이면 사용 금지 상태로 간주한다.  
- **액션 아이템**: TBD 항목은 위 SQL 결과를 근거로 채우고 git에 업데이트.

### 2.2 제한 초과 예외 코드

| 영역 | 예외 | 메시지/코드 |
| --- | --- | --- |
| 실시간 순위 조회 | `AuthenticationException` | `limitCount + "회 사용횟수를 모두 사용하였습니다."` |

## 3. 월간/일간 한도 및 초기화

| 구분 | 현재 구현 | 목표/정책 | 구현 포인트 |
| --- | --- | --- | --- |
| 월간 한도 | `UseLogRepository.findByUser...between(firstDayOfMonth, lastDayOfMonth)` 패턴. (`NplaceServiceApiV1.getRealtime` 참고) | 상점등록, 키워드 추적은 월간 고정 유지. | `UsageResetService`가 월간 한도 초기화 시 `ServiceSort.NPLACE_SHOP_REGISTER`는 제외하도록 수정됨 (2025-11-20). |
| 일간 한도 | 아직 적용 X (전부 월간으로 집계) | 실시간 조회/키워드 조회/블로그 검사/리뷰 답글은 일간 측정으로 전환. | `MembershipUsage` 도입 후 `UsagePeriodType.DAILY` 레코드 생성, `UseLog` 조회 범위를 `today atStartOfDay`~`endOfDay`로 조정. |
| 업그레이드 시 초기화 | 상위 플랜 진입 즉시 `UsageResetService.resetUsage()` 호출. | **상점등록 제외** → 나머지 기능만 초기화. | `UseLogRepository.deleteByUserAndCreatedBetweenExcludingSort` 쿼리 도입 (PR 2025-11-20). |
| 다운그레이드 | 다음 결제일에 `finalizePendingPlanIfNecessary` 실행. | 새 플랜 적용 시 `UsageResetService` 호출 여부 선택. | 일단 월간 유지, 일간 전환 시 `period_type`별로 별도 초기화 필요. |
| 해지/환불 | 7일 이내 미사용 전액 환불 시 즉시 FREE 복귀. | 이미 적용 (`PaymentServiceApiV1.applyImmediateCancellationAfterFullRefund`). | UsageLog는 해지 시점에서 이미 `SubscriptionService`가 EXPIRED 처리. |

### 3.1 리셋 타이밍 체크리스트

1. **월간 기준**: 매월 1일 00:00 KST ~ 말일 23:59:59.  
2. **일간 기준 (도입 예정)**: 매일 00:00 KST. `UsageKey`는 ISO 날짜.  
3. **업그레이드 직후**: 즉시 초기화 + 한도 재로딩 (상점등록 제외).  
4. **다운그레이드**: 예약 적용일(`nextBillingDate`)에 초기화.  
5. **환불 후 FREE 복귀**: 즉시 사용량 차단 → FREE 기본 한도 부여.

## 4. 요금제 가격 (월간/연간)

| 플랜 | 월간 가격 | 연간 가격 (12개월 × 20% 할인) | 비고 |
| --- | --- | --- | --- |
| FREE | 0원 | 0원 | 회원가입 즉시 자동 부여. 7일 무료체험 메시지 유지. |
| 실속 점주 | **19,000원/월** | **182,400원/년** | 월 매출 500만 원 이하 타겟. |
| 성장 오너 | **39,000원/월** | **374,400원/년** | 월 매출 500만~1,500만 원. |
| 프리미엄 마스터 | **59,000원/월** | **566,400원/년** | 월 매출 1,500만 원 이상. |

- 연간 가격 = `월간 가격 × 12 × 0.8` (20% 할인) 공식을 적용했다.  
- 할인율 변경 시 월간 금액뿐 아니라 `SubscriptionServiceApiV1`의 청구 금액, `MembershipDetail` 표시, 프런트 요금제 카드(섹션 4.2)를 동시에 업데이트한다.

### 4.1 가격 데이터 조회 SQL

```sql
SELECT name, price, price_yearly
FROM MEMBERSHIP
WHERE delete_date IS NULL
ORDER BY sort_order;
```

> **주의**: 결제 프로세스는 PortOne V2 기준. 가격 변경 시 `SubscriptionServiceApiV1.applyUpgradeWithPayment` / `UpgradeCalculator` / `PaymentServiceApiV1`를 동시에 조정해야 함.

## 5. 사용량 계산 & 결제 시나리오 예시

### 5.1 차액이 발생하는 업그레이드 (월간 → 월간)

1. `UpgradeCalculator.calculateSameCycle` 호출.  
2. `differenceAmount = prorated(targetPlan) - prorated(currentPlan)` (0 미만이면 0).  
3. PortOne 빌링키 결제 → 성공 시 `UsageResetService` 실행.  
4. 사용량 예시  
   - FREE (30/30 사용) → 실속 (150 한도)  
   - 차액: 실속(월) - FREE(월).  
   - 초기화: 실시간·키워드·블로그·리뷰 = 0, 상점등록/추적 사용량은 유지.

### 5.2 추가 결제가 필요한 케이스 (연간 → 월간 상위 플랜)

1. `processYearlyToMonthlyRefund`에서 잔여 연간 금액 환불.  
2. `applyUpgradeWithPayment`가 월간 상위 플랜 금액(예: 성장 39,000원) 즉시 청구.  
3. 환불 실패 시 전체 트랜잭션 롤백 (업그레이드 취소).  
4. 사용량: 새 상위 플랜 적용 즉시 초기화.  
5. BillingKey: 기존 예약 삭제 후 월간 예약 재생성.

### 5.3 전액 환불 + FREE 복귀 (7일 이내, 사용량 0)

1. `RefundCalculator.calculate` → `fullRefund=true`.  
2. `PaymentServiceApiV1.cancelSubscription`이 환불 실행 후 `applyImmediateCancellationAfterFullRefund` 호출.  
3. BillingKey 즉시 삭제 (`PortOneV2Client.deleteBillingKey`).  
4. `UsageResetService` 호출 없이 FREE 멤버십 재부여 (사용량 = FREE 기본 한도).  
5. 고객 안내: “전액 환불 / 즉시 FREE 플랜 전환”.
