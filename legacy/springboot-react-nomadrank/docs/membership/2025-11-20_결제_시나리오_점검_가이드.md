# 결제 시나리오 점검 가이드 (2025-11-20)

> 목적: 업그레이드/해지/환불 등 핵심 시나리오를 표준화된 항목으로 검증한다.

---

## 1. 플랜 업그레이드 (월 ↔ 월)

- [x] `UpgradeCalculator.calculateSameCycle` 차액 계산 정상 여부 (차액 < 0 ⇒ 0)  
      → `UpgradeCalculator`가 월/연 전환별 계산을 모두 캡슐화하고 `difference.signum() < 0` 시 0으로 보정.
- [x] `UsageResetService.resetUsage` 호출 (상점등록 제외)  
      → `SubscriptionServiceApiV1`가 등급 상승/연→월 상향일 때 `resetUsage=true`.
- [x] BillingKey 존재 시 PortOne 결제 성공 로그 확인  
      → `applyUpgradeWithPayment`가 BillingKey 결제 후 `markAsPaid`, 예약 갱신 로그 기록.
- [x] MembershipUserLog: `UPGRADE_RESET`, `ACTIVATE_AFTER_PAYMENT`
- [x] 프런트/웹훅: 성공 메시지, 사용량 UI 즉시 반영 (API 응답/로그 확인)

## 2. 월 → 연 업그레이드

- [x] `calculateMonthlyToYearly` 결과에 따라 연간 금액 청구

- [x] 연간 기간 (`nextBillingDate + 1년`) 업데이트 (`nextBillingDate.plusYears(1)`)
- [x] 사용량 초기화 없음 (표준 정책 그대로 유지)
- [x] 예약/빌링키 재생성 여부 확인 (`rebuildRecurringReservation`)
