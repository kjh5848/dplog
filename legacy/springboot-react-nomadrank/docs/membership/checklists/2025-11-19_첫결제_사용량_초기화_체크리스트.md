# 첫 결제(구독 시작) 사용량 초기화 체크리스트 (2025-11-19)

1. **시나리오 정의**
   - [ ] FREE 사용자 또는 이전 멤버십 주기가 없는 상태에서 첫 유료 구독을 시작할 때 사용량을 전부 리셋해야 함.
   - [ ] 기존 멤버십 주기가 있는 경우와 동일한 `UsageResetService` 로직을 재사용하지만, 범위 계산이 과거 FREE 사용 로그까지 포함하도록 해야 함.

2. **설계**
   - [ ] `resolveCycleStart`가 신규 멤버십 `startDate`만 보는 대신, 직전 멤버십(또는 FREE 기본 주기) 기준일을 사용할 수 있게 확장.
   - [ ] Subscription 생성 시 “첫 결제” 여부 플래그(예: `subscription.isFirstPayment()`)를 판단하도록 Service 정책 정의.

3. **구현**
   - [ ] Subscription 생성/첫 결제 완료 직후 `UsageResetService.resetUsage`를 호출하는 진입점을 추가.
   - [ ] FREE → 유료 전환 시 기존 FREE 사용 로그까지 삭제하도록 `resolveCycleStart` 로직을 보완(예: `MembershipUserLog`에서 직전 주기 시작일 조회).

4. **테스트**
   - [ ] FREE 사용자 최초 결제 → 업그레이드 성공 후 사용량 0인지 검증.
   - [ ] 기존 유료 사용자 업그레이드 시에도 범위 계산이 정상적으로 되는지 회귀 테스트.
