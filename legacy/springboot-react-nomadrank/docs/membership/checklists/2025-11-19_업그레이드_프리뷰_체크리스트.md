# 업그레이드 Preview 및 결제 프로필 카드 요구 체크리스트 (2025-11-19)

## 요구 사항 요약

### 업그레이드 Preview
- `/v1/subscription/upgrade` 요청에 `preview` 플래그를 추가한다.
- `preview=true`이면 `UpgradeCalculator` 계산만 수행하고 PortOne 결제/환불, `UsageResetService`, 정기 결제 재예약 등 상태 변경 로직은 실행하지 않는다.
- preview 응답에는 `differenceAmount`, `refundableAmount`, `newChargeAmount`, `upgradeType`, `currentPlanPrice`, `targetPlanPrice`, `remainingDays`, `nextBillingDate`, `message`를 채우고 `paymentRequested=false`, `refundPaymentId=null`을 보장한다.
- 실제 업그레이드에서도 동일 스키마를 사용하되 위 필드를 모두 채운다.
- API 형태는 `preview` 플래그 또는 `POST /v1/subscription/upgrade/quote` 신설 중 하나를 선택하되 내부 구현은 동일 로직을 재사용한다.

### 결제 프로필 카드 & 결제수단 관리
- 카드 데이터: `/v1/membership/current`, `/v1/membership/profile-summary`, 신규 `GET /v1/payments/status` 응답을 조합한다.
- `/v1/membership/current`는 `billingKeyStatus`, `currentPlanPrice`, `periodLabel`, `pendingCancel`, `pendingDowngrade`를 포함한다.
- `/v1/membership/profile-summary`는 월간 사용량/리셋 정보(`usage`, `resetDate`).
- `GET /v1/payments/status`는 최근 결제 이력, 금액, 상태, paymentId, 결제 수단 brand/issuer/lastFour/status, `nextPaymentDate`를 반환하며 cache: no-store.
- 결제수단(빌링키) 업데이트: `POST /v1/payments/billing-key/update` (또는 issue API 확장)으로 카드 정보를 갱신하고 BillingKey/SubscriptionEntity/PaymentScheduler를 최신화한다.

### 1. DTO/검증 단계
- [ ] `SubscriptionUpgradeRequest`에 `Boolean preview` 필드 추가(기본 false).
- [ ] `PaymentStatusResponse`, `BillingKeyUpdateRequest` 등 신규 DTO 정의 및 Swagger 문서화.
- [ ] `/v1/membership/current` 응답 스키마에 `billingKeyStatus`, `currentPlanPrice`, `periodLabel` 추가.
- [ ] `/v1/membership/profile-summary`의 `resetDate` 노출 여부 확인, `/v1/payments/status`는 cache: no-store 설정.

### 2. UpgradeCalculator & Response / 결제 상태 응답 확장
- [ ] `UpgradeCalculator.Result`에 `refundableAmount`, `upgradeType` 추가.
- [ ] `SubscriptionUpgradeResponse`에 `refundableAmount`, `newChargeAmount`, `upgradeType`, `refundPaymentId`, `paymentRequested` 필드를 추가하고 기본값을 처리.
- [ ] `PaymentStatusResponse`에 최근 결제일/금액/상태/paymentId, 결제 수단 brand/issuer/lastFour/status, `nextPaymentDate`, `pendingCancel`, `pendingDowngrade`를 포함.
- [ ] `/v1/membership/profile-summary`의 usage/reset 정보를 카드 요약에 맞게 노출.

### 3. 서비스/엔드포인트 구현
- [ ] `SubscriptionServiceApiV1.upgradeSubscription` preview 분기(상태 변경 로직 skip) 및 실제 업그레이드 시 확장된 응답 필드 채우기.
- [ ] `GET /v1/payments/status` 구현: PaymentRepository에서 최근 성공 결제 + BillingKey 상태를 조합해 반환.
- [ ] `POST /v1/payments/billing-key/update` (또는 issue API 확장) 구현: PortOne 빌링키 업데이트 → BillingKey/SubscriptionEntity/PaymentScheduler 갱신 → `/v1/membership/current`·`/v1/payments/status` 캐시 무효화.
- [ ] `/v1/membership/current`·`/v1/membership/profile-summary` 응답 확장을 적용해 PaymentProfileCard 데이터 공급.
