# 구독 해지 환불 & 테스트 스케줄 가이드
> 작성일: 2025.11.17

## 1. 7일 이내 미사용 전액 환불 로직
- `PaymentServiceApiV1.cancelSubscription`는 해지 요청 시 `UseLogRepository`를 통해 구독 시작일부터 해지일까지의 사용 로그를 전부 스캔합니다.
- 사용 로그가 하나도 없고 `startDate` 기준 7일 이내라면 `RefundCalculator`가 자동으로 **전액 환불** 결론(`fullRefund=true`)을 반환합니다.
- 환불 가능 금액이 0보다 크면 `PortOneV2Client.cancelPayment`가 호출되어 최근 성공 결제를 전액/부분 취소하고, `PaymentHistory` 상태를 `REFUNDED`로 갱신합니다.
- 환불 실패 시 `ExternalServiceException`으로 롤백되므로, CS에서 재시도하거나 PortOne 대시보드에서 수동 취소한 뒤 재호출하면 됩니다.

## 2. 사용량 기반 환불 계산기
- `RefundCalculator`는 `AbstractSubscriptionCalculator` 기반으로 **7일 이내 미사용 전액 환불** 정책만 허용합니다.
- 파라미터: `SubscriptionEntity`, `MembershipEntity`, 해지일, 사용 로그 존재 여부.
- 작동 흐름
  1. 구독 시작일~해지일까지 사용 일수를 계산.
  2. `UseLogRepository`에서 한 건이라도 사용 이력이 있으면 “사용함”으로 판정.
  3. `startDate` 기준 7일 이내 **그리고 사용 이력이 전혀 없을 때만** 전액 환불.
  4. 위 조건을 만족하지 못하면 환불 불가 메시지를 반환하고 PortOne 취소 요청도 발생하지 않음.
- 예시
  - 월 30,000원, 결제 5일 차 해지, 사용 기록 없음 → **전액 환불**(`refundEligible=true`, `refundAmount=30000`).
  - 결제 5일 차 해지이나 키워드 조회 1건이라도 존재 → “사용 이력이 있어 환불 불가”.
  - 결제 10일 차 해지 → “환불 가능 기간 초과”로 거절.

## 3. 분 단위 테스트 예약 & 스케줄 변수 가이드
- `RecurringPaymentScheduleService`에 테스트용 오버라이드가 추가되었습니다.
- 설정 값 (`application-*.yml`):
  ```yaml
  payment:
    recurring:
      test-mode:
        enabled: ${PAYMENT_TEST_SCHEDULE_ENABLED:false}
        minute-interval: ${PAYMENT_TEST_SCHEDULE_MINUTE_INTERVAL:0}
  ```
- **테스트 절차**
  1. 로컬에서 `PAYMENT_TEST_SCHEDULE_ENABLED=true` 와 `PAYMENT_TEST_SCHEDULE_MINUTE_INTERVAL=5`(예: 5분 후 결제) 환경변수를 지정합니다.
  2. `./gradlew bootRun` 실행 → `RecurringPaymentScheduleService`가 예약 시점을 “현재 + 5분”으로 강제합니다.
  3. PortOne 대시보드에서 5분 단위 예약 및 결제 결과 확인.
  4. 다른 주기를 검증하고 싶으면 interval 값을 변경하거나 OFF로 전환하면 즉시 기본(월/년) 플랜으로 복귀합니다.
- 운영에서는 기본적으로 `enabled=false`로 유지되며, dev/test 프로필에서만 짧은 간격 실험을 허용하는 구조입니다.
