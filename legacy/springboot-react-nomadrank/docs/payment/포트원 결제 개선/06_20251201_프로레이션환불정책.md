# 프로레이션/환불 정책 고도화

## 배경 / 문제
- 환불 정책이 7일 미사용 전액 환불로 단순화되어 다양한 시나리오(사용 중 해지, 업/다운그레이드 시 잔여 일수 정산)를 다루지 못함.
- 업그레이드 차액/환불 로직이 제한적이며 정책이 코드에 하드코딩.

## 목표
- 월/연간, 업/다운그레이드, 즉시 해지/말일 해지 등 주요 시나리오별 프로레이션/환불 규칙을 명시하고 설정화.
- 정책 변경 시 코드 수정 없이 설정/테이블 기반으로 조정 가능하게 함.

## 설계 방향
- 정책 스키마: `refund_policy` 테이블에 `plan`, `cycle`, `grace_days`, `proration_method(DAY|HOUR)`, `usage_deduction`, `full_refund_conditions` 저장.
- 프로레이션 계산기 확장: 인보이스 라인/기간 기반으로 사용량·잔여 일수 산정, 부분 환불/크레딧 발행 지원.
- 해지 플로우: 즉시 해지(부분 환불) vs. 말일 해지(다음 주기 비청구) 옵션을 사용자에게 제공.
- 업/다운그레이드: 차액 청구/크레딧 발행을 인보이스 레벨에서 처리.

## 작업 항목
- [ ] 정책 정의 문서화(시나리오별 기대 금액 예시 포함) 후 비즈니스 확정.
- [ ] `RefundCalculator`, `UpgradeCalculator`를 정책/인보이스 기반으로 재작성.
- [ ] 정책 조회 캐시/파싱 로직 추가 및 단위 테스트 강화(경계값, 윤년/말일 케이스).
- [ ] 사용자/어드민 노출용 환불/정산 금액 프리뷰 API 추가.

## 리스크 / 대응
- 정책 변경에 따른 회계/세금 이슈: 회계/재무 협의 후 정책 잠금 및 버전 관리.
- 계산 복잡도 증가: 단위 테스트/프로퍼티 기반 테스트로 수치 검증.

## 측정 지표
- 환불/차액 계산 오류 문의 건수.
- 예상 금액과 실제 결제/환불 금액 차이 평균.
