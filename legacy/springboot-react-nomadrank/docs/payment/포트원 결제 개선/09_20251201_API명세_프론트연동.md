# 프론트 연동용 API 명세 (인보이스/차지/구독 관련)

## 목적/범위
- 인보이스 조회 전용 API를 프론트에서 사용해 목록/상세/요약 뷰를 구성한다.
- 생성/수정/삭제는 제공하지 않는다(읽기 전용).

## 인증
- 공통: `Authorization: Bearer <token>` (역할: ADMIN 필요, 추후 ROLE_FINANCE 등으로 조정 가능)
- 모든 요청/응답은 `application/json` 기본, CSV 응답 시 `text/plain`.

## 공통 규칙
- 날짜 쿼리: `from`, `to`는 `yyyy-MM-dd` (issuedAt 기준).
- 인보이스 시간 필드: `yyyy-MM-dd HH:mm` (KST, TZ 미포함).
- Charge `paidAt`: `yyyy-MM-dd'T'HH:mm:ss` (LocalDateTime 기본 포맷, TZ 미포함).
- 정렬: `sort` 미지정 시 `issuedAt,DESC` 기본 적용.
- 필터 enum은 영문 대문자 기준.
- statusLabel 매핑: PENDING=대기, PAID=결제완료, VOID=무효, REFUNDED=환불완료 (status가 노출되는 응답에는 statusLabel을 함께 내려줌).
- lineTypeLabel 매핑: PRORATION=일할차액, MEMBERSHIP=멤버십, DISCOUNT=할인, CREDIT=크레딧, ADJUSTMENT=조정, REFUND=환불.
- 정렬 허용 필드/방향: `issuedAt | dueAt | userId | subscriptionId` + `ASC | DESC`. 프런트는 옵션을 그대로 노출하고 기본값(`issuedAt,DESC`)을 명시 전달 권장.

## 인보이스 목록 조회
- `GET /api/invoices`
- Query
  - `status` (optional): `PENDING | PAID | VOID | REFUNDED`
  - `from` / `to` (optional): `yyyy-MM-dd`, issuedAt 필터
  - `userId` (optional): 사용자 ID
  - `subscriptionId` (optional): 구독 ID
  - `lineType` (optional): `MEMBERSHIP | PRORATION | DISCOUNT | CREDIT | ADJUSTMENT | REFUND`
  - `lineDescription` (optional): 라인 description contains 검색
  - `format` (optional, default json): `csv` 지정 시 CSV 반환
  - `page` / `size` / `sort` (Spring Pageable, default issuedAt,DESC, 허용 필드/방향은 상단 공통 규칙 참고)
- Response (json)
  - 페이징 객체(`content`, `totalElements`, `totalPages` ...)
  - `content` 항목: `invoiceId`, `subscriptionId`, `userId`, `status`, `statusLabel`, `amountDue`, `currency`, `issuedAt`, `dueAt`, `paidAt`
  - `lineType` / `lineTypeLabel` (optional): 목록 응답에 라인 합산 정보가 포함될 경우 함께 노출, 없으면 생략
- Response (csv)
  - 헤더: `invoice_id,subscription_id,user_id,status,amount_due,currency,issued_at,due_at,paid_at,line_count,line_amount_sum`
  - `status`는 영문 enum 그대로 출력(한글 변환 없음)

## 인보이스 상세 조회
- `GET /api/invoices/{invoiceId}`
- Response
  - `invoice`: 위와 동일 + `statusLabel`
  - `lines`: 배열
    - `lineId`, `invoiceId`, `lineType` (MEMBERSHIP|PRORATION|DISCOUNT|CREDIT|ADJUSTMENT|REFUND)
    - `lineTypeLabel` (한글 라벨), `description`, `amount`, `currency`
  - `charges`: 배열
    - `paymentId`, `invoiceId`, `userId`, `chargeType`, `paymentType`, `status`, `amount`, `currency`, `paidAt`
    - `chargeType`: `INITIAL | RECURRING | RETRY | ONE_TIME | UPGRADE | PRORATION | ADJUSTMENT | REFUND` (nullable)
    - `paymentType`: `INITIAL | RECURRING | RETRY | ONE_TIME | UPGRADE | DOWNGRADE`
    - `status`: `PENDING | PAID | FAILED | CANCELLED | PARTIAL_CANCELLED | REFUNDED`

## 인보이스 요약 조회
- `GET /api/invoices/summary`
- Query
  - `from` / `to` (optional): `yyyy-MM-dd`
  - `status` (optional): `PENDING | PAID | VOID | REFUNDED`
- Response
  - 배열: `month`(yyyy-MM-01), `totalAmount`, `count`
  - status가 포함될 경우 `statusLabel`을 함께 노출. 요약에 status가 없을 때는 하단 매핑 테이블을 기반으로 프런트에서 라벨링 가능.

## 업그레이드/다운그레이드 프로레이션 라인 정책(프런트 표시용)
- 업그레이드 결제 성공 시
  - PRORATION 라인(양수): `업그레이드 차액 <remaining>/<total>일, <currentPrice>→<targetPrice>`
  - Charge 금액 = 차액, Charge `chargeType=UPGRADE/PRORATION`
- 다운그레이드 예약 시
  - CREDIT 라인(음수): `다운그레이드 크레딧 <remaining>/<total>일, <currentPrice>→<targetPrice>`
  - 결제 없음, 다음 주기부터 새 플랜 라인 생성
- 환불
  - CREDIT/REFUND 라인(음수): `환불: <policyMessage>, 금액=<refundableAmount>`

## 프런트 개발 체크리스트
- 목록/상세 API 호출 시 토큰 포함, 관리자만 접근하도록 UI/라우팅 제한.
- CSV 다운로드 버튼: `format=csv`로 호출 후 파일 저장.
- 인보이스 상세 화면: 라인 타입별 아이콘/컬러 구분(PRORATION는 추가 청구, CREDIT/REFUND는 차감).
- 업/다운/환불 결과 화면: 서버가 반환한 인보이스/라인 정보를 표시해 금액 근거를 투명하게 안내.

## 백엔드에 추가 요청/확인 필요 사항 (프런트 정합성)
아래 선택지 중 맞는 것을 선택해 주세요.
1) 날짜 포맷: `yyyy-MM-dd HH:mm`(TZ 미포함, KST 기준).
2) status enum 응답: 한글로 포맷(예: PENDING→대기, PAID→결제완료, VOID→무효, REFUNDED→환불완료).
3) sort 기본/허용: 기본 `issuedAt,DESC`, 허용 `issuedAt,dueAt,userId,subscriptionId` (page/size 기본 page=0,size=20).
4) 권한: `UserAuthoritySort`(ADMIN, DISTRIBUTOR_MANAGER, USER) 중 ADMIN 전용으로 제한, Authorization 헤더 사용.
5) CSV: 현행 헤더/쉼표/UTF-8 유지.
6) lineType/description: 한글 설명 포함 노출 확정(업/다운/환불 문구 한글).
7) statusLabel/lineTypeLabel: 목록/상세/요약에서 status/lineType가 있으면 라벨을 함께 내려주는 것으로 확정.

## 프런트 전달용 요약 (현재 백엔드 상태)
- 날짜 포맷: `yyyy-MM-dd HH:mm` (KST, TZ 미포함)으로 응답.
- status/statusLabel: status는 영문 enum, statusLabel은 한글(대기/결제완료/무효/환불완료)로 함께 반환.
- 라인 타입: lineType과 lineTypeLabel(일할차액/멤버십/할인/크레딧/조정/환불) 동시 반환, description은 서버 설정 그대로 노출.
- 권한: ADMIN 전용(`@PreAuthorize("hasRole('ADMIN')")`), Authorization 헤더 기반 가정.
- 목록 API: `GET /api/invoices?status&from&to&userId&subscriptionId&lineType&lineDescription&page&size&sort`
  - `sort` 미지정 시 `issuedAt,DESC` 기본 적용, 허용 필드 `issuedAt|dueAt|userId|subscriptionId` 와 `ASC|DESC`.
- 상세 API: `GET /api/invoices/{invoiceId}` (인보이스 + 라인 + 차지).
- 요약 API: `GET /api/invoices/summary` (월별 합계/건수, status가 포함되면 statusLabel 동반).
- CSV: `format=csv` 시 invoice_id,...,line_count,line_amount_sum, 쉼표 구분, UTF-8.
- Charge paidAt은 `yyyy-MM-dd'T'HH:mm:ss` 형식(로컬 기준)으로 내려옴.
