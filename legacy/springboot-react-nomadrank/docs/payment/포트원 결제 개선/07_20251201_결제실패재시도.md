# 결제 실패/재시도/알림 플로우 개선

## 배경 / 문제
- 실패 누적 3회 → `SUSPENDED`로 단순 처리, 실패 원인별 재시도 정책/알림/카드 재인증 흐름이 없음.
- 실패 시 PortOne 예약과 내부 상태를 어떻게 다룰지 명확하지 않아 재시도 중복 가능.

## 목표
- 실패 원인별 재시도 간격/횟수를 세분화하고, 사용자 알림/카드 갱신 UX를 포함한 회복 경로를 제공.
- 실패 이벤트를 멱등하게 처리하며, 재시도 중 중복 청구를 방지.

## 설계 방향
- 실패 원인 맵핑: `INSUFFICIENT_FUNDS`, `CARD_EXPIRED`, `NETWORK_ERROR`, `UNKNOWN` 등 → 재시도 간격/최대 횟수/사용자 액션 정의.
- 재시도 스케줄: PortOne 예약의 재시도 기능 또는 내부 예약 재생성을 활용, 재시도 중엔 단일 락/플래그로 중복 실행 방지.
- 알림: 실패 직후, 최종 실패 전, 정지 시점 알림(이메일/푸시). 카드 만료/교체 CTA 포함.
- 카드 재인증: 새 빌링키 등록 시 기존 예약 삭제 + 신규 예약 재생성.

## 작업 항목
- [ ] 실패 원인 코드 → 정책 매핑 테이블 추가(간격, 횟수, 알림 템플릿 ID).
- [ ] 실패 처리 로직을 정책 기반으로 변경, PortOne 예약 재시도/삭제 여부 결정.
- [ ] 재시도 큐/배치 설계(예: 실패 후 6시간/24시간/72시간).
- [ ] 알림 템플릿/전송 훅 추가, 테스트 더블로 로컬에서 검증.
- [ ] 카드 갱신 플로우 UX/API 정리(기존 예약 cleanup 포함).

## 리스크 / 대응
- 과도한 재시도로 중복 청구 위험: 재시도 시 멱등 키(결제용 paymentId)와 락 사용.
- 알림 스팸: 템플릿/쿨다운 설정 및 사용자 옵트아웃 반영.

## 측정 지표
- 실패 → 성공 전환율, 평균 재시도 횟수.
- 카드 만료/재인증 성공률.
