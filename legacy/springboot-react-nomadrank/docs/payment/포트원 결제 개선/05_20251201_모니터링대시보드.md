# 예약/웹훅 모니터링 대시보드 설계

## 목표
- PortOne 예약/웹훅 처리의 실패율·지연을 한눈에 파악하고, 이상 징후에 알림을 보낸다.
- 멱등 차단/재시도의 효과를 지표로 확인한다.

## 데이터 소스
- `payment_schedule_events`: CREATE/DELETE/EXECUTE/MISS, status(SUCCESS/FAILED), error_code, occurred_at.
- `webhook_events`: source(PAYMENT_PAID/FAILED/CANCELLED/BILLINGKEY_*), status(RECEIVED/PROCESSED/FAILED), processed_at, idempotency_key.
- `subscription_events`(기존): 상태 전이 및 멱등 로그.

## 핵심 지표
- 예약 성공률/실패률: `payment_schedule_events` type=CREATE, status 분포.
- 예약 삭제 실패률: type=DELETE, status 분포.
- 예약 누락 탐지 건수: type=MISS 발생 추이.
- 웹훅 처리 성공률: `webhook_events` status 분포(FAILED 원인 Top N).
- 웹훅 처리 지연: `processed_at - payload.published_at`(원본에 포함 시) 또는 수신~처리 소요 시간.
- 멱등 차단 건수: 동일 `event_id` 재수신 시 처리 스킵 카운트(옵션).

## 대시보드(예: Grafana)
- 패널 예시
  - Time-series: 예약 성공/실패 건수 (stacked), 웹훅 처리 성공/실패 건수.
  - Single stat: 최근 1h/24h 예약 실패율, 웹훅 실패율.
  - Table: 최근 50건 실패 이벤트(타입, 코드, 메시지, user/schedule/paymentId).
  - Alert rule: 예약 실패율 > X%, 웹훅 실패율 > Y%, MISS 발생 시 Slack/Email 알림.
- 템플릿 변수: event_type, status, source, error_code.

## 알림 채널
- Slack(운영 채널), Email(옵션), Webhook(Ops 툴).
- 임계치: 5분 이동평균 실패율, MISS 발생 즉시 알림, 웹훅 처리 지연 p95 기준 임계.

## 운영 가이드
- 민감정보: payload 저장 시 카드/연락처 마스킹 또는 화이트리스트 필드만 저장.
- 보관 기간: 이벤트 로그 90일, 필요 시 S3 아카이빙/파티셔닝.
- 재시도/장애 대응: 대시보드에서 실패 이벤트 drill-down 후 수동 재시도 혹은 보상 로직 실행.
