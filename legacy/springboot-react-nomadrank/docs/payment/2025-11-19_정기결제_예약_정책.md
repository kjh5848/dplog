# PortOne 정기 결제 예약 운영 계획

## 1. 목적
- 빌링키 기반 정기 결제 예약을 모든 시나리오(최초 가입, 업/다운그레이드, 주기 변경)에서 일관되게 관리한다.
- PortOne 스케줄과 내부 `Subscription` 정보를 항상 동기화해 결제 금액·주기·다음 결제일 불일치를 제거한다.

## 2. 공통 처리 흐름
1. `RecurringPaymentScheduleService`가 PortOne API 호출(조회/삭제/생성)을 전담한다.
2. 예약 재구성이 필요하면 아래 순서로 수행한다.
   1. 활성 빌링키 조회 → 상태 확인 (`BillingKeyStatus.ACTIVE`).
   2. 기존 PortOne 예약 전체 조회 후 삭제(404 무시, 기타 오류는 사용자에게 안내).
   3. 다음 결제일/주기/금액을 계산해 새 예약을 생성한다.
   4. 새 예약 ID는 로그로만 추적하고, `Subscription` PK는 유지한다(내부 식별자 안정화 목적).

## 3. 시나리오별 동작

### 3.1 최초 빌링키 발급 & 첫 결제 완료 (`PaymentServiceApiV1#completeBillingKey`)
- 첫 결제 성공 시 멤버십을 ACTIVATE → 다음 결제일 계산 → `RecurringPaymentScheduleService.createReservation`으로 즉시 PortOne 예약.
- 예약 생성 실패 시 결제도 보상 취소(`compensateActivationFailure`).

### 3.2 업그레이드 (`SubscriptionServiceApiV1#upgradeSubscription`)
| 케이스 | 처리 순서 |
| --- | --- |
| 월→월 업그레이드 | 차액 결제가 존재하면 우선 수납 → `subscription.amount/billingCycle/nextBillingDate` 갱신 → 기존 예약 전체 삭제 후 새 예약 생성 → 최신 금액·주기로 PortOne에 재예약 |
| 월→연 업그레이드 | 연간 주기로 `nextBillingDate` 계산 → 월간 예약 삭제 → 연간 금액/`interval=YEAR`로 재예약 |
| 추가 결제 없는 업그레이드 | `applyUpgradeWithoutPayment`에서도 동일하게 예약 재생성 수행 |

### 3.3 다운그레이드 (`SubscriptionServiceApiV1#scheduleDowngrade` + `finalizePendingPlanIfNecessary`)
1. 요청 시점에는 `pending_*` 필드에 대상 플랜을 저장하고 다음 결제일에 적용 예정 안내.
2. 실제 결제 성공 직후 `finalizePendingPlanIfNecessary`가 호출되면:
   - 멤버십/요금/주기를 새 값으로 반영.
   - `RecurringPaymentScheduleService.deleteAllSchedules` 후 다운된 금액으로 다시 예약.
   - 예약 성공 시 `pending_*` 필드 초기화.

### 3.4 구독 해지
- 해지 예약 시 빌링키 기준으로 PortOne 스케줄을 일괄 삭제(`RecurringPaymentScheduleService.fetch/delete`).
- 삭제에 실패하면 사용자에게 “예약 결제 삭제 실패” 메시지를 반환하고 재시도 유도.

## 4. 예외 처리 정책
- PortOne 404 응답은 “이미 삭제됨”으로 간주하고 무시한다.
- 그 외 PortOne 오류는 `BadRequestException` 또는 `ExternalServiceException`으로 감싸 사용자/운영자에게 노출.
- 예약 생성 실패 시에는 즉시 트랜잭션을 롤백해 DB/PortOne 상태 불일치를 방지한다.
- 빌링키가 비활성(`status != ACTIVE`)이면 예약 갱신을 생략하고 워닝 로그만 남긴다.

## 5. 코드 변경 포인트
- `RecurringPaymentScheduleService`: 예약 조회/삭제/생성을 담당하는 신규 컴포넌트.
- `PaymentServiceApiV1`:
  - `subscribe`, `completeBillingKey`, `cancelSubscription`이 위 컴포넌트를 사용.
- `SubscriptionServiceApiV1`:
  - 업그레이드(차액 결제 여부와 무관)와 다운그레이드 확정 시 PortOne 예약을 자동 재구성.
- 문서/로그에 모든 재예약 액션을 `[RecurringSchedule]` 태그로 남겨 추적성을 확보.
