# 결제 보상(취소) 플로우

## 목적
- 포트원 결제는 성공했지만 내부 구독/멤버십 처리가 실패한 경우 사용자에게 부담을 주지 않도록 즉시 결제를 취소한다.
- 취소 결과를 `PaymentHistory`에 반영하고, 추가 조치가 필요한 경우 운영자가 추적할 수 있도록 에러 메시지를 남긴다.

## 트리거
- `PaymentServiceApiV1.completeBillingKey`에서 첫 결제(빌링키 결제)가 성공한 뒤 `activateUserMembershipAfterPayment` 또는 `createSubscription` 단계에서 예외가 발생한 경우.

## 처리 단계
1. **결제 취소 요청**  
   - `PaymentServiceApiV1.compensateActivationFailure`가 `PortOneV2Client.cancelPayment`를 호출해 `paymentId` 기준으로 취소 API를 요청한다.
   - 취소 금액은 최초 결제 금액과 동일하게 전달한다.
2. **결제 이력 갱신 (REQUIRES_NEW 트랜잭션)**  
   - 취소 성공 시 `PaymentHistory.markAsCancelled` 호출로 상태를 `CANCELLED`로 변경하고 `cancelledAt`을 기록한다.
   - 취소 실패 시 `PaymentHistory.errorMessage`에 실패 사유를 저장해 운영자 확인이 필요함을 남긴다.
3. **사용자 응답**  
   - 취소 성공: “구독 처리 중 오류가 발생하여 결제를 취소했습니다. 잠시 후 다시 시도해주세요.”  
   - 취소 실패: “구독 처리 중 오류가 발생했고 결제 취소에 실패했습니다. 관리자에게 문의해주세요.”

## 로그&모니터링
- 결제 취소 성공/실패 로그를 INFO/ERROR 레벨로 기록한다.
- 취소 실패 시 원인(HTTP 상태, PortOne 메시지, 예외 메시지)을 함께 출력해 운영자가 즉시 확인 가능하도록 한다.
