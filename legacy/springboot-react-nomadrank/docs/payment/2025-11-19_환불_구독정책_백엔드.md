# 환불 및 구독 정책 (백엔드 관점)

본 문서는 현재 레포지토리의 코드/데이터 흐름을 기반으로 **환불(구독 해지)**, **다운그레이드**, **업그레이드** 정책을 정리하고, 구현과 정책 간 간극을 명확히 드러내기 위한 개발 문서입니다. 운영/기획 정책이 변경될 경우 해당 문서를 갱신하고, 필요한 개발 작업을 식별하는 것이 목적입니다.

---

## 1. 업그레이드 (현재 상태)

| 항목 | 내용 |
| --- | --- |
| 서비스 계층 | `SubscriptionServiceApiV1.upgradeSubscription` |
| 조건 | (1) 상위 멤버십, (2) 월간→연간 전환만 허용 |
| 차액 계산 | `UpgradeCalculator` Hybrid 모델 (남은 기간×사용량) |
| 결제 | `PortOneV2Client.payWithBillingKey` 호출 → 차액 결제 |
| 실패 처리 | 결제 실패 시 예외 발생, `compensateActivationFailure`로 보상 |
| 로그/DB | `SubscriptionEntity`, `MembershipUserLogEntity`, `PaymentHistoryEntity` 갱신 |

현 구현은 정책과 일치하며, 추가 개발 필요 없음.

---

## 2. 다운그레이드 (미구현 영역)

| 항목 | 내용 |
| --- | --- |
| 현재 상태 | 자동 다운그레이드 API/예약/스케줄러 모두 미구현 |
| 문서 정책 | 일부 문서에서 “다음 결제일부터 자동 적용”으로 안내 → 실제와 불일치 |
| 임시 대응 | 해지 후 하위 플랜 재가입 또는 CS 수동 처리 |

### 2.1 필요한 개발 과제
1. **데이터 모델**: `subscription_change_request`(가칭) 엔티티 설계 (다운그레이드 예약 정보 저장).
2. **서비스 로직**: `SubscriptionServiceApiV1`에 다운그레이드 예약 API 추가 (`downgradeSubscription`).
3. **스케줄러**: ChangeRequestProcessor 확장 → `status=REQUESTED` + `requestedStartDate <= today` 처리.
4. **환불 정책 확정**: 선불 유지 vs. 부분 환불 중 택일. 환불 지원 시 PortOne 결제 취소/부분 환불 연동 필요.
5. **UI/문서 정비**: 기능 출시 전까지는 “자동 다운그레이드 미지원” 안내 유지.

### 2.2 임시 문서화 방안
- 현재 기능 불가 → 정책/FAQ에 “다운그레이드는 해지 후 재가입 또는 고객센터 문의”로 명시.
- 개발 일정 확정 시 `docs/payment/subscription-downgrade-design.md` 참고.

## 3. 환불(구독 해지) – 구현 vs 정책

| 항목 | 현재 구현 | 정책 문서 요구사항 |
| --- | --- | --- |
| 해지 처리 | `PaymentServiceApiV1.cancelSubscription`에서 즉시 해지<br/>- PortOne 예약 삭제<br/>- PortOne 빌링키 삭제<br/>- `SubscriptionEntity.status = CANCELLED`, `endDate = LocalDate.now()`<br/>- `MembershipUser` 상태 `EXPIRED` | 결제된 기간 동안 서비스 이용 가능<br/>/ 부분 환불 없음 |
| 환불/취소 | 자동 환불 로직 없음. PortOne 결제 취소 호출도 미구현 | 부분 환불/즉시 환불 없음 (코멘트 일치) |
| 예외 환불 | CS 수동 처리 전제 | 정책에서 “장애 시 환불 가능” → 수동 프로세스 필요 |

### 3.1 정합성 이슈
- 정책 문서에는 “이미 결제한 기간 동안 계속 이용 가능”이라고 되어 있지만, **현재 코드는 해지 즉시 사용 종료**.
- 해결 방법:
  1. **코드 수정**: 해지 로직을 변경하여 `endDate`를 `nextBillingDate`로 유지하고, 멤버십 상태 변경을 결제 주기 종료 시점에 수행. → 기존 로직 영향 분석 필요 (예: BillingKey 삭제 시 정기 결제 스케줄 취소).
  2. **정책 수정**: “해지 즉시 이용 종료”로 안내 (향후 개선 전까지 사용자에게 명확히 알림).

### 3.2 향후 개발 항목
1. 해지 로직 보완: `endDate` 유지, 접근 권한 체크 로직 개선.
2. 환불 자동화: 포트원 결제 취소 API 연동 (`PortOneV2Client.cancelPayment`/부분 취소) 고려.
3. 장애 환불 프로세스 문서화: CS/재무팀용 Runbook 작성.
