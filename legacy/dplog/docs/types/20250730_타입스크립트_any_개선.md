# TypeScript any 타입 개선 작업

## 📋 개요

이 문서는 프로젝트에서 `any` 타입 사용을 제한하고 타입 안전성을 향상시키기 위한 개선 작업을 기록합니다.

## 🎯 목표

- `any` 타입 사용 최소화
- 타입 안전성 향상
- 코드 가독성 및 유지보수성 개선
- 런타임 오류 방지

## ✅ 완료된 개선 사항

### 1. 로거 유틸리티 개선
- **파일**: `src/utils/logger.ts`
- **개선 내용**:
  - `LogContext` 인터페이스의 `any` → `unknown` 변경
  - `apiResponse` 메서드에 제네릭 타입 추가
  - `logApiResponse` 함수에 제네릭 타입 추가

### 2. 인증 관련 타입 개선
- **파일**: `src/viewModel/useAuthLoginViewModelLocal.tsx`, `src/viewModel/useAuthJoinViewModelLocal.tsx`
- **개선 내용**:
  - 함수 매개변수 `any` → 구체적 타입 (`LoginRequest`, `JoinRequest`)
  - 에러 처리 `any` → `unknown` + 타입 가드
  - `src/types/auth.ts`에 `JoinRequest` 인터페이스 추가

### 3. Track 관련 타입 개선
- **파일**: `src/viewModel/nplace/track/nplaceRankTrackViewModel.ts`, `src/model/TrackRepository.ts`
- **개선 내용**:
  - `TrackGroup` 타입 정의 및 적용
  - API 응답 타입 구체화
  - 제네릭을 사용한 타입 안전성 향상

### 4. 컴포넌트 타입 개선
- **파일**: `src/components/nplrace/rank/track/TrackFilter.tsx`
- **개선 내용**:
  - `groupList: any[]` → `groupList: TrackGroup[]`
  - `onChangeGroupModalSubmit: (data: any)` → `(data: GroupChangeData)`
  - 에러 처리 개선

### 5. 차트 컴포넌트 타입 개선
- **파일**: `src/components/nplrace/rank/track/id/(report)/TrackReportView.tsx`
- **개선 내용**:
  - `ChartPayload` 인터페이스 정의
  - `payload: any[]` → `payload: ChartPayload[]`
  - `safeNumber` 함수의 매개변수 `any` → `unknown`

### 6. 엑셀 업로드 컴포넌트 개선
- **파일**: `src/components/nplrace/rank/track/ExcelUploadModal.tsx`
- **개선 내용**:
  - `(row: any)` → `(row: unknown[])`

### 7. 프로필 컴포넌트 개선
- **파일**: `src/components/profile/ProfileView.tsx`
- **개선 내용**:
  - `(loginUser as any).email` → `loginUser.email`
  - LoginUser 인터페이스에 누락된 필드 추가

### 8. 결제 페이지 개선
- **파일**: `src/app/(home)/payment/page.tsx`
- **개선 내용**:
  - `TossPaymentsError` 인터페이스 정의
  - `(error as any).code` → `error as TossPaymentsError`

## 📊 개선 통계

### Before (개선 전)
- `any` 타입 사용: 50+ 곳
- 타입 단언 `as any`: 10+ 곳
- 에러 처리: `catch (error: any)`

### After (개선 후)
- `any` 타입 사용: 5곳 (외부 라이브러리만)
- 타입 단언 `as any`: 2곳 (필수적인 경우만)
- 에러 처리: `catch (error: unknown)` + 타입 가드

## 🔧 적용된 패턴

### 1. 제네릭 사용
```typescript
// Before
function processResponse(response: any) { }

// After
function processResponse<T>(response: ApiResponse<T>) { }
```

### 2. 인터페이스 정의
```typescript
// Before
interface User {
  data: any;
}

// After
interface User {
  data: UserData;
}
```

### 3. 타입 가드 사용
```typescript
// Before
} catch (error: any) {
  console.error(error.message);
}

// After
} catch (error: unknown) {
  if (error instanceof Error) {
    console.error(error.message);
  } else {
    console.error('Unknown error occurred');
  }
}
```

### 4. unknown 타입 사용
```typescript
// Before
[key: string]: any;

// After
[key: string]: unknown;
```
