# 환경 변수 설정 가이드

## 기본 설정

`.env.local` 파일을 생성하고 다음 환경 변수를 설정하세요:

```env
# API 엔드포인트
NEXT_PUBLIC_API_URL=https://api.dplog.kr

# 사업자등록번호 조회 API (선택사항)
NEXT_PUBLIC_BUSINESS_API_KEY=YOUR_API_KEY_HERE

# Sentry 설정 (에러 추적 및 모니터링)
NEXT_PUBLIC_SENTRY_DSN=YOUR_SENTRY_DSN_HERE
SENTRY_DSN=YOUR_SENTRY_DSN_HERE

# 앱 버전 (선택사항)
NEXT_PUBLIC_APP_VERSION=1.0.0
APP_VERSION=1.0.0
```

## Sentry 설정 방법

### 1. Sentry 프로젝트 생성
1. [Sentry.io](https://sentry.io)에 가입
2. 새 프로젝트 생성 (Next.js 템플릿 선택)
3. DSN 키 복사

### 2. 환경 변수 설정
```env
# 클라이언트용 DSN (브라우저에서 사용)
NEXT_PUBLIC_SENTRY_DSN=https://your-dsn@sentry.io/project-id

# 서버용 DSN (서버에서 사용)
SENTRY_DSN=https://your-dsn@sentry.io/project-id
```

### 3. 프로덕션 배포 시
- 환경 변수가 올바르게 설정되었는지 확인
- Sentry 대시보드에서 에러 수신 확인

## 환경별 설정

### 개발 환경 (.env.development)
```env
NODE_ENV=development
NEXT_PUBLIC_API_URL=http://localhost:3000/api
NEXT_PUBLIC_SENTRY_DSN=  # 개발 환경에서는 비워둠
```

### 프로덕션 환경 (.env.production)
```env
NODE_ENV=production
NEXT_PUBLIC_API_URL=https://api.dplog.kr
NEXT_PUBLIC_SENTRY_DSN=https://your-dsn@sentry.io/project-id
SENTRY_DSN=https://your-dsn@sentry.io/project-id
```

## 보안 주의사항

- `.env.local` 파일은 절대 Git에 커밋하지 마세요
- 프로덕션 DSN은 안전하게 관리하세요
- 민감한 정보는 환경 변수로 관리하세요
