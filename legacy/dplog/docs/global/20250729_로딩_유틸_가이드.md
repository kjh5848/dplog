# 로딩 & 인증 유틸리티 가이드

## 📋 개요

이 가이드는 D-PLOG 프로젝트에서 사용하는 로딩과 인증 관련 유틸리티 함수들의 사용법을 설명합니다.

## 🎯 목적

- **일관된 로딩 메시지**: 모든 페이지에서 동일한 로딩 메시지 표시
- **표준화된 인증 처리**: 인증 상태에 따른 일관된 처리 로직
- **사용자 경험 개선**: 상황에 맞는 적절한 피드백 제공
- **코드 재사용성**: 중복 코드 제거 및 유지보수성 향상

## 🔐 인증 유틸리티 (`src/utils/auth.ts`)

### 기본 사용법

```typescript
import { useAuthStatus, useGuestRedirect, useAuthGuard } from '@/src/utils/auth';

// 컴포넌트에서 사용
function MyComponent() {
  const { checkAuth, isLogoutPending } = useAuthGuard();
  
  const authResult = checkAuth();
  
  if (authResult.shouldShowLoading) {
    return <LoadingFallback config={loadingUtils.guestAuth()} />;
  }
}
```

### 인증 상태 확인 (`useAuthStatus`)

```typescript
const { 
  loginUser,        // 로그인된 사용자 정보
  isAuthPending,    // 인증 확인 중 여부
  isLogoutPending,  // 로그아웃 처리 중 여부
  isLoading,        // 로딩 상태 (인증 확인 중)
  isGuest,          // 게스트 상태 (로그인 안됨)
  isAuthenticated   // 인증된 사용자 여부
} = useAuthStatus();
```

### 게스트 리다이렉트 (`useGuestRedirect`)

```typescript
const { isGuest, isLogoutPending, handleGuestRedirect } = useGuestRedirect();

// 게스트 리다이렉트 처리
const handleRedirect = () => {
  if (isGuest) {
    // 로그아웃 중이 아닐 때만 토스트 메시지 표시
    handleGuestRedirect();
  }
};
```

**특징:**
- 로그아웃 중일 때는 토스트 메시지 표시하지 않음
- 중복 토스트 방지를 위한 ID 설정
- 자동 리다이렉트 처리

### 인증 가드 (`useAuthGuard`)

```typescript
const { 
  isLoading, 
  isGuest, 
  isAuthenticated, 
  isLogoutPending,
  checkAuth,
  isLoggingOut 
} = useAuthGuard();

// 인증 상태 확인
const authResult = checkAuth();

// 결과 객체
interface AuthResult {
  shouldShowLoading: boolean;  // 로딩 컴포넌트 표시 여부
  shouldRedirect: boolean;     // 리다이렉트 발생 여부
}
```

## 🎨 LoadingFallback 컴포넌트

### Props

```typescript
interface LoadingFallbackProps {
  message?: string;                                    // 직접 메시지
  size?: 'sm' | 'md' | 'lg';                          // 직접 크기
  type?: LoadingType;                                  // 로딩 타입
  config?: { message: string; size: 'sm' | 'md' | 'lg' }; // 설정 객체
}
```

### 설정 우선순위

1. **config** (최우선)
2. **type** + message/size
3. **message** + size (기본값)

### 사용 예시

```typescript
// 1. 설정 객체 사용 (권장)
<LoadingFallback config={loadingUtils.userAuth()} />

// 2. 타입 사용
<LoadingFallback type="auth" message="커스텀 메시지" />

// 3. 직접 props 사용
<LoadingFallback message="로딩 중..." size="lg" />

// 4. 로그아웃 상태별 처리
{isLogoutPending ? (
  <LoadingFallback config={loadingUtils.logoutAuth()} />
) : (
  <LoadingFallback config={loadingUtils.guestAuth()} />
)}
```

## 📝 실제 사용 예시

### 1. 기본 페이지 컴포넌트

```typescript
"use client";

import { useAuthGuard } from '@/src/utils/auth';
import { loadingUtils } from '@/src/utils/loading';
import LoadingFallback from '@/src/components/common/LoadingFallback';

export default function MyPage() {
  const { checkAuth, isLogoutPending } = useAuthGuard();
  
  // 인증 상태 확인
  const authResult = checkAuth();
  
  if (authResult.shouldShowLoading) {
    // 로그아웃 중일 때는 다른 메시지 표시
    if (isLogoutPending) {
      return <LoadingFallback config={loadingUtils.logoutAuth()} />;
    }
    return <LoadingFallback config={loadingUtils.guestAuth()} />;
  }

  return (
    <div>
      {/* 페이지 콘텐츠 */}
    </div>
  );
}
```

### 2. 데이터 로딩 처리

```typescript
function DataComponent() {
  const [isLoading, setIsLoading] = useState(false);
  
  if (isLoading) {
    return <LoadingFallback config={loadingUtils.dataLoading()} />;
  }
  
  return <div>데이터 표시</div>;
}
```

### 3. 폼 로딩 처리

```typescript
function LoginForm() {
  const [isSubmitting, setIsSubmitting] = useState(false);
  
  if (isSubmitting) {
    return <LoadingFallback config={loadingUtils.formLoading()} />;
  }
  
  return <form>로그인 폼</form>;
}
```

### 4. 파일 업로드 처리


## 🔄 마이그레이션 가이드

### 기존 코드에서 새로운 유틸리티로 변경

#### Before (기존)
```typescript
// 기존 방식
const { loginUser, isAuthPending } = useAuthStore();
const isLoading = loginUser === undefined || isAuthPending;
const isGuest = loginUser === null;

if (isLoading) {
  return <LoadingFallback message="사용자 인증 확인 중..." />;
}

if (isGuest) {
  toast.error("로그인 후 이용해주세요.");
  redirect("/login");
  return null;
}
```

#### After (개선)
```typescript
// 새로운 방식
const { checkAuth, isLogoutPending } = useAuthGuard();
const authResult = checkAuth();

if (authResult.shouldShowLoading) {
  if (isLogoutPending) {
    return <LoadingFallback config={loadingUtils.logoutAuth()} />;
  }
  return <LoadingFallback config={loadingUtils.guestAuth()} />;
}
```

### 변경 사항 요약

1. **복잡한 조건문 → 단순한 함수 호출**
2. **중복 코드 → 재사용 가능한 유틸리티**
3. **일관성 없는 메시지 → 표준화된 메시지**
4. **수동 토스트 처리 → 자동 처리**

## 🔧 문제 해결

### Q: 로그아웃 후에도 "로그인 후 이용해주세요" 메시지가 표시됩니다.
**A:** `isLogoutPending` 상태를 확인하여 로그아웃 중일 때는 토스트 메시지를 표시하지 않도록 처리합니다.

### Q: 로딩 메시지가 일관되지 않습니다.
**A:** `loadingUtils`의 표준 함수를 사용하여 일관된 메시지를 표시합니다.

### Q: 인증 체크 로직이 복잡합니다.
**A:** `useAuthGuard`의 `checkAuth()` 함수를 사용하여 단순화합니다.

### Q: 커스텀 로딩 메시지가 필요합니다.
**A:** `createLoadingConfig()` 함수를 사용하여 타입 기반의 커스텀 메시지를 생성합니다.
