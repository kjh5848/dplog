# 폴더 구조 가이드

## 개요
프로젝트의 폴더 구조를 명확하게 분리하여 유지보수성과 확장성을 향상시켰습니다.

## 폴더 구조

### `/src/hooks/` - 순수 UI 훅
React UI와 관련된 순수한 훅들을 포함합니다.

```
/src/hooks/
├── useCountUp.ts          # 카운트 애니메이션 훅
├── useInViewCount.ts      # 뷰포트 카운트 훅
├── useMediaQuery.ts       # 미디어 쿼리 훅
├── useScrollAnimation.ts  # 스크롤 애니메이션 훅
└── index.ts              # 모든 UI 훅 export
```

**특징:**
- React UI 로직에만 집중
- 순수 함수적 접근
- 재사용 가능한 UI 컴포넌트

### `/src/features/` - 도메인별 비즈니스 로직 훅
특정 도메인이나 기능과 관련된 복잡한 비즈니스 로직을 포함합니다.

```
/src/features/
├── common/               # 공통 비즈니스 로직
│   ├── useDebounce.ts    # 디바운스 훅
│   ├── useMediaQuery.ts  # 미디어 쿼리 훅 (useIsMobile 포함)
│   ├── usePendingFunction.ts # 펜딩 상태 훅
│   └── index.ts
├── track/                # 트래킹 관련 로직
│   ├── useTrackContent.ts # 트래킹 컨텐츠 훅
│   └── index.ts
├── auth/                 # 인증 관련 로직 (향후 확장)
│   └── index.ts
└── index.ts              # 모든 features export
```

**특징:**
- 도메인별로 분리된 비즈니스 로직
- 복잡한 상태 관리
- API 통신과 연동

### `/src/utils/` - 순수 유틸리티 함수
React와 무관한 순수 함수들을 포함합니다.

```
/src/utils/
├── api/                  # API 관련 유틸리티
│   ├── responseHandler.ts # API 응답 처리
│   ├── kakaoSdk.ts       # 카카오 SDK 유틸리티
│   └── index.ts
├── browser/              # 브라우저 관련 유틸리티
│   ├── cookieUtils.ts    # 쿠키 관리 유틸리티
│   └── index.ts
├── report/               # 리포트 생성 유틸리티
│   ├── reportHelper.ts   # 리포트 헬퍼
│   ├── CanvasRenderer.ts # 캔버스 렌더러
│   ├── FileDownloader.ts # 파일 다운로더
│   ├── interfaces/       # 인터페이스 정의
│   └── index.ts
└── index.ts              # 모든 utils export
```

**특징:**
- React와 무관한 순수 함수
- 재사용 가능한 유틸리티
- 테스트하기 쉬운 구조

## 사용 가이드

### 훅 선택 기준

1. **UI 관련 로직** → `/src/hooks/`
   ```typescript
   import { useScrollAnimation } from '@/src/hooks';
   ```

2. **도메인별 비즈니스 로직** → `/src/features/`
   ```typescript
   import { useIsMobile } from '@/src/hooks/useMediaQuery';
   import { useTrackContent } from '@/src/features/track';
   ```

3. **순수 유틸리티 함수** → `/src/utils/`
   ```typescript
   import { processApiResponse } from '@/src/utils/api';
   import { getCookie } from '@/src/utils/browser';
   ```

### Import 경로 규칙

- **직접 import**: 특정 기능만 필요한 경우
  ```typescript
  import { useIsMobile } from '@/src/hooks/useMediaQuery';
  ```

- **Index import**: 여러 기능을 사용하는 경우
  ```typescript
  import { useIsMobile, useDebounce } from '@/src/hooks';
  ```

## 마이그레이션 완료 사항

### 이동된 파일들
- `/src/use/useDebounce.tsx` → `/src/features/common/useDebounce.ts`
- `/src/use/useMobile.tsx` → `/src/hooks/useMediaQuery.ts`
- `/src/use/usePendingFunction.tsx` → `/src/features/common/usePendingFunction.ts`
- `/src/use/useTrackContent.ts` → `/src/features/track/useTrackContent.ts`

### 이동된 유틸리티들
- `/src/utils/responseHandler.ts` → `/src/utils/api/responseHandler.ts`
- `/src/utils/kakaoSdk.ts` → `/src/utils/api/kakaoSdk.ts`
- `/src/utils/cookieUtils.ts` → `/src/utils/browser/cookieUtils.ts`
- `/src/utils/reportHelper.ts` → `/src/utils/report/reportHelper.ts`

### 업데이트된 Import 경로들
모든 파일의 import 경로가 새로운 구조에 맞게 업데이트되었습니다.

## 향후 확장 계획

1. **Auth Features**: 인증 관련 복잡한 로직을 `/src/features/auth/`에 추가
2. **Payment Features**: 결제 관련 로직을 `/src/features/payment/`에 추가
3. **Analytics Features**: 분석 관련 로직을 `/src/features/analytics/`에 추가

## Cursor Rules 적용

각 폴더에 대한 Cursor Rules가 생성되어 자동으로 적용됩니다:

- `.cursor/rules/hooks-patterns.mdc` - UI 훅 패턴
- `.cursor/rules/utils-patterns.mdc` - 유틸리티 패턴
- `.cursor/rules/model-repository-patterns.mdc` - 모델 패턴
- `.cursor/rules/types-definitions.mdc` - 타입 정의 패턴
