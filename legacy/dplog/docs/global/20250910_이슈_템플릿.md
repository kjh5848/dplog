### 문제점
1. **Hook 순서 불일치**: 조건부 렌더링이 Hook 호출 사이에 있어서 Hook 순서가 일관되지 않음
2. **React Hooks 규칙 위반**: 모든 Hook이 컴포넌트 최상단에서 호출되지 않음
3. **렌더링 불안정성**: 홈페이지에서 검색창을 통해 URL 이동 시 오류 발생

## ✅ 해결 방안

### 수정된 코드 구조
```typescript
export default function TrackClientPage() {
  const router = useRouter();
  const { isLoading, isGuest, isLogoutPending } = useAuthGuard();
  
  // ✅ 모든 Hook을 최상단에서 호출
  const { error, refetchShopList } = useNplaceRankTrackViewModel();
  
  useEffect(() => {
    if (error) {
      toast.error("데이터를 불러오는 중 오류가 발생했습니다.");
    }
  }, [error]);

  useEffect(() => {
    if (isGuest && !isLogoutPending) {
      router.push("/login");
    }
  }, [isGuest, isLogoutPending, router]);

  // ✅ Hook 호출 후에 조건부 렌더링
  if (isLoading) {
    return <LoadingFallback config={loadingUtils.guestAuth()} />;
  }

  if (isGuest) {
    return <LoadingFallback config={loadingUtils.loginRedirect()} />;
  }

  return (
    <main className="container min-h-screen">
      <Suspense fallback={<LoadingFallback config={loadingUtils.pageLoad()} />}>
        <TrackContent />
      </Suspense>
    </main>
  );
}
```

### 주요 변경사항
1. **Hook 호출 순서 통일**: 모든 Hook을 컴포넌트 최상단에서 호출
2. **조건부 렌더링 위치 조정**: Hook 호출 후에 조건부 렌더링 처리
3. **코드 구조 개선**: 일관된 패턴 적용
