# 실시간 조회 최근 검색 히스토리 정렬/TTL 계획

## 목표
- 실시간 조회 페이지(`/realtime`)의 최근 검색 UX를 키워드 조회와 동일한 형태로 통일하고, 멤버십 등급에 따라 TTL/최대 보관 개수를 적용한다.

## 적용 정책 (멤버십별)
- 무료(FREE): TTL 1일, 최대 3개
- 실속(SILSOK, 레벨 1): TTL 7일, 최대 10개
- 성장(SEONGJANG, 레벨 2): TTL 30일, 최대 30개
- 마스터(MASTER, 레벨 3): TTL 무제한, 최대 50개
- 동일 검색 파라미터(지역+검색타입+값+키워드)는 최신 결과로 교체, 최신순 정렬 후 초과분 제거.

## 구현 포인트
- 멤버십 판정: `/v1/membership/current` 응답의 `membershipId`/`membershipName`을 사용해 레벨 매핑(`resolveMembershipLevel`).
- 매핑 기준: 체험/무료(id=2) → FREE(0), 실속(id=11) → SILSOK(1), 성장(id=12) → SEONGJANG(2), 마스터(id=13) → MASTER(3). 이름 패턴(체험/무료/실속/성장/마스터)도 함께 매칭.
- 저장/로딩 함수에 레벨 전달:
  - `saveRecentSearch(params, level)` / `getRecentSearches(level)`
  - `saveSearchResults(params, data, level)` / `getRecentSearchResults(level)`
  - `cleanupWithPolicy`로 TTL 및 개수 적용, 최신순 정렬.
- 중복 키 기준: `filterType + filterValue + keyword + province` 조합이 같으면 최신 항목으로 교체.
- UI/UX: 키워드 조회 히스토리 UI와 동일한 카드/리스트 스타일 유지. 최신순 표시, 타임스탬프 노출.
- 업셀링: 한도 초과 시 최근 검색 보기 CTA 유지, 멤버십 업그레이드 링크 노출.

## UI 정렬 계획 (키워드 조회와 동일)
- 컴포넌트: `RealtimeForm`의 최근 검색 섹션을 리스트형으로 교체 (키워드 `최근 조회` 섹션 스타일 복제).
- 구조:
  - 헤더: “최근 검색” + `기록 삭제` 버튼.
  - 리스트 아이템: 좌측에 검색 타입/값/키워드/지역 표시, 우측에 타임스탬프. `button` 형태로 클릭 시 즉시 재검색.
  - 빈 상태: “최근 검색 기록이 없습니다.” 안내.
- 스타일: `rounded-md bg-white px-3 py-2 hover:bg-gray-100` 리스트, 컨테이너는 `rounded-lg border border-gray-100 bg-gray-50 p-3`.
- 데이터: `recentSearchResults`를 최신순으로 사용, 타임스탬프는 `toLocaleString()` 표기.

## 영향 범위
