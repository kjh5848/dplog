# Server Actions 아키텍처 설계

## 1. 개요

현재 프로젝트는 외부 API 서버(`https://api.dplog.kr`)를 직접 호출하는 Repository 패턴을 사용하고 있습니다. Server Actions 도입 시 기존 Repository 레이어는 그대로 유지하면서, 새로운 Server Actions 레이어를 추가하여 점진적 마이그레이션을 가능하게 합니다.

## 2. 아키텍처 구조

```
┌─────────────────────────────────────────────────────────────┐
│                    Client Components                        │
│  (React Components with useTransition, useFormState)       │
└─────────────────────┬───────────────────────────────────────┘
                      │
┌─────────────────────▼───────────────────────────────────────┐
│                  Server Actions                             │
│  (app/actions/ - 서버 사이드 데이터 처리)                   │
└─────────────────────┬───────────────────────────────────────┘
                      │
┌─────────────────────▼───────────────────────────────────────┐
│                Repository Layer                             │
│  (src/model/ - 기존 Repository 클래스들)                    │
└─────────────────────┬───────────────────────────────────────┘
                      │
┌─────────────────────▼───────────────────────────────────────┐
│                External API Server                          │
│  (https://api.dplog.kr)                                     │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 Server Actions 패턴

#### 기본 패턴
```typescript
// app/actions/auth/login.ts
'use server'

import { AuthRepository } from '@/src/model/AuthRepository';
import { LoginRequest } from '@/src/types/auth';
import { revalidatePath } from 'next/cache';
import { redirect } from 'next/navigation';

export async function loginAction(formData: FormData) {
  try {
    const email = formData.get('email') as string;
    const password = formData.get('password') as string;
    
    const loginRequest: LoginRequest = { email, password };
    const response = await AuthRepository.postLogin(loginRequest);
    
    if (response.code === '0') {
      revalidatePath('/');
      redirect('/dashboard');
    } else {
      return { error: response.message };
    }
  } catch (error) {
    return { error: '로그인 중 오류가 발생했습니다.' };
  }
}
```

#### 데이터 조회 패턴
```typescript
// app/actions/track/get-shop-list.ts
'use server'

import { TrackRepository } from '@/src/model/TrackRepository';
import { unstable_cache } from 'next/cache';

export const getShopListAction = unstable_cache(
  async () => {
    try {
      const response = await TrackRepository.getShopList();
      return response;
    } catch (error) {
      return {
        code: '-1',
        data: null,
        message: '상점 목록을 불러오는데 실패했습니다.'
      };
    }
  },
  ['shop-list'],
  {
    revalidate: 300, // 5분 캐시
    tags: ['shop-list']
  }
);
```

#### 데이터 수정 패턴
```typescript
// app/actions/track/add-shop.ts
'use server'

import { TrackRepository } from '@/src/model/TrackRepository';
import { revalidateTag } from 'next/cache';

export async function addShopAction(shopData: any) {
  try {
    const response = await TrackRepository.addShop(shopData);
    
    if (response.code === '0') {
      revalidateTag('shop-list');
      return { success: true, data: response.data };
    } else {
      return { success: false, error: response.message };
    }
  } catch (error) {
    return { success: false, error: '상점 추가에 실패했습니다.' };
  }
}
```

## 4. 기존 Repository와의 통합

### 4.1 Repository 레이어 유지
- 기존 `src/model/` 디렉토리의 Repository 클래스들은 그대로 유지
- 외부 API 호출 로직은 변경하지 않음
- Server Actions에서 Repository를 호출하는 래퍼 역할

### 4.2 점진적 마이그레이션 전략
1. **1단계**: 새로운 기능에만 Server Actions 적용
2. **2단계**: 기존 API 호출을 Server Actions로 점진적 교체
3. **3단계**: 클라이언트 컴포넌트에서 Server Actions 사용으로 전환

## 5. 장점

### 5.1 성능 향상
- 서버 사이드 캐싱 (`unstable_cache`)
- 자동 리밸리데이션 (`revalidatePath`, `revalidateTag`)
- 네트워크 요청 최적화

### 5.2 보안 강화
- 서버 사이드에서 API 키 관리
- 클라이언트에 민감한 정보 노출 방지
- CSRF 보호 자동 적용

### 5.3 개발자 경험
- 타입 안전성 향상
- 자동 에러 처리
- Progressive Enhancement 지원

## 6. 구현 우선순위

### 6.1 높은 우선순위
- 인증 관련 (login, logout, check-auth)
- 데이터 조회 (shop-list, profile)
- 실시간 검색

### 6.2 중간 우선순위
- 데이터 수정 (add-shop, update-profile)
- 구독 관리
- 사용자 관리

### 6.3 낮은 우선순위
- 복잡한 데이터 처리
- 배치 작업
- 리포트 생성
