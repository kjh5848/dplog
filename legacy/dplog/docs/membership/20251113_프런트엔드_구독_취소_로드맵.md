# Subscription Cancel Frontend Roadmap (Pending Backend Update)

## 1. 현재 상황
- 해지 CTA 및 모달은 현재 UI에서 제거된 상태이며, 다운그레이드 예약/취소와 구독 상태 표시만 유지하고 있습니다.
- 백엔드 정책 개편(해지 예약 → 다음 결제일까지 이용 가능 → 스케줄러가 최종 해지)을 반영해 프런트도 상태 기반 UX로 전환해야 합니다.

## 2. 목표
- 사용자가 해지를 요청해도 결제된 기간 종료일까지 서비스 접근이 유지됨을 명확히 안내.
- `PENDING_CANCEL` 상태와 예정 종료일(`cancelScheduledAt`/`nextBillingDate`)을 UI에서 표시.
- 다운그레이드 예약과 해지 예약이 동시에 존재할 때 우선순위를 명시하고, 버튼 비활성화 정책을 정립.

## 3. 필수 연동 사양 (예상)
- 해지 API (예시) `POST /api/subscriptions/{id}/cancel`
  - 응답: `{ status: "PENDING_CANCEL", effectiveDate, message }`
- 상태 조회 `GET /api/payments/subscriptions/status`
  - `currentSubscription`
  - `pendingDowngrade`
  - `pendingCancel` (effectiveDate, requestedAt 등)
