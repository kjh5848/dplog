# 프런트엔드 멤버십 업그레이드 조치 (2025-11-17)

백엔드에서 `/v1/subscription/upgrade` 정책과 `UpgradeCalculator`가 갱신되어 다음 시나리오가 보장된다.
- 플랜 업그레이드(FREE/실속 → 상위 플랜)
- 기간 업그레이드(동일 플랜 월간 → 연간)
- **신규: 연간 플랜 사용자가 상위 월간 플랜으로 즉시 전환 (연간 → 월간)**

이에 맞춰 프런트엔드는 업그레이드 버튼, 확인 모달, 결제 호출, 사용량 표기, 안내 텍스트를 동시에 수정해야 한다.

## 1. 영향을 받는 화면/모듈
- `src/app/(home)/membership/page.tsx`
  - 요금제 카드 CTA, 기간 토글, 무료 플랜 제한 배지.
- `src/app/(home)/payment/PaymentClientPage.tsx`
  - `getSubscriptionAction`, 업그레이드 확인 모달, `/api/payments/upgrade` 호출, 성공 후 상태 갱신.
- `src/model/PaymentRepository.ts`
  - `upgradeSubscription` 요청/응답 타입 업데이트.
- `src/lib/services/paymentService.ts` & `/api/payments/upgrade` Route Handler
  - 백엔드 응답 신규 필드 전달, 오류 메시지 표준화.
- `src/store/membership/useMembershipUIStore.ts` 및 `useMembershipSummary` 관련 쿼리
  - 업그레이드 직후 한도, 사용량, 상태 뱃지 리셋.

## 2. 플랜 업그레이드 (등급 상승)
1. **사용량 초기화 안내**
   - 업그레이드 확인 모달과 결과 토스트에 “업그레이드 완료 시 사용량이 즉시 0으로 초기화됩니다” 문구 추가.
   - FREE 제한 배지/모달(`useMembershipUIStore.freeLimitModal`)은 업그레이드 성공 시 바로 닫히고, 추적 화면 제한 배지도 숨긴다.
2. **데이터 다시 불러오기**
   - `await queryClient.invalidateQueries(['membershipProfileSummary'])` 등으로 `/api/membership/profile-summary`, `/api/membership/current`, `/api/membership/list`를 모두 새로 고침.
   - `MembershipUsageService.buildUsage` 결과가 즉시 반영되도록 사용자 정보 패널을 다시 렌더하도록 한다.
3. **가드 해제**
   - 상위 플랜 권한이 필요한 컴포넌트(`src/features/track`, `src/features/report`)는 `membershipLevel` 전역 상태를 즉시 읽어 재연산하도록 `useMembershipUIStore.setCurrentLevel` 호출과 동시에 이벤트를 발생시킨다.
4. **로깅/Telemetry**
   - 업그레이드 성공 시 기존/신규 플랜, upgradeType(`PLAN_UPGRADE`)을 analytics로 남겨 CS/데이터팀과 동일한 지표를 사용.

## 3. 기간 업그레이드 (월간 → 연간)
1. **조건 분기 보강**
   - `getSubscriptionAction`에서 동일 플랜 + `selectedPeriod === 'YEARLY'` 인 경우만 기간 업그레이드로 간주하고 usage reset 문구를 숨긴다.
2. **기능 변화 없음 안내**
   - 확인 모달에 “사용량은 기존 월간 리셋 규칙(매월 1일)을 그대로 따릅니다” 메시지를 추가.
3. **다음 결제일 노출**
   - `/api/payments/upgrade` 응답의 `nextBillingDate`(월간 종료 다음 날 + 1년)를 업그레이드 완료 토스트 및 구독 카드에 표시.
4. **연간 혜택 표기**
   - 연간 전용 혜택이 존재하면 `selectedPeriod === 'YEARLY'` 상태에서 즉시 뱃지/툴팁을 노출하여 고객 문의를 줄인다.

## 4. 연간 → 월간 상위 플랜 전환 (신규)
1. **액션 감지**
   - 현재 구독 주기가 `YEARLY` 이고, 사용자가 **상위 등급 월간 플랜**을 선택했을 때 `getSubscriptionAction`이 `upgrade` 대신 `yearly_to_monthly_upgrade` 타입을 반환하도록 확장.
   - 비교 기준: 요금제 level 혹은 월간 가격(`selectedPlan.price > currentPlanPrice`).
2. **확인 모달 & 동의**
   - `DifferenceAmount` 외에 신규 응답 필드 `refundableAmount`, `newChargeAmount`, `message`를 노출.
   - 표준 문구: “잔여 기간 약 XX원이 환불되고, [신규 플랜] 월간 XX원이 즉시 결제됩니다. 환불/결제에 동의하십니까?”
   - 두 개의 체크박스(환불 동의, 즉시 결제 동의)를 모두 체크해야 `확인` 버튼 활성화.
3. **API 호출**
   - `PaymentRepository.upgradeSubscription` 호출 시 `billingCycle: 'MONTHLY'` 로 고정, 결제 수단은 서버가 기존 빌링키를 재사용하므로 `paymentMethod` 생략 가능.
   - 응답 타입 확장: `interface SubscriptionUpgradeResponse`에 `refundableAmount?: number`, `refundableCurrency?: Currency`, `refundPaymentId?: string | null`, `upgradeType?: 'PLAN' | 'PERIOD' | 'YEARLY_TO_MONTHLY'`, `message?: string` 추가.
4. **사용량 초기화 처리**
   - 연→월 상향은 “플랜 업그레이드”로 간주되므로 2장과 동일한 Usage Reset UI 를 태운다.
   - 성공 시 `UsageLimit` 관련 토스트/배지는 즉시 삭제.
5. **결과 UI**
   - 성공 토스트: “환불 XX원 + 월간 결제 XX원 처리가 완료되었습니다. 성장 월간 플랜이 활성화되었습니다.”
   - 구독 카드에는 새로운 월간 `nextBillingDate = 결제일 + 1개월` 과 상태를 즉시 반영.
6. **오류 처리**
   - `SubscriptionServiceApiV1`가 "연→월 전환 불가" 에러를 던질 경우 메시지를 그대로 노출하고 CTA 를 disabled 상태(tooltip 포함)로 전환.
   - 환불 단계 실패(`refundFailed: true`)가 응답에 추가되면 기존 연간 플랜 유지 메시지를 보여주고 재시도 유도.

## 5. API & 타입 업데이트
- `/api/payments/upgrade` Route Handler
  - 백엔드 응답 JSON 을 그대로 전달하고, 연→월 케이스에서 `refundableAmount` 등의 필드를 포함.
- `src/types/payment.ts`
  - `SubscriptionUpgradeRequest` 는 그대로 두되, `SubscriptionUpgradeResponse` 에 항목 추가.
- `src/model/PaymentRepository.ts`
  - 응답 파싱 시 신규 필드를 로깅하고, 실패 시 사용자 친화적인 메시지(`getUserFriendlyErrorMessage`)를 노출하도록 확장.
- `PaymentClientPage.tsx`
  - `upgradePaymentLabel` 계산 시 월간/연간/연→월 케이스에 따라 `differenceAmount`, `newChargeAmount` 등을 선택.

## 6. 후속 체크리스트
1. 업그레이드 성공 직후 `router.refresh()` 또는 필요한 쿼리 invalidate 로 Plan 카드/사용량을 재계산한다.
2. `/api/membership/profile-summary` 응답이 비어 있을 경우 Skeleton 이 아닌 “초기화 중” 문구를 잠깐 표기해 고객 혼란 방지.
3. QA 시나리오
   - FREE → 실속 → 성장 업그레이드에서 사용량 초기화가 즉시 반영되는지 확인.
   - 월간 실속 → 연간 실속 전환 시 사용량이 유지되는지 확인.
   - 연간 실속(2개월 사용) → 월간 성장 업그레이드: 환불/결제 금액 안내, 사용량 초기화, 다음 결제일(+1개월) 갱신 여부 확인.
   - 연→월 전환 실패(mock) 시 기존 연간 플랜이 그대로 유지되고 Toaster 가 올바른 지 확인.
4. CS 가이드를 위한 문구 협업: “연간 → 월간 상위 플랜” FAQ 를 새로 추가하고 프런트 툴팁 문구를 동일하게 유지.
