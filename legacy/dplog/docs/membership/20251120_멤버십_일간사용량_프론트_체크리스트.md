# 일간 사용량 전환 프런트 대응 (2025-11-20)

> 출처: 2025-11-20 백엔드 “월간→일간 사용량 전환” 체크리스트. 프런트 관점에서 필요한 대응 범위·TODO를 정리한다.

## 1. 백엔드 변경 요약
- `ServiceSort`가 월간(`NPLACE_SHOP_REGISTER`, `NPLACE_RANK_TRACK`)과 일간(`NPLACE_RANK_REALTIME`, `NPLACE_KEYWORD`, `NPLACE_KEYWORD_RELATION`, `BLOG_QUALITY_CHECK`, `NPLACE_REVIEW_REPLY`)으로 구분된다.
- 일간 항목은 `MembershipUsage`/`UseLog`가 `yyyy-MM-dd` 키 기준으로 집계하며, DTO(`MembershipUsageService.buildUsage`)에 `periodType`(DAILY/MONTHLY)과 해당 일자 한도 값이 포함된다.
- `UsageResetService`는 월간 항목만 초기화하고, 일간 항목은 자정(로컬) 자동 리셋된다 → UI/카피에 “매일 0시 기준 초기화” 문구 필요.
- 실시간·키워드·블로그·리뷰 기능은 백엔드 단계에서 일간 한도를 검증하므로, 프런트 CTA/Disabled 문구도 일간 기준과 동일해야 한다.

## 2. 프런트 영향 범위
- `src/app/(home)/membership/page.tsx`: 플랜 카드의 기능 한도를 `/월` → `/일` 기준으로 교체하고, “≈월간” 환산치는 툴팁/FAQ 용도로만 표기.
- `src/components/profile/ProfileView.tsx` (및 `docs/membership/20251105_멤버십_프로필_요약_계획.md`): 사용량 게이지에 `periodType`을 반영해 “오늘(일간)”/“이번 달(월간)”을 명시하고 일간 항목에는 자정 리셋 배지 부착.
- `src/types/membership.ts` 및 관련 API 타입: `MembershipUsageMetric`에 `periodType: 'DAILY' | 'MONTHLY'`(+ `periodKey`, `resetAt` 등) 추가, `ServiceSort` ↔ UI 레이블/limitUnit 매핑 테이블 생성.
- FAQ/툴팁/가이드: `/membership`, `/profile`, CS 문구, 마케팅 페이지 모두 “월간 유지 vs 일간 전환”을 명시한다.
- 운영/관리자 도구: 일간 사용량 조회 API 도입 시 히스토리 화면에 일간/월간 토글을 제공하도록 요구사항을 수집한다.

## 3. 요금제 카드 & UI 카피 기준

| 프런트 표기 | ServiceSort | 기준 | FREE | 실속 점주 | 성장 오너 | 프리미엄 마스터 | 비고 |

| 실시간 순위 조회 | `NPLACE_RANK_REALTIME` | 일간 | 5회/일 (≈150/월) | 20회/일 (≈600/월) | 60회/일 (≈1,800/월) | 무제한 | `/일` 표기 고정 |

| 블로그 저품질 | `BLOG_QUALITY_CHECK` | 일간 | 5회/일 | 10회/일 | 30회/일 | 무제한 |  |

- 일간 기능은 “오늘 X/Y회” 강조, 월간 기능은 “이번 달 X/Y개”로 구분되는 컴포넌트 필요.
- FAQ에는 “≈월간” 수치를 병기하되, 실제 차단 기준은 일간임을 명시.
