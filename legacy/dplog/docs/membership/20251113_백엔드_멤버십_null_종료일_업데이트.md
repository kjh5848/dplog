# Backend Membership Null End-Date Update

## 1. 주요 변경 요약
- `MEMBERSHIP_USER.end_date` 컬럼을 `NULL` 허용으로 변경하고, 엔티티(`MembershipUserEntity`)에 이를 반영했습니다.
- 구독 해지 시 `pending*` 필드를 초기화하고, 스케줄러·다운그레이드 로직에서 무제한 멤버십을 안전하게 처리하도록 `SubscriptionServiceApiV1` 관련 흐름을 보강했습니다.
- 무료 멤버십 자동 발급 로직(`KakaoOAuthService.assignDefaultMembership`)은 `membership.free-trial-days <= 0`일 때 종료일을 기록하지 않도록 수정했으며, 환경 설정 기본값을 `-1`(무제한)로 조정했습니다 (`application-dev.yml`, `application-prod.yml`).
- 멤버십 이력 조회/커서 로직이 `NULL` 종료일을 다룰 수 있도록 `MembershipUserRepository`, `MembershipServiceApiV1`를 개편하고, `ResMembershipCurrentDTOApiV1` 등 DTO/요청 객체에서 `endDate`가 `NULL`인 경우 자연스럽게 응답하도록 처리했습니다.
- 구독 엔티티, 사용자 멤버십 저장 DTO, 다운그레이드·취소 테스트를 함께 업데이트하면서 무제한 멤버십이 정상 동작하는지 검증하는 단위 테스트(`ResMembershipCurrentDTOApiV1Tests`)를 추가했습니다.

## 2. 프런트엔드 영향도
- `endDate`가 `null`인 경우(무제한/무료 멤버십) “다음 결제일”, “남은 기간”, “자동결제” 섹션을 숨기도록 처리해야 합니다.
- 예약 다운그레이드/취소 UI는 `pending*` 필드가 서버에서 초기화되는 시점을 기준으로 상태를 갱신합니다. 구독 상태 API 응답에서 `pendingDowngrade`가 빠지면 예약 해제된 것으로 간주됩니다.
- FAQ/정책 문구 업데이트 시 “무료 멤버십(무제한)” 케이스를 별도 표기해야 합니다.

## 3. 테스트 및 체크리스트
- 무료 멤버십 부여 시 프로필/구독 카드에서 종료일 정보가 나타나지 않는지 확인합니다.
- 다운그레이드 예약 → 스케줄러 적용 → 새 요금제 반영 플로우에서 `endDate`가 `null`이어도 정상 처리되는지 검증합니다.
- `MembershipUserLog` 및 이력 API에서 `NULL` 종료일을 가진 레코드가 문제 없이 조회되는지 확인합니다.
