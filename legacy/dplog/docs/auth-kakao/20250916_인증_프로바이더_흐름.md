# 로그인/프로필 플로우 정리

본 문서는 LOCAL(일반 로그인)과 KAKAO(카카오 로그인) 기반의 로그인 처리, 사용자 저장, 프로필 표시/편집 로직을 1부터 순서대로 정리한 자료입니다. 또한 프로필에 네이버 아이디/비밀번호 필드 추가 사항을 포함합니다.

## 1. 일반 로그인 플로우
1) 아이디/비밀번호 입력 → 유효성 검사 통과
2) 로그인 요청: `AuthRepository.postLogin` 호출
3) 성공 시 사용자 저장: `setLoginUser({ ...result.user, provider: 'LOCAL' })`
   - 구현 파일: `src/components/common/auth/LoginForm.tsx`
4) 프로필 초기화: `loginUser` 기준으로 프로필 스토어 세팅(표시명/아바타/권한/프로바이더)
   - 구현 파일: `src/components/profile/ProfileView.tsx`

## 2. 카카오 로그인 플로우
1) 카카오 인증 시작 → 사용자가 동의 후 콜백 URL 진입
2) 콜백 파라미터 검증: `code/state` 확인 + CSRF `state` 일치 검증
3) 백엔드 콜백 POST: `/v1/auth/kakao/callback` 호출(세션 쿠키 포함)
4) 성공 여부 확인: HTTP OK + `result.code` 체크
5) 세션 재검증: `AuthRepository.checkAuth()` 호출
6) 사용자 저장 분기
   - 6-1) 표준 유저 제공 시 → `mapLocalUserToLoginUser(info.data.user)` → `setLoginUser(...)`
   - 6-2) 비어있거나 카카오 원본만 제공 시 → `mapKakaoDataToLoginUser(result.data)` → `setLoginUser(...)`
   - 구현 파일: `src/app/(home)/(auth)/kakao/callback/KakaoCallbackClient.tsx`
7) 완료 라우팅: `router.push('/track')`

## 3. 프로필 표시/편집 로직
1) 초기 세팅: `loginUser`로 프로필 스토어 채움(표시명, 아바타, 권한, `provider`, `providerId`)
   - 구현 파일: `src/components/profile/ProfileView.tsx`
2) 로그인 방식 배지(상단 카드)
   - `provider === 'KAKAO'` → “카카오 로그인” 배지
   - 그 외 → “일반 로그인” 배지
3) 표시 우선순위
   - 이름: `nickname → name → username`
   - 아바타: `profileImage → profileImageUrl → thumbnailImageUrl`
4) 이메일 편집 제한
   - `provider === 'KAKAO'`이면 이메일 입력 비활성 + “카카오에서 관리되는 정보” 안내 문구
5) 네이버 계정 필드(클라이언트 저장)
   - 입력: “네이버 아이디”(`naverId`), “네이버 비밀번호”(`naverPassword`)
   - 보기: 비밀번호는 “저장됨/미입력”만 표시(값 미노출)
6) 저장 동작: 로컬 스토어 `updateProfile`로 반영(백엔드 저장/암호화는 별도)

## 4. 검증 체크리스트
1) 일반 로그인 후 배지 “일반 로그인” 표시, 기본 편집 가능
2) 카카오 로그인 후 배지 “카카오 로그인” 표시, 닉네임/아바타 반영
3) 카카오 로그인 시 이메일 입력 비활성 + 안내 문구 노출
4) 프로필에 네이버 아이디/비밀번호 입력 가능, 비밀번호 미노출
5) 로그아웃/세션 만료 UX는 기존 로직과 충돌 없음

## 5. 적용 파일 요약
- 유틸/매핑: `src/utils/auth/mapUser.ts`
  - `mapLocalUserToLoginUser`, `mapKakaoDataToLoginUser`(별칭: `mapKakaoResponseToLoginUser`)
- 일반 로그인: `src/components/common/auth/LoginForm.tsx`
  - 성공 시 `setLoginUser({ ...result.user, provider: 'LOCAL' })`
- 카카오 콜백: `src/app/(home)/(auth)/kakao/callback/KakaoCallbackClient.tsx`
  - `checkAuth()` 결과에 따라 표준 유저/카카오 원본 매핑 분기 저장
- 프로필: `src/components/profile/ProfileView.tsx`
  - 로그인 방식 배지, 이메일 편집 제한, 네이버 아이디/비밀번호 필드
- 프로필 스토어 타입: `src/viewModel/profile/nplaceProfileViewModel.ts`
  - `provider`, `providerId`, `naverId`, `naverPassword` 필드 추가

## 6. 보안/후속 과제
- 현재 네이버 비밀번호는 클라이언트 상태 내에서만 관리합니다. 서버 저장이 필요하면 암호화(KMS/해시+솔트)와 접근 정책을 포함한 백엔드 API를 별도로 설계/구현해야 합니다.
