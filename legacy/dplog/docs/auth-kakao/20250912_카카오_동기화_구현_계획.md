**개요**
- 목표: 기존 카카오 로그인 플로우에 카카오 싱크(간편가입+약관 동의) 도입
- 범위: 카카오 개발자 콘솔 설정, 프론트 버튼/SDK 연동, 콜백/세션 처리, UI/로그/테스트

**사전 준비**
- 카카오 앱 정보 확인: `앱 키(JS/REST)`, `앱 아이콘/이름`, `개발/운영` 구분
- 서비스 도메인/리다이렉트 URI 정리: `http(s)://localhost:3000`, 배포 도메인 등
- 백엔드 CORS, 쿠키 정책: `SameSite=None; Secure`, `Access-Control-Allow-Credentials: true`

**1단계. 콘솔 설정(카카오 싱크)**
- 카카오 개발자 콘솔 → 해당 앱 → 카카오 로그인
  - 카카오 싱크 활성화(간편가입 ON)
  - 동의 항목 설정(필수/선택): 닉네임, 프로필 이미지, 이메일, 연락처, 성별/생년월일 등 서비스 정책에 맞게
  - 서비스 약관 등록: 서비스 이용약관, 개인정보 처리방침, 선택 약관들(마케팅 수신 등)
  - 리다이렉트 URI 등록: `.env.local`의 `NEXT_PUBLIC_KAKAO_REDIRECT_URI`와 동일하게 등록
  - 플랫폼(Web) 도메인 등록: `http(s)://localhost:3000`, 배포 도메인

**2단계. 환경 변수 점검**
- 파일: `.env.local`
  - `NEXT_PUBLIC_KAKAO_JS_API_KEY=<카카오 JavaScript 키>`
  - `NEXT_PUBLIC_KAKAO_REST_API_KEY=<카카오 REST API 키>`
  - `NEXT_PUBLIC_KAKAO_REDIRECT_URI=http(s)://<도메인>/kakao/callback`
  - `NEXT_PUBLIC_API_URL=https://localhost:8443`(백엔드) 유지

**3단계. 프론트 버튼 교체(카카오 싱크 전용)**
- 목적: 기존 “카카오 로그인” 버튼 → 카카오 싱크의 “동의하고 가입하기” 버튼
- 구현 옵션
  - 표준 버튼(API): `Kakao.Auth.createLoginButton({ container, scope, state, redirectUri })`
  - 커스텀 버튼: 기존 버튼 onClick에서 `Kakao.Auth.authorize({ scope, state, redirectUri })` 호출
- 권장 스코프 예시(서비스 정책에 맞춰 조정)
  - `scope: 'profile_nickname profile_image account_email phone_number'`
  - 선택 항목은 동의 화면에서 별도 체크됨
- 상태값(state) 처리
  - 기존처럼 `state`를 난수로 생성해 `localStorage`에 저장하고 콜백에서 검증 유지

**4단계. 코드 변경 지시서**
- 신규 컴포넌트 추가: `src/components/common/auth/KakaoSyncButton.tsx`
  - 마운트 시 `window.Kakao` 초기화(`initKakaoSdk`) 보장
  - `createLoginButton`로 표준 버튼 생성 또는 커스텀 버튼에서 `Kakao.Auth.authorize` 호출
  - 컨테이너 예: `<div id="kakao-sync-btn" />`
- 로그인 페이지 교체: `src/components/common/auth/LoginForm.tsx`
  - 기존 카카오 버튼 영역을 `KakaoSyncButton`으로 교체
- SDK 유틸 재사용: `src/utils/api/kakaoSdk.ts`
  - `initKakaoSdk()`는 유지
  - `getKakaoAuthUrl()`은 레거시 대비용으로 보관(사용처 제거 가능)
- 콜백 페이지 유지: `src/app/(home)/(auth)/kakao/callback/*`
  - 이미 `credentials: 'include'` + `/v1/auth/info` 순차 호출 적용됨
  - 로그인 완료 시 대시보드(`/track`)로 이동 유지

**5단계. 콜백/세션 처리(백엔드 상호작용)**
- 콜백 호출: `POST ${NEXT_PUBLIC_API_URL}/v1/auth/kakao/callback` with `{ code, state }`, `credentials: 'include'`
- 성공 시: 즉시 `/v1/auth/info` 재호출 → 동일 세션 검증 → 클라이언트 스토어에 `loginUser` 저장
- 필드 매핑(프론트 표시)
  - 표시명: `nickname || username`
  - 아바타: `profile_image`(http → https 치환 고려)
  - 이메일: 동의한 경우에만 사용
- 에러 처리
  - 동의 거부, scope 미부여, state 불일치 시 에러 메시지 후 `/login`으로 복귀

**6단계. UI/로딩/접근성**
- 로딩: 모든 로그인 처리 문구는 통일 → “로그인 중입니다.”
- 버튼 라벨: “카카오로 동의하고 시작하기” 또는 “동의하고 가입하기”
- 접근성: 버튼에 `aria-label` 추가, 스피너는 `prefers-reduced-motion` 고려

**7단계. 로깅/모니터링**
- 클라이언트: `logInfo/logError/logUserAction`으로 버튼 클릭, authorize 시작/완료, 콜백 결과 기록
- Sentry: 오류(토큰 교환 실패, state mismatch, 네트워크 실패) 캡처

**8단계. 테스트 체크리스트**
- Happy Path
  - 버튼 클릭 → 동의 화면 → 콜백 성공 → `/track` 이동 → 상단 사용자 프로필 닉네임/아바타 노출
- 에러 케이스
  - 동의 거부 → 로그인 페이지 복귀, 안내 토스트
  - state 불일치 → 에러 처리 및 재시도 유도
  - 이메일/연락처 동의 미포함 → UI에서 null 허용 경로 확인
- 크로스 도메인/HTTPS
  - `https://localhost:3000`/`https://localhost:8443` 혼용 시 쿠키 포함 및 세션 유지 확인

**예상 코드 스케치**
- `src/components/common/auth/KakaoSyncButton.tsx`
```
"use client";
import { useEffect } from 'react';
import { initKakaoSdk, getKakaoAuthUrl } from '@/src/utils/api/kakaoSdk';
import { logInfo } from '@/src/utils/logger';

export default function KakaoSyncButton() {
  useEffect(() => {
    initKakaoSdk();
    const state = crypto.getRandomValues(new Uint32Array(4)).join('');
    localStorage.setItem('kakao_oauth_state', state);

    if (window.Kakao?.Auth?.createLoginButton) {
      window.Kakao.Auth.createLoginButton({
        container: '#kakao-sync-btn',
        scope: 'profile_nickname profile_image account_email phone_number',
        state,
        redirectUri: process.env.NEXT_PUBLIC_KAKAO_REDIRECT_URI,
        success: () => logInfo('Kakao Sync authorize started'),
        fail: (err: any) => console.error('Kakao Sync error', err),
      });
    }
  }, []);

  return <div id="kakao-sync-btn" />;
}
```
