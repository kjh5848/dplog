# 카카오 로그인 백엔드 API 구현 요청서

## 📋 개요

**프로젝트**: DP Log 네이버 플레이스 순위 서비스  
**요청 일자**: 2025-01-09  
**우선순위**: 높음  
**예상 소요 시간**: 1-2일  

카카오 OAuth 2.0을 활용한 소셜 로그인 기능의 백엔드 API 구현이 필요합니다.  
프론트엔드 구현은 완료된 상태이며, 백엔드 API만 구현하면 즉시 연동 가능합니다.

## 🔧 현재 프론트엔드 구현 상태

### ✅ 완료된 부분
- 카카오 SDK 초기화 및 OAuth URL 생성
- 카카오 로그인 버튼 UI 및 이벤트 핸들러
- OAuth 콜백 페이지 (`/kakao/callback`)
- CSRF 방지 (state 파라미터) 검증
- 로딩/에러 UI 및 사용자 피드백

### 📤 프론트엔드 → 백엔드 요청 형태
```typescript
// 프론트엔드에서 다음과 같이 API 호출
POST /api/auth/kakao/callback
Content-Type: application/json

{
  "code": "카카오에서 받은 인증 코드",
  "state": "CSRF 검증용 랜덤 문자열"
}
```

## 🎯 백엔드 구현 요구사항

### 1. API 엔드포인트

#### `POST /api/auth/kakao/callback`

**목적**: 카카오 OAuth 인증 코드를 받아 사용자 로그인 처리

**Request Body**:
```json
{
  "code": "string",    // 카카오 인증 코드 (필수)
  "state": "string"    // CSRF 검증용 state (필수)
}
```

**Success Response (200)**:
```json
{
  "success": true,
  "user": {
    "id": "string",           // 내부 사용자 ID
    "username": "string",     // 로그인용 username (예: kakao_1234567890)
    "nickname": "string",     // 표시명
    "email": "string",        // 이메일 (선택적)
    "profileImage": "string", // 프로필 이미지 URL
    "provider": "KAKAO",      // 고정값
    "providerId": "string",   // 카카오 사용자 ID
    "createdAt": "datetime",  // 계정 생성일
    "lastLoginAt": "datetime" // 마지막 로그인일
  },
  "message": "로그인 성공"
}
```

**Error Response (400/401/500)**:
```json
{
  "success": false,
  "error": "ERROR_CODE",
  "message": "사용자 친화적 에러 메시지"
}
```

### 2. 카카오 API 연동 플로우

#### 2-1. 액세스 토큰 발급
```http
POST https://kauth.kakao.com/oauth/token
Content-Type: application/x-www-form-urlencoded

grant_type=authorization_code
&client_id=${카카오_REST_API_키}
&client_secret=${카카오_CLIENT_SECRET}
&redirect_uri=${리다이렉트_URI}
&code=${인증_코드}
```

**응답 예시**:
```json
{
  "token_type": "bearer",
  "access_token": "액세스_토큰",
  "expires_in": 21599,
  "refresh_token": "리프레시_토큰",
  "refresh_token_expires_in": 5183999,
  "scope": "account_email profile_nickname"
}
```

#### 2-2. 사용자 정보 조회
```http
GET https://kapi.kakao.com/v2/user/me
Authorization: Bearer ${액세스_토큰}
Content-Type: application/x-www-form-urlencoded
```

**응답 예시**:
```json
{
  "id": 1234567890,
  "connected_at": "2023-01-01T00:00:00Z",
  "properties": {
    "nickname": "홍길동",
    "profile_image": "https://k.kakaocdn.net/dn/.../img.jpg",
    "thumbnail_image": "https://k.kakaocdn.net/dn/.../img.jpg"
  },
  "kakao_account": {
    "profile_needs_agreement": false,
    "profile": {
      "nickname": "홍길동",
      "thumbnail_image_url": "https://k.kakaocdn.net/dn/.../img.jpg",
      "profile_image_url": "https://k.kakaocdn.net/dn/.../img.jpg"

  }

```

### 3. 비즈니스 로직 요구사항

#### 3-1. 사용자 생성/조회 로직
```pseudocode
1. 카카오 사용자 ID로 기존 사용자 조회
2. 기존 사용자가 있는 경우:
   - 프로필 정보 업데이트 (닉네임, 프로필 이미지)
   - 마지막 로그인 시간 업데이트
3. 신규 사용자인 경우:
   - 새 사용자 계정 생성
   - username: "kakao_" + 카카오_사용자_ID
   - 카카오 정보 저장 (providerId, email, nickname, profileImage)
4. 세션 생성 및 쿠키 설정 (JSESSIONID)
5. 사용자 정보 반환
```

#### 3-2. 에러 처리
- `INVALID_CODE`: 유효하지 않은 인증 코드
- `INVALID_STATE`: CSRF 검증 실패
- `KAKAO_TOKEN_ERROR`: 카카오 토큰 발급 실패
- `KAKAO_USER_ERROR`: 카카오 사용자 정보 조회 실패
- `USER_CREATE_ERROR`: 사용자 생성 실패
- `SESSION_CREATE_ERROR`: 세션 생성 실패

### 4. 환경 변수 설정

```properties
# application.properties 또는 .env
kakao.rest-api-key=your_kakao_rest_api_key
kakao.client-secret=your_kakao_client_secret  
kakao.redirect-uri=https://yourdomain.com/kakao/callback
```

## 🔒 보안 고려사항

### 필수 보안 조치
1. **HTTPS 통신**: 모든 카카오 API 통신은 HTTPS 사용
2. **State 검증**: CSRF 공격 방지를 위한 state 파라미터 검증
3. **토큰 보안**: 액세스 토큰은 서버 메모리에서만 사용, DB 저장 금지
4. **입력 검증**: 모든 입력 데이터 검증 및 sanitization
5. **에러 로그**: 보안 관련 에러는 상세 로그 기록
6. **Rate Limiting**: 무차별 로그인 시도 방지

### 선택적 보안 강화
- 카카오 사용자 정보 암호화 저장
- 로그인 시도 횟수 제한
- 의심스러운 IP 차단
- 세션 타임아웃 설정

## 🧪 테스트 케이스

### 성공 시나리오
1. **신규 사용자 가입**: 카카오 첫 로그인 → 계정 생성 → 세션 생성
2. **기존 사용자 로그인**: 카카오 재로그인 → 정보 업데이트 → 세션 생성
3. **프로필 업데이트**: 카카오에서 닉네임/이미지 변경 후 로그인

### 실패 시나리오
1. **잘못된 인증 코드**: 400 에러 반환
2. **State 불일치**: 401 에러 반환
3. **카카오 API 장애**: 502/503 에러 반환
4. **DB 연결 실패**: 500 에러 반환

## 🚀 구현 후 테스트 방법

1. **프론트엔드 환경변수 설정**:
```env
NEXT_PUBLIC_KAKAO_JS_API_KEY=your_javascript_key
NEXT_PUBLIC_KAKAO_REST_API_KEY=your_rest_api_key  
NEXT_PUBLIC_KAKAO_REDIRECT_URI=http://localhost:3000/kakao/callback
```

2. **카카오 개발자 콘솔 설정**:
   - 플랫폼 등록: Web 플랫폼 추가
   - 리다이렉트 URI: `http://localhost:3000/kakao/callback`
   - 동의항목: 닉네임, 프로필 이미지, 카카오계정(이메일)

3. **통합 테스트**:
   - 로그인 페이지에서 카카오 버튼 클릭
   - 카카오 로그인 완료 후 메인 페이지로 리다이렉트 확인
   - 사용자 정보 표시 확인
