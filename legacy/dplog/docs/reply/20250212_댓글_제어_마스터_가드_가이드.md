# 엔플레이스 댓글 제어 · 마스터 멤버십 가드 가이드

## 목적
- `/reply` 페이지를 **로그인 + 마스터 멤버십** 사용자만 접근 가능하게 제한한다.
- 댓글 제어는 “샾(place)을 직접 등록 → 활성화 요청 전송” 흐름을 따른다. **조회 응답으로 플레이스 ID가 새로 내려오지 않으므로, 사용자가 placeId를 입력해 POST로 등록/활성화해야 한다.**

## 1. 접근 가드 설계 (기존 유지)
- 신규 프록시: `/api/membership/current` → 백엔드 `/v1/membership/current`
  - 위치: `src/app/api/membership/current/route.ts`
  - 구현: `proxyBackendJson` 사용, `GET`, `label: "membership/current"`, 응답은 `{ success, code, error, data }` 포맷 유지.
- React Query 키: `["membership", "current"]`
- 관심 필드: `membershipId`, `membershipName`, `membershipState`
- 마스터 판정: `membershipId === 13` **또는** `/마스터|master/i.test(membershipName)`

### Reply 페이지 가드 흐름 (`src/components/nplrace/reply/ReplyClientPage.tsx`)
1. `useAuthGuard`로 로그인 여부 확인, 게스트는 `/login` 리다이렉트.
2. 로그인 상태에서 `["membership","current"]` 쿼리 실행.
3. 로딩: 기존 로딩 UI 재사용.
4. 에러/401: 토스트 후 `/login` 또는 일반 오류 안내.
5. **마스터 아님** → 본문 대신 가드 블록 렌더
   - 메시지: “마스터 요금제 전용 기능입니다.”
   - CTA: “마스터로 업그레이드” → `/membership?planId=13&period=YEARLY`
   - 보조: 현재 플랜 이름/상태 표기.
6. **마스터** → 기존 `ReplyContent` 렌더.

### 비마스터 가드 UI 예시
- 붉은 톤 배경 박스 + 경고 아이콘
- 본문: “리뷰 댓글 제어는 마스터 요금제 전용입니다. 업그레이드하면 바로 이용할 수 있어요.”
- 버튼: “마스터로 업그레이드” (CTA 링크 위와 동일)
- 서브텍스트: 현재 플랜 이름/상태 노출

## 2. 댓글 제어 호출 순서 & API 스펙 (프런트 플로우 반영)
> 핵심: **새 placeId는 GET으로 자동 조회되지 않는다.** 사용자가 placeId를 입력해 `POST /v1/reply`로 등록/활성화해야 이후 GET에서 보인다.

1. **세션 로그인** – 기존 Auth API로 `authInfo` 세션 확보.
2. **네이버 계정 상태 조회** `GET /v1/user/naver/status`
   - `exists=false` → 계정 등록 필요.
3. **네이버 계정 등록/수정**
   - 미등록: `POST /v1/user/naver`
   - 수정: `PUT /v1/user/naver`
4. **현재 댓글 상태 조회** `GET /v1/reply`
   - 반환 리스트는 “이미 POST/DELETE로 저장된 placeId/active”만 포함.
   - Pumpingstore 자동 조회 없음.
5. **댓글 ON/OFF 및 등록** `POST /v1/reply`
   - Body: `{ nplaceReplyInfo: { placeId: "사용자가 입력", active: true|false } }`
   - 내부에서 네이버 계정 확인 → Pumpingstore `post/put seller-nvid` → DB 저장/로그 기록.
6. **댓글 제어 정보 삭제(선택)** `DELETE /v1/reply`
   - Pumpingstore `delete seller-nvid` + 로컬 Reply/Log 삭제.

### 응답 예시 요약
- 공통 성공: `{ code: 0, message: "success", ... }`
- `/v1/user/naver/status`: `{ data: { exists: boolean } }`
- `/v1/reply`: `{ data: { nplaceReplyList: [{ id, placeId, active }] } }`

### 예외/에러 패턴
- 세션 만료: `code != 0`, 메시지 예) “재인증이 필요한 사용자입니다. 로그인 후 다시 시도해주세요.”
- 네이버 계정 없음: 메시지 “네이버 인증 정보가 없습니다.” → 계정 등록 화면으로 유도.
- Pumpingstore 오류: `NomadscrapException` 메시지 그대로 전달 → 사용자 안내/재시도 옵션 제공.

### 프런트 구현 메모
- 입력된 placeId를 그대로 `POST /v1/reply`에 사용하고, 성공 시 React Query 캐시를 무효화해 `GET /v1/reply` 리스트를 최신화.
- 비밀번호는 저장하지 말고 즉시 전송용으로만 사용.
- 모든 API는 `authInfo` 세션이 전제이므로, 로그인 가드 후 호출.

## 3. 테스트 체크리스트
- 로그인 없음 → `/login` 리다이렉트.
- 로그인 + 무료/실속/성장 → 가드 배너 노출, `/membership` CTA 동작.
- 로그인 + 마스터 → placeId 입력 후 `POST /v1/reply` 성공, 이어서 `GET /v1/reply`에 방금 등록된 placeId 표시.
- `/api/membership/current` 401/에러 → 토스트/가드 처리.
- 네이버 계정 미등록 상태 → 상태 조회 후 등록 요청이 정상 동작.

## 참고
- 인증 응답을 비대하게 만들지 않도록, 멤버십 조회는 App Route로 분리 유지.
- React Query `staleTime`을 적절히 설정해 중복 호출 최소화.
- 향후 사용 한도(usage limit) 토스트와 충돌하지 않도록 토스트 ID 분리 권장.
