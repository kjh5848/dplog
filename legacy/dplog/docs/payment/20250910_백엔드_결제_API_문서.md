# 백엔드 API 문서

## 🏗️ 전체 API 구조

**기본 설정**
- Base URL: `https://api.dplog.kr`
- 인증: Session 기반 (Cookie: JSESSIONID)
- Content-Type: `application/json`

### 7. 결제 히스토리 조회
```http

```

**Headers:**
```http
Cookie: JSESSIONID={session_id}
```

- `page` (number, optional): 페이지 번호 (기본값: 1)
- `limit` (number, optional): 페이지당 항목 수 (기본값: 20, 최대: 100)
- `startDate` (string, optional): 조회 시작일 (YYYY-MM-DD)
- `endDate` (string, optional): 조회 종료일 (YYYY-MM-DD)
- `status` (string, optional): 결제 상태 ("SUCCESS", "FAILED", "PENDING", "CANCELLED")

**Response:**
```json
{
  "code": "0",
  "message": "결제 히스토리를 성공적으로 조회했습니다.",
  "data": {
    "payments": [
      {
        "id": "payment_123",
        "orderId": "order_20240101_abc123",
        "amount": 29000,
        "planName": "프로 플랜",
        "paymentMethod": "CARD",
        "status": "SUCCESS",
        "paidAt": "2024-01-01T10:00:00Z",
        "receipt": {
          "url": "https://api.dplog.kr/v1/payment/receipt/payment_123"
        }
      }
    ],
    "pagination": {
      "page": 1,
      "limit": 20,
      "totalItems": 45,
      "totalPages": 3,
      "hasNext": true,
      "hasPrev": false

}
```

## 🔒 보안 고려사항

1. **CORS 설정**: 프론트엔드 도메인만 허용
2. **Rate Limiting**: API 호출 제한
3. **HTTPS 강제**: 모든 API는 HTTPS만 허용
4. **세션 관리**: 
   - HttpOnly, Secure 쿠키 사용
   - 세션 타임아웃 설정
5. **입력 검증**: 모든 입력값 검증 및 Sanitization
6. **로깅**: 모든 API 호출 및 에러 로깅
