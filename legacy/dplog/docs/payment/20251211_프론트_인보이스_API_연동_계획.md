# 인보이스/차지/구독 프론트 연동 계획 (2025-12-11)

## 1. 목표와 범위
- 인보이스 조회 전용 API로 목록/상세/요약/CSV 뷰를 구성한다(생성/수정/삭제 없음).
- 모든 호출은 Next.js App Route(`src/app/api`)를 경유하고, 서비스 로직은 `src/lib/services/invoiceService.ts`(신규)로 캡슐화한다.
- TanStack Query 기반으로 목록/상세 캐싱을 제공하되, 뷰모델 계층 없이 페이지/컴포넌트 수준 훅에서 직접 사용한다.

## 2. 인증 및 공통 규칙
- 인증: `Authorization: Bearer <token>` (ADMIN 전용, 추후 ROLE_FINANCE 등으로 확장 가능).
- 요청/응답: 기본 `application/json`, CSV 응답은 `text/plain`.
- 날짜 쿼리: `from`, `to`는 `yyyy-MM-dd` (issuedAt 기준).
- 인보이스 시간 필드: `yyyy-MM-dd HH:mm` (KST, TZ 미포함).
- Charge `paidAt`: `yyyy-MM-dd'T'HH:mm:ss` (LocalDateTime 포맷, TZ 미포함).
- 정렬: `sort` 미지정 시 `issuedAt,DESC` 기본 적용.
- 필터 enum은 영문 대문자 기준.
- `lineTypeLabel` 매핑: PRORATION=일할차액, MEMBERSHIP=멤버십, DISCOUNT=할인, CREDIT=크레딧, ADJUSTMENT=조정, REFUND=환불.

## 3. API 요약 (프론트 관점)
| 엔드포인트 | 용도 | 주요 쿼리/경로 | 응답 요약 | 비고 |
| --- | --- | --- | --- | --- |
| `GET /api/invoices` | 인보이스 목록 | `status`, `from`, `to`, `userId`, `subscriptionId`, `lineType`, `lineDescription`, `format`, `page`, `size`, `sort` | 페이징 객체 + `invoice[]` (`invoiceId`, `subscriptionId`, `userId`, `status`, `amountDue`, `currency`, `issuedAt`, `dueAt`, `paidAt`) | `format=csv` 시 `text/plain` CSV 헤더 `invoice_id,...,line_count,line_amount_sum` |
| `GET /api/invoices/{invoiceId}` | 인보이스 상세 | path param `invoiceId` | `invoice` + `lines[]` + `charges[]` | lineType/chargeType별 시각적 구분 필요 |
| `GET /api/invoices/summary` | 인보이스 요약 | `from`, `to`, `status` | 월별 배열 (`month`, `totalAmount`, `count`) | 대시보드 요약에 사용 |

### 인보이스 상세 응답 필드
- `invoice`: 목록과 동일 필드.
- `lines[]`
  - `lineId`, `invoiceId`, `lineType` (MEMBERSHIP|PRORATION|DISCOUNT|CREDIT|ADJUSTMENT|REFUND)
  - `lineTypeLabel` (한글 라벨), `description`, `amount`, `currency`
- `charges[]`
  - `paymentId`, `invoiceId`, `userId`, `chargeType`, `paymentType`, `status`, `amount`, `currency`, `paidAt`
  - `chargeType`: `INITIAL | RECURRING | RETRY | ONE_TIME | UPGRADE | PRORATION | ADJUSTMENT | REFUND` (nullable)
  - `paymentType`: `INITIAL | RECURRING | RETRY | ONE_TIME | UPGRADE | DOWNGRADE`
  - `status`: `PENDING | PAID | FAILED | CANCELLED | PARTIAL_CANCELLED | REFUNDED`

## 4. App Route/BFF 설계
- 공통
  - `Authorization: Bearer <token>` 헤더는 서버에서 세션/쿠키를 읽어 주입한다(ADMIN 역할 검증 실패 시 401/403).
  - App Route → `invoiceService` → 백엔드 호출. 백엔드 4xx/5xx는 `NextResponse`에 표준화(`{ success: false, error }`).
  - CSV 요청은 `Content-Type: text/plain`, `Content-Disposition: attachment; filename=invoices_<yyyymmdd>.csv`로 반환.
- `src/app/api/invoices/route.ts`
  - 쿼리 스키마 검증(날짜, status enum, page/size 기본값) 후 백엔드 전달.
  - CSV 요청은 스트리밍/텍스트 그대로 전달, JSON은 `ClientApiResponse` 래핑.
- `src/app/api/invoices/[invoiceId]/route.ts`
  - `invoiceId` 검증 후 상세 조회. 404 → notFound 처리, line/charge 배열 그대로 전달.
- `src/app/api/invoices/summary/route.ts`
  - `from`, `to`, `status` 검증 후 요약 배열 전달.

## 5. UI/UX 및 기능 요구
- 목록 화면
  - 필터: 상태(enum), 기간(from/to), 사용자 ID, 구독 ID, 라인 타입, 라인 설명 검색, 정렬(issuedAt,DESC 기본). 입력 변경 시 쿼리스트링 동기화.
  - 테이블 컬럼: 인보이스ID, 구독ID, 사용자ID, 상태(배지), 금액/통화, 발행일, 만기일, 납부일. 상태별 색상/아이콘 매핑.
  - 페이지네이션: Spring Pageable(`page`, `size`, `totalPages`), `size` 선택 지원.
  - CSV 다운로드 버튼: 현재 필터 그대로 `format=csv` 호출 후 파일 저장.
- 상세 화면
  - 헤더: 인보이스 메타(상태, 금액, 통화, issued/due/paidAt)와 구독/사용자 링크.
  - 라인 섹션: `lineTypeLabel`과 `description` 그대로 노출, 라인 타입별 아이콘/컬러 분리.
    - PRORATION: 추가 청구(양수 강조).
    - CREDIT/REFUND: 차감(음수 강조).
    - MEMBERSHIP/DISCOUNT/ADJUSTMENT: 중립 톤.
  - 합계 영역: 라인 합계와 `amountDue` 비교 표시로 금액 근거 설명.
  - 차지 섹션: 결제 상태/유형/금액/paidAt 표시.
- 요약 화면
  - 월별 합계/건수 그래프 또는 카드형 요약 제공.

## 6. 업그레이드/다운그레이드/환불 표시 정책
- 업그레이드 결제 성공 시
  - PRORATION 라인(양수): `업그레이드 차액 <remaining>/<total>일, <currentPrice>→<targetPrice>`
  - Charge 금액 = 차액, Charge `chargeType=UPGRADE/PRORATION`
- 다운그레이드 예약 시
  - CREDIT 라인(음수): `다운그레이드 크레딧 <remaining>/<total>일, <currentPrice>→<targetPrice>`
  - 결제 없음, 다음 주기부터 새 플랜 라인 생성
- 환불
  - CREDIT/REFUND 라인(음수): `환불: <policyMessage>, 금액=<refundableAmount>`

## 7. 데이터/타입 설계
- `src/types/invoice.ts` 신설
  - `InvoiceStatus = "PENDING" | "PAID" | "VOID" | "REFUNDED"`
  - `InvoiceLineType = "MEMBERSHIP" | "PRORATION" | "DISCOUNT" | "CREDIT" | "ADJUSTMENT" | "REFUND"`
  - `ChargeStatus = "PENDING" | "PAID" | "FAILED" | "CANCELLED" | "PARTIAL_CANCELLED" | "REFUNDED"`
  - `Invoice`, `InvoiceLine`, `InvoiceCharge`, `InvoiceListResponse`, `InvoiceSummaryItem` 정의.
- `src/model/InvoiceRepository.ts` (클라이언트 전용)
  - BASE_PATH=`/api/invoices`, `fetch` 공통 옵션 `credentials: "include"`.
  - `getInvoices(params)` → `ClientApiResponse<InvoiceListResponse>` 반환.
  - `getInvoiceDetail(invoiceId)` → `ClientApiResponse<{ invoice: Invoice; lines: InvoiceLine[]; charges: InvoiceCharge[] }>` 반환.
  - `getInvoiceSummary(params)` → `ClientApiResponse<InvoiceSummaryItem[]>` 반환.
- TanStack Query는 페이지/컴포넌트 훅에서 직접 사용(`queryKey`: `["invoices", filters]`, `["invoice", invoiceId]`, `["invoiceSummary", filters]`).
- CSV 다운로드는 fetch 후 `blob()` → `URL.createObjectURL` → `<a download>` 패턴.

## 8. 권한, 에러, 로딩 처리
- 라우팅 가드: 관리자 전용 섹션에 배치(`(dplog)` 내부 admin 전용 레이아웃)하고, 비관리자 접근 시 `/` 리다이렉트.
- 에러: 401/403 → 로그인/권한 안내, 404 → 인보이스 없음 화면, 기타 → 재시도 버튼 + 로그(`logError`).
- 로딩: 스켈레톤/스피너 제공, CSV 다운로드 중에는 버튼 disabled + progress 토스트.

## 9. 테스트 계획 (Playwright 우선)
- 목록 필터/페이지네이션이 동일한 쿼리스트링과 API 호출을 만드는지 검증.
- CSV 다운로드가 현재 필터와밋 동일한 결과를 반환하고 파일 헤더가 맞는지 확인.
- 상세 페이지에서 lineType별 배지/아이콘 색상 및 금액 부호 표시 검증.
- 요약 화면에서 월별 합계/건수가 정확히 노출되는지 확인.
- 업/다운/환불 플로우에서 서버가 준 description/amount를 그대로 보여주는지 스냅샷 검사.
- 권한 없는 사용자가 접근 시 로그인/권한 안내로 리다이렉트 되는지 확인.

## 11. 백엔드에 추가 요청/확인 필요 사항 (프런트 정합성)
아래 선택지 중 맞는 것을 선택해 주세요.
1) 날짜 포맷: `yyyy-MM-dd HH:mm`(TZ 미포함, KST 기준).
2) status enum 응답: 한글 포맷(대기/결제완료/무효/환불완료) 여부.
3) sort 기본/허용: 기본 `issuedAt,DESC`, 허용 `issuedAt,dueAt,userId,subscriptionId` (page/size 기본 page=0,size=20).
4) 권한: ADMIN 전용(`@PreAuthorize("hasRole('ADMIN')")`), Authorization 헤더 사용.
5) CSV: 현행 헤더/쉼표/UTF-8 유지.
6) lineType/description: 한글 설명 포함 노출 확정(업/다운/환불 문구 한글).

## 12. 프런트 전달용 요약 (현재 백엔드 상태)
- 날짜 포맷: `yyyy-MM-dd HH:mm` (KST, TZ 미포함)으로 응답.
- status: 한글 변환(대기/결제완료/무효/환불완료).
- 라인 타입: 한글 라벨(`lineTypeLabel`) 제공, description은 서버 설정 그대로 노출.
- 권한: ADMIN 전용(`@PreAuthorize("hasRole('ADMIN')")`), Authorization 헤더 기반 가정.
- 목록 API: `GET /api/invoices?status&from&to&userId&subscriptionId&lineType&lineDescription&page&size&sort`
  - `sort` 미지정 시 `issuedAt,DESC` 기본 적용.
- 상세 API: `GET /api/invoices/{invoiceId}` (인보이스 + 라인 + 차지).
- 요약 API: `GET /api/invoices/summary` (월별 합계/건수).
- CSV: `format=csv` 시 `invoice_id,...,line_count,line_amount_sum`, 쉼표 구분, UTF-8.
- Charge `paidAt`은 `yyyy-MM-dd'T'HH:mm:ss` 형식(로컬 기준)으로 내려옴.

## 13. 최근 업데이트 요약
- 인보이스 상세 응답에 `charges` 포함(결제 트랜잭션 노출).
- 라인 필터 추가: `lineType`, `lineDescription`으로 인보이스 검색 가능.
- CSV 응답에 `line_amount_sum`(라인 금액 합계) 컬럼 추가.
- `sort` 미지정 시 `issuedAt,DESC` 기본 적용으로 목록 정렬 안정화.
- 라인 타입 한글 라벨(`lineTypeLabel`)을 응답에 포함해 프런트 표시에 사용.
