이 워크플로우는 인스타그램 피드에 다중 이미지 포스트를 자동 발행하는 전체 파이프라인입니다.
에이전트는 아래 단계를 **순서대로** 수행하십시오.

## Step 1. 매장명 확인

사용자에게 어떤 매장의 원고를 발행할지 확인합니다.
- 사용자가 매장명을 명시하지 않았다면: `scripts/asset/` 하위 디렉토리를 `list_dir`로 조회하여 목록을 보여주고 선택하게 합니다.
- 매장명이 확인되면 이후 모든 Step에서 `STORE_NAME` 변수로 사용합니다.

## Step 2. 원고 파일 검증

`scripts/asset/{STORE_NAME}/insta_draft.md` 파일을 `view_file`로 열어 다음을 확인합니다:

1. **이미지 배열**: `![번호](파일명)` 형태, **최대 10장** 제한 확인
2. **이미지 실물 확인**: 각 파일이 `scripts/asset/{STORE_NAME}/` 폴더에 실제 존재하는지 `list_dir`로 교차 확인
3. **캡션**: `---` 구분선 아래에 캡션 텍스트가 존재하는지 확인
4. **해시태그**: 캡션 내 해시태그 포함 여부

> [!WARNING]
> 인스타그램은 한 게시물당 **최대 10장**까지만 업로드 가능합니다. 10장 초과 시 스크립트가 자동으로 10장까지만 잘라냅니다.

> [!IMPORTANT]
> 인스타그램은 정사각형(1:1) 또는 4:5 비율의 이미지를 권장합니다. 가로가 극단적으로 긴 이미지는 자동 크롭됩니다.

## Step 3. 환경 설정 점검

다음 환경 요소들을 점검합니다:

1. **세션 파일** (`scripts/insta_session.json`):
   - 존재 시: "기존 세션 재사용 가능" 안내
   - 없을 시: **치명적** — "대시보드(SNS Dashboard)를 통해 인스타그램 로그인을 먼저 수행하세요" 안내 후 **워크플로우 중단**
2. **Playwright** 설치 여부: `python -c "from playwright.async_api import async_playwright; print('OK')"` 로 임포트 테스트

> [!CAUTION]
> 인스타그램은 네이버와 달리 ID/PW 직접 로그인이 불가능합니다 (2FA + 봇 탐지).
> 반드시 기존 세션 파일을 통한 쿠키 인증만 사용하십시오.

## Step 4. 발행 스크립트 실행

// turbo
```bash
cd /Users/nomadlab/Desktop/김주혁/workspace/eo-website/dplog/scripts && python -m sns_publish.open_insta --store {STORE_NAME}
```

- `{STORE_NAME}` 자리에 Step 1에서 확인한 매장명을 치환합니다.
- 스크립트는 **headless=False, slow_mo=300** 으로 실행되어 브라우저 동작이 시각적으로 보입니다.
- 이미지 10장 기준 전체 실행 시간은 약 2~4분 소요됩니다.

## Step 5. 결과 확인 및 후처리

콘솔 출력을 확인하여 발행 성공 여부를 판단합니다:

| 출력 메시지 | 의미 |
|---|---|
| `✅ 인스타그램 다중 업로드 최종 완료` | ✅ 성공 |
| `❌ 업로드 엘리먼트 탐색 실패` | ❌ UI 구조 변경 가능성 — DOM 셀렉터 업데이트 필요 |
| `❌ 세션 파일이 없습니다` | ❌ Step 3 재점검 |
| `공유 또는 대기 에러` | ❌ 네트워크 타임아웃 또는 인스타 서버 장애 |

**성공 시:**
1. 사용자에게 발행 완료를 알립니다.
2. Threads 동시 공유 여부를 확인합니다 (스크립트가 자동으로 토글 ON 시도).
3. 네이버 블로그 동시 발행 여부를 제안합니다: "네이버에도 발행하시겠습니까? (`/publish-naver`)"

**실패 시:**
1. 스크린샷 파일(step*_error.png)이 생성되었다면 `view_file`로 확인하여 실패 지점을 분석합니다.
2. `docs/planning/TROUBLESHOOTING.md` 참조를 권합니다.
