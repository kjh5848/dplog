이 워크플로우는 새로운 매장의 SNS 콘텐츠를 기획부터 초안까지 일괄 생산하는 파이프라인입니다.
에이전트는 아래 단계를 **순서대로** 수행하십시오.

> **골든 템플릿:** 모든 산출물은 `scripts/asset/철이네/` 폴더의 기존 파일들을 포맷 기준으로 참고합니다.

---

## Step 1. 상점 정보 생성 (`store_info.md`)

사용자에게 다음 정보를 수집합니다. 누락된 항목이 있으면 반드시 되물어 확인합니다.

### 필수 항목
| 항목 | 예시 |
|------|------|
| 가게 이름 | 철이네 |
| 주소 / 네이버 플레이스 URL | https://naver.me/GNWmuDWk |
| 영업시간, 브레이크타임, 휴무일 | 17:00~02:00, 없음, 없음 |
| 예약/연락처 | 010-4191-7814 |
| 대표 메뉴 (시그니처) | 숯불뼈닭발, 오징어순대 |
| 방문 조건 (체험권 금액, 제한사항) | 5만원 체험권, 주류 별도 |

### SEO 항목
| 항목 | 예시 |
|------|------|
| 타겟 키워드 5개 | 해운대술집, 장산역맛집, ... |
| 제목 규칙 | 핵심 키워드 1개 + 업체명 포함 |
| 본문 키워드 반복 | 3회 이상 자연 삽입 |

### 선택 항목
- 숏폼 해시태그 5개
- 특이사항 (인테리어, 사장님 특징 등)
- 톤앤매너 지정 (기본값: 30대 남성 식도락 / 음슴체)

수집된 정보를 `scripts/asset/{STORE_NAME}/store_info.md` 에 저장합니다.
포맷 참고: `scripts/asset/철이네/store_info.md`

## Step 2. 미디어 인덱싱 (`media_index.md`)

`scripts/asset/{STORE_NAME}/` 폴더의 모든 미디어 파일을 **1:1로 직접 열람**하여 텍스트 인덱스를 생성합니다.

### 처리 순서
1. `list_dir`로 폴더 내 미디어 파일 전수 조사 (jpeg, png, gif, mp4, mov)
2. **이미지**: `view_file`로 1장씩 직접 확인 → 내용, 구도, 추천 용도 기술
3. **동영상/GIF**: `view_file`로 확인 가능하면 확인, 불가하면 파일명+크기 기반 추정
4. 촬영 시간대/주제별 **파트 그룹핑**
5. `촬영 파트별 활용 가이드` 매핑 테이블 생성

### 출력 구조
```
# 📂 [{STORE_NAME}] 미디어 1:1 텍스트 인덱싱

## 📸 1. 이미지 파일 목록 (총 N장)
### 파트 A: {주제명} (파일범위)
- **파일명.jpeg**
  - 내용 : {상세 설명}
  - 추천 용도 : {블로그 어느 파트에 쓸지}

## 🎬 2. 동영상/GIF 파일 목록 (총 N개)
- **파일명.mp4**
  - 내용 : {설명}
  - 추천 용도 : {용도}

## 🗂️ 3. 촬영 파트별 활용 가이드
| 블로그 파트 | 추천 이미지/영상 |
|---|---|
| 가게 소개 | 파일명 |
```

포맷 참고: `scripts/asset/철이네/media_index.md`

> [!IMPORTANT]
> 이 단계가 파이프라인의 **병목**입니다. 이미지를 1:1로 읽기 때문에 시간이 소요되지만, 이 캐시가 있어야 이후 시나리오/초안의 사진 매칭 정확도가 올라갑니다.

## Step 3. 시나리오 작성 (`blog_scenario.md`)

`store_info.md` + `media_index.md`를 기반으로 **섹션별 블로그 시나리오**를 작성합니다.

### 시나리오 구조
```
1. 📌 포스팅 기본 설계
   - 제목 후보 3개
   - 썸네일 후보
   - 톤앤매너
   - 총 섹션 수 (8개 전후)

2. 🗂️ 섹션별 시나리오 (SECTION 1 ~ N)
   각 섹션 포함 항목:
   > 분위기 설명
   📸 사진 배치 (파일명 지정)
   🎬 GIF 배치 (파일명 지정)
   ✍️ 원고 초안 (텍스트 블록)

3. 🔄 사진 풀 배치 맵 (포스팅 순서 테이블)

4. 📝 원고 작성 전 체크리스트
```

### 작성 규칙
- 톤앤매너: `store_info.md`에 지정된 페르소나 따름 (기본: 30대 남성 음슴체)
- SEO: `store_info.md`의 키워드 지침 반영
- 사진 최소 15장 이상, GIF 최소 3개 이상 배치
- 포맷 참고: `scripts/asset/철이네/blog_scenario.md`

## Step 4. 플랫폼별 초안 생성

**사용자에게 어떤 SNS 초안을 생성할지 확인합니다.**

다음과 같이 안내합니다:
```
어떤 플랫폼의 초안을 생성할까요? (복수 선택 가능)

1️⃣ 네이버 블로그 (blog_draft.md)
2️⃣ 인스타그램 (insta_draft.md)
3️⃣ Threads (threads_draft.md)
4️⃣ Twitter/X (twitter_draft.md)
5️⃣ Facebook (facebook_draft.md)
🅰️ 전체 (1~5 모두)

예: "1, 2, 3" 또는 "전체"
```

사용자가 선택한 플랫폼만 생성합니다. 각 플랫폼의 규칙은 아래를 따릅니다.

---

### 4-1. 네이버 블로그 (`blog_draft.md`)

| 규칙 | 값 |
|------|-----|
| 글자 수 | **공백 포함 2,000자 이상** (SEO 롱폼) |
| 이미지 | **15장 이상** (다양한 앵글) |
| GIF/동영상 | **3개 이상** |
| 구분선 | 섹션 전환 시 `---` |
| 볼드 | `**텍스트**` 형식 |
| 해시태그 | 첫 줄 + 마지막 줄에 배치 |
| 지도 링크 | 결론부에 네이버 지도 URL |
| 파서 호환 | `![alt](<파일명>)` 이미지 형식 필수 |

**상위노출 핵심 전략 (C-Rank + D.I.A+):**
- 키워드를 제목 + 본문 3회 이상 자연 삽입 (도배 ❌)
- 직접 촬영 사진 + 구조화된 소제목 → 체류 시간 극대화
- 스토리텔링 흐름 설계 → 궁금증 단계적 해소
- AI 뻔한 문장 금지 → 실제 경험 + 감정 담기

포맷 참고: `scripts/asset/철이네/blog_draft.md`

---

### 4-2. 인스타그램 (`insta_draft.md`)

| 규칙 | 값 |
|------|-----|
| 글자 수 | **2,200자 이내** |
| 이미지 | **최대 10장** (엄선) |
| 구조 | 이미지들 → `---` → 캡션 전체 |
| 톤 | 캐주얼, 이모지 활발 |
| 해시태그 | 캡션 끝에 3~5개 (키워드 중심) |
| CTA | "저장해두세요 💾", "같이 갈 친구 태그!" |
| 파서 호환 | `![번호](<파일명>)` 이미지 형식 필수 |

**상위노출 핵심 전략 (공유/저장 중심):**
- DM 공유 ⭐⭐⭐⭐⭐ > 저장 > 체류시간 > 댓글 > 좋아요 순
- 캡션에 키워드 자연 삽입 (해시태그 30개 → 3~5개로 축소)
- 고화질 1080x1350 (4:5) 이미지 / 오리지널 콘텐츠만
- 게시 후 1시간 내 댓글 응답 (관계 시그널)

포맷 참고: `scripts/asset/철이네/insta_draft.md`

---

### 4-3. Threads (`threads_draft.md`)

| 규칙 | 값 |
|------|-----|
| 글자 수 | **파트당 500자** (셀프 리플라이로 체이닝) |
| 이미지 | **파트당 최대 20장** (캐러셀) |
| 동영상 | 파트당 5분 이내 |
| 톤 | **대화체**, 친구한테 카톡하듯 |
| 해시태그 | 마지막 파트에 토픽 태그 1~3개 |
| CTA | 질문형 → 리플라이 유도 |

**상위노출 핵심 전략 (벨로시티 + 대화 깊이):**
- 엔게이지먼트 벨로시티 ⭐⭐⭐⭐⭐ (게시 30~60분 내 반응 속도)
- 리플라이 깊이 ⭐⭐⭐⭐⭐ (답글의 답글이 핵심)
- 질문형/의견 유도 → 대화 깊이 극대화
- 텍스트 + 이미지 혼합 (텍스트만보다 참여율 ↑)
- ❌ 타 플랫폼 콘텐츠 그대로 복붙 금지 (네이티브 느낌)

**초안 구조:**
```markdown
## [PART 1 — 후킹] (≤500자)
스크롤 멈추게 하는 한 줄 + 핵심 이미지 2~3장

---

## [PART 2 — 본문] (≤500자)
메뉴별/주제별 상세 + 이미지

---

## [PART 3 — CTA] (≤500자)
위치정보 + 토픽태그 + 질문형 마무리
```

각 파트는 `---` 구분선으로 분리. 파트별 개별 포스팅 → 셀프 리플라이 체이닝.

---

### 4-4. Twitter/X (`twitter_draft.md`)

| 규칙 | 값 |
|------|-----|
| 글자 수 | **트윗당 280자** (셀프 리플라이로 스레드) |
| 이미지 | **트윗당 최대 4장** |
| 동영상 | 트윗당 140초 이내 (무료) |
| URL | 자동 단축 (23자 고정, 글자 수에 포함) |
| 톤 | **초단문 + 위트**, 호기심 유발 |
| 해시태그 | 2~3개 (글자 수에 포함!) |

**상위노출 핵심 전략 (드웰타임 + SimClusters):**
- 리플라이(특히 원작자 응답) ⭐⭐⭐⭐⭐
- 드웰타임 ⭐⭐⭐⭐ (멈춰서 읽는 시간)
- 네이티브 미디어(직접 업로드) 사용 → 외부 링크 본문 삽입 ❌
- ❌ 링크는 리플라이에 배치 (본문 링크 = 노출 억제)

**초안 구조:**
```markdown
## [TWEET 1 — 후킹] (≤280자)
후킹 문장 + "🧵 스레드 ↓" + 대표 이미지 1장

---

## [TWEET 2~3 — 본문] (≤280자)
메뉴별 소개 + 이미지 2~4장

---

## [TWEET 4 — CTA] (≤280자)
위치/연락처 + 해시태그 (280자 안에 모두 포함)
```

각 트윗은 `---` 구분선으로 분리. 해시태그와 URL은 글자 수에 포함됨에 유의.

---

### 4-5. Facebook (`facebook_draft.md`)

| 규칙 | 값 |
|------|-----|
| 글자 수 | **후킹 40~80자** + 본문 (모바일 125자 이후 잘림) |
| 이미지 | **캐러셀 최대 10장** |
| 동영상 | 릴스 60초 이내 권장 (최대 240분) |
| 비율 | 4:5 (1080×1350) 또는 1:1 (1080×1080) |
| 톤 | **스토리텔링**, 커뮤니티 대화 유도 |
| CTA | 질문형 "여기 가본 분?", "공유해주세요" |

**상위노출 핵심 전략 (릴스 + 커뮤니티):**
- 공유(Share) ⭐⭐⭐⭐⭐
- 릴스 완주율 ⭐⭐⭐⭐ / 오리지널 콘텐츠 ⭐⭐⭐⭐
- 80/20 법칙: 80% 가치 제공 / 20% 프로모션
- 커뮤니티 그룹 연동 시 노출 ↑
- ❌ 타 플랫폼 워터마크 리포스트, 어그로 참여 유도

**초안 구조:**
```markdown
[후킹 — 40~80자]
모바일에서 잘리지 않는 핵심 한 줄

[본문 — 스토리텔링]
경험 기반 리뷰 + 위치 정보

[CTA]
질문형 마무리 ("여기 가본 분?" / "후기 공유해주세요!")

[이미지 — 캐러셀 최대 10장]
![1](<파일명>)
...
```

## Step 5. 검증 & 완료

모든 초안 생성 후 다음을 자동 검증합니다:

### 공통 검증
1. 각 초안에 참조된 **이미지 파일이 실제로 존재**하는지 `list_dir`로 교차 확인
2. 각 초안의 **글자 수가 플랫폼 제한에 맞는지** 카운트

### 플랫폼별 검증

| 검증 항목 | 네이버 | 인스타 | Threads | Twitter | Facebook |
|----------|--------|--------|---------|---------|----------|
| 글자 수 | ≥2,000 | ≤2,200 | 파트당 ≤500 | 트윗당 ≤280 | 후킹 ≤80 |
| 이미지 수 | ≥15 | ≤10 | 파트당 ≤20 | 트윗당 ≤4 | ≤10 |
| 키워드/태그 | SEO 3회+ | 3~5개 | 토픽태그 | 2~3개 | 시맨틱 |
| 파서 호환 | ✅ | ✅ | — | — | — |
| CTA | 지도링크 | 저장/태그 | 대화유도 | 스레드↓ | 질문/공유 |

### 완료 안내

검증 완료 후 사용자에게 다음을 안내합니다:

```
✅ {STORE_NAME} 콘텐츠 생성 완료!

생성된 파일:
├── store_info.md

├── blog_scenario.md
├── [선택한 플랫폼 초안들]

📌 다음 단계:
- 네이버 발행: /publish-naver
- 인스타 발행: /publish-instagram
- Threads/Twitter/Facebook: 수동 발행 (초안 복사-붙여넣기)
```
