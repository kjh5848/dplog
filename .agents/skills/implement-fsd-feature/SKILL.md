# FSD Feature Implementation 가이드

이 지침은 프론트엔드 프로젝트에서 새로운 기능을 추가하거나 기존 레거시 코드를 마이그레이션할 때, **반드시 지켜야 할 아키텍처, 디자인 시스템, 백엔드 연동, 특수 컨벤션**을 정의합니다. 에이전트는 이 워크플로우를 수행할 때 아래 내용을 엄격히 준수하십시오.

## 1. 글로벌 컨벤션 규칙 (필수 준수)

- **주석 언어**: 생성하는 모든 코드의 주석 및 설명은 반드시 **한국어**로 작성하십시오.
- **마크다운 볼드체 띄어쓰기 규정**:
  - `**강조(텍스트)** 을` (X) -> `**"강조(텍스트)"** 을` 또는 괄호가 포함될 경우 `**텍스트()** ` 처럼 강조 구문 뒤에 반드시 공백을 넣거나 조사 처리에 유의합니다.
  - 일반적인 텍스트를 `**`로 강조할 때, 그 뒤에 오는 조사나 단어 사이에는 **반드시 공백**을 넣어야 합니다.
    - 예시: `**"강조"** 함`, `**"문서(Vector)에서 찾을 것인가?"** 를`
- **Mermaid 다이어그램 괄호 규정**:
  - 다이어그램 노드 라벨 작성 시 괄호 `()`를 포함해야 한다면, 반드시 텍스트 전체를 쌍따옴표 `""`로 감싸야 합니다.
  - 잘못된 예: `A[노드 (설명)]`
  - 올바른 예: `A["노드 (설명)"]`
