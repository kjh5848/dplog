# HTML 미리보기 빌더 스킬

이 스킬은 작성된 블로그 마크다운 초안을 파싱하여 깔끔한 웹 미리보기(HTML) 파일로 변환합니다. 변환 후 즉시 브라우저에서 확인할 수 있는 절대 경로를 사용자에게 제공합니다.

## 사용 시점 (When to use)
- 사용자가 "마크다운 초안 웹에서 보여줘", "HTML로 만들어줘", "미리보기 띄워줘" 라고 요청할 때.
- `create-sns-content` 등 원고 초안(`blog_draft.md`) 생성이 완료된 후, 사용자에게 결과물을 깔끔하게 시각적으로 제공해야 할 때.
- 네이버 블로그 원고의 이미지/GIF 묶음이 2열 그리드, 3열 격자, 슬라이드형 갤러리로 의도대로 보이는지 검수할 때.

## 원고 레이아웃 규칙

자세한 작성 규칙은 `docs/03_가이드라인/네이버블로그_미디어_레이아웃_글쓰기_규칙.md`를 따른다.

- 이미지/GIF 1장만 단독 배치하면 일반 단일 이미지 블록으로 렌더링한다.
- 이미지/GIF 2장 또는 4장을 연속으로 배치하면 2열 그리드로 렌더링한다.
- 이미지/GIF 3장을 연속으로 배치하면 3열 격자로 렌더링한다.
- 이미지/GIF 5장 이상을 연속으로 배치하면 가로 스크롤 슬라이드형 갤러리로 렌더링한다.
- 본문 문장, 제목, 구분선, 지도, URL이 중간에 들어가면 이미지 묶음이 끊긴다.
- MP4/MOV/WEBM은 동영상 블록으로 처리된다. 네이버 이미지형 발행이 목표라면 `make-gif` 스킬로 GIF 변환 후 `.gif`를 원고에 넣는다.

## 실행 절차 (Workflow)

1. **대상 매장 확인**
   어떤 매장의 초안을 빌드할지 `{STORE_NAME}`을 확인합니다.

2. **빌드 스크립트 실행**
   아래 명령어를 실행하여 파이썬 스크립트를 구동합니다.
   // turbo
   ```bash
   cd /Users/nomadlab/Desktop/김주혁/workspace/eo-website/dplog && ./scripts/.venv/bin/python .agents/skills/build-html-preview/scripts/build_html.py --store {STORE_NAME}
   ```

3. **결과 링크 제공**
   스크립트 실행이 성공하면 터미널에 `file:///.../index.html` 형태의 절대 경로가 출력됩니다. 이 경로를 바탕으로 사용자에게 브라우저에서 클릭하여 열어볼 수 있도록 마크다운 링크 형태로 안내합니다.
   - 답변 예시: `👉 [미리보기 열기](file:///...)`
