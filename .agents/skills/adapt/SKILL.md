다양한 컨텍스트(화면 크기, 디바이스, 플랫폼, 사용 사례 등)에서 효과적으로 작동하도록 기존 디자인을 조정하세요.

## 필수 준비 사항

`frontend-design` 스킬을 사용하세요. 이 스킬에는 디자인 원칙, 안티 패턴 및 **컨텍스트 수집 프로토콜(Context Gathering Protocol)**이 포함되어 있습니다. 작업을 시작하기 전에 프로토콜을 따르세요. 아직 디자인 컨텍스트가 없다면 가장 먼저 `teach-impeccable`을 실행해야 합니다. 추가로 대상 플랫폼/디바이스와 사용 컨텍스트를 파악하세요.

## 조정(Adaptation) 과제 평가

무엇을 왜 조정해야 하는지 이해하세요:

1. **소스 컨텍스트 파악**:
   - 원래 어떤 용도로 디자인되었나요? (데스크톱 웹? 모바일 앱?)
   - 어떤 가정이 있었나요? (스마트폰보다 큰 화면? 마우스 입력? 빠른 연결 속도?)
   - 현재 컨텍스트에서 잘 작동하는 부분은 무엇인가요?

2. **타겟 컨텍스트 파악**:
   - **디바이스**: 모바일, 태블릿, 데스크톱, TV, 스마트워치, 인쇄물?
   - **입력 방식**: 터치, 마우스, 키보드, 음성, 게임패드?
   - **화면 제약 조건**: 크기, 해상도, 화면 방향?
   - **연결 상태**: 빠른 Wi-Fi, 느린 3G, 오프라인?
   - **사용 컨텍스트**: 이동 중 vs 책상 앞, 빠르게 훑어보기 vs 집중해서 읽기?
   - **사용자 기대치**: 해당 플랫폼에서 사용자가 기대하는 바는 무엇인가요?

3. **조정 과제 파악**:
   - 화면에 맞지 않는 부분은 무엇인가요? (콘텐츠, 내비게이션, 기능)
   - 제대로 작동하지 않는 부분은 무엇인가요? (터치 환경에서의 hover 상태, 너무 작은 터치 영역)
   - 부적절한 패턴은 무엇인가요? (모바일에 데스크톱 패턴 적용, 데스크톱에 모바일 패턴 적용)

**중요(CRITICAL)**: 조정은 단순히 크기를 조절하는 것이 아닙니다. 새로운 컨텍스트에 맞춰 경험을 다시 생각하는 과정입니다.

## 조정 전략 수립

컨텍스트에 맞는 전략을 세우세요:

### 모바일 조정 (데스크톱 → 모바일)

**레이아웃 전략**:
- 다단 레이아웃 대신 단일 열(Single column) 레이아웃 사용
- 요소를 나란히 배치하는 대신 수직으로 쌓기
- 고정 너비 대신 전체 너비(Full-width) 컴포넌트 사용
- 상단/측면 내비게이션 대신 하단 내비게이션 사용

**인터랙션 전략**:
- 터치 타겟 최소 44x44px (hover에 의존하지 않기)
- 적절한 곳에 스와이프 제스처 사용 (목록, 캐러셀)
- 드롭다운 대신 바텀 시트(Bottom sheets) 사용
- 엄지손가락 중심 디자인 (조작 요소를 엄지 영역 내에 배치)
- 더 넓은 터치 영역과 충분한 간격

**콘텐츠 전략**:
- 점진적 정보 공개 (한 번에 모든 것을 보여주지 않기)
- 주요 콘텐츠 우선 배치 (부가적인 콘텐츠는 탭/아코디언에 숨기기)
- 더 짧고 간결한 텍스트
- 더 큰 텍스트 크기 (최소 16px)

**내비게이션 전략**:
- 햄버거 메뉴 또는 하단 내비게이션
- 내비게이션 복잡성 줄이기
- 컨텍스트 유지를 위한 스티키 헤더(Sticky headers)
- 내비게이션 흐름 안에 위치한 뒤로 가기 버튼

### 태블릿 조정 (하이브리드 접근 방식)

**레이아웃 전략**:
- 2단 레이아웃 (단일 열이나 3단 레이아웃 피하기)
- 부가 콘텐츠를 위한 사이드 패널
- 마스터-디테일 뷰 (목록 + 상세 정보)
- 화면 방향(가로/세로)에 따른 어댑티브 디자인

**인터랙션 전략**:
- 터치와 포인터 모두 지원
- 터치 타겟은 44x44px로 하되 모바일보다 더 밀도 있는 레이아웃 허용
- 측면 내비게이션 드로어(Side navigation drawers)
- 가능한 경우 다단 폼(Multi-column forms) 사용

### 데스크톱 조정 (모바일 → 데스크톱)

**레이아웃 전략**:
- 다단 레이아웃 (가로 공간 활용)
- 항상 표시되는 측면 내비게이션
- 여러 정보 패널을 동시에 표시
- max-width 제한을 둔 고정 너비 (4K 화면 끝까지 늘어나지 않도록 주의)

**인터랙션 전략**:
- 추가 정보를 위한 hover 상태 활용
- 키보드 단축키
- 우클릭 컨텍스트 메뉴
- 필요한 곳에 드래그 앤 드롭 지원
- Shift/Cmd 조합으로 다중 선택

**콘텐츠 전략**:
- 점진적 정보 공개를 줄이고 처음에 더 많은 정보를 보여줌
- 여러 열로 구성된 데이터 테이블
- 더 풍부한 데이터 시각화
- 더 상세한 설명

### 인쇄물 조정 (화면 → 인쇄물)

**레이아웃 전략**:
- 논리적인 지점에서 페이지 나누기
- 내비게이션, 푸터, 상호작용 요소 제거
- 흑백 (또는 제한된 색상)
- 제본에 적합한 여백 설정

**콘텐츠 전략**:
- 축약된 콘텐츠 펼치기 (전체 URL 및 숨겨진 섹션 표시)
- 페이지 번호, 헤더, 푸터 추가
- 메타데이터 포함 (인쇄 날짜, 페이지 제목)
- 차트를 인쇄에 적합한 버전으로 변환

### 이메일 조정 (웹 → 이메일)

**레이아웃 전략**:
- 좁은 너비 (최대 600px)
- 단일 열 레이아웃만 사용
- 인라인 CSS (외부 스타일시트 사용 불가)
- 테이블 기반 레이아웃 (이메일 클라이언트 호환성 확보)

**인터랙션 전략**:
- 크고 명확한 CTA (텍스트 링크 대신 버튼)
- hover 상태 지양 (이메일에서 안정적이지 않음)
- 복잡한 상호작용은 딥 링크를 통해 웹 앱으로 유도

## 조정 사항 구현

다음 기법들을 체계적으로 적용하세요:

### 반응형 분기점 (Breakpoints)

적절한 분기점 선택:
- 모바일: 320px - 767px
- 태블릿: 768px - 1023px
- 데스크톱: 1024px 이상
- 또는 콘텐츠 기반 분기점 (디자인이 깨지는 지점에서 설정)

### 레이아웃 조정 기법

- **CSS Grid/Flexbox**: 레이아웃 자동 재정렬
- **Container Queries**: 뷰포트가 아닌 컨테이너 크기에 기반한 조정
- **`clamp()`**: 최소/최대 값 사이에서 유동적으로 크기 조정
- **Media queries**: 컨텍스트에 따라 다른 스타일 적용
- **Display 속성**: 컨텍스트별로 요소 표시/숨기기

### 터치 환경 조정

- 터치 타겟 크기 확대 (최소 44x44px)
- 상호작용 요소 간격 넓히기
- hover에 의존하는 인터랙션 제거
- 터치 시각 피드백 추가 (리플 효과, 하이라이트)
- 엄지손가락 반경 고려 (화면 상단보다 하단에 접근하기 쉬움)

### 콘텐츠 조정

- `display: none` 사용 최소화 (여전히 리소스를 다운로드함)
- 점진적 향상 (핵심 콘텐츠 먼저 구현 후 큰 화면에서 추가 정보 향상)
- 화면 밖 콘텐츠에는 지연 로딩(Lazy loading) 적용
- 반응형 이미지 사용 (`srcset`, `picture` 요소)

### 내비게이션 조정

- 모바일에서는 복잡한 내비게이션을 햄버거/드로어로 변환
- 모바일 앱은 주로 하단 내비게이션 바 적용
- 데스크톱에서는 측면 내비게이션을 고정 노출
- 작은 화면에서는 경로 파악을 위한 브레드크럼(Breadcrumbs) 제공

**중요**: 브라우저 개발자 도구가 아닌 실제 디바이스에서 테스트하세요. 디바이스 에뮬레이터는 유용하지만 완벽하지 않습니다.

**절대 안 됨 (NEVER)**:
- 모바일에서 핵심 기능 숨기기 (중요하다면 모바일에서도 작동하게 만드세요)
- 데스크톱 사용자가 항상 최신/고사양 기기를 쓸 것으로 단정하기 (접근성, 구형 기기 고려)
- 컨텍스트 간 정보 구조를 다르게 구성하기 (사용자에게 혼란을 줌)
- 특정 플랫폼에 대한 사용자의 기대를 저버리기 (모바일 사용자는 모바일 문법을 기대합니다)
- 모바일/태블릿의 가로 모드 (Landscape) 고려 누락
- 아무 이유 없이 일반적인 분기점 값 사용하기 (콘텐츠 기반 분기점을 활용하세요)
- 데스크톱 화면에서 터치 지원 무시하기 (많은 데스크톱/노트북이 터치스크린을 지원함)

## 결과 확인

다양한 환경에서 철저하게 테스트하세요:

- **실제 디바이스**: 실제 스마트폰, 태블릿, 데스크톱에서 테스트
- **화면 방향**: 세로 모드(Portrait) 및 가로 모드(Landscape)
- **여러 브라우저**: Safari, Chrome, Firefox, Edge 등
- **OS 테스트**: iOS, Android, Windows, macOS
- **입력 장치별**: 터치, 마우스, 키보드
- **예외 케이스**: 아주 작은 화면(320px)부터 초대형 화면(4K)까지 테스트
- **통신 상태**: 네트워크 속도를 제한(Throttle)한 환경에서 테스트

기억하세요: 당신은 크로스 플랫폼 디자인 전문가입니다. 핵심 기능과 브랜드의 일관성을 유지하면서, 각각의 컨텍스트에 맞춰 네이티브 앱처럼 자연스러운 경험을 만들어야 합니다. 의도적으로 조정하고 꼼꼼하게 테스트하세요.
