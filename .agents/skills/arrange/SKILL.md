단조롭고, 가시성이 떨어지거나 시각적 구조가 약한 레이아웃과 간격을 평가하고 개선합니다 — 뻔한 배치를 의도적이고 리듬감 있는 레이아웃으로 변환하세요.

## 필수 준비 사항

`frontend-design` 스킬을 활용하세요 — 여기에는 디자인 원칙, 안티 패턴 및 **컨텍스트 수집 프로토콜(Context Gathering Protocol)**이 포함되어 있습니다. 작업을 시작하기 전에 반드시 지침을 따르십시오 — 아직 디자인 컨텍스트 정보가 없다면 무엇보다 먼저 `teach-impeccable`을 실행해야 합니다.

## 현재 레이아웃 평가

현재의 공간 디자인에서 취약한 부분을 분석하세요:

1. **간격 (Spacing)**:
   - 간격 체계가 일관적인가요, 아니면 무작위인가요? (원칙 없는 임의의 패딩/마진 사용 여부)
   - 모든 여백의 값이 동일한가요? (어디나 똑같은 패딩 = 리듬감 부족)
   - 연관성이 높은 요소들은 조밀하게 그룹화되어 있고, 서로 다른 그룹 간에는 넉넉한 여백이 존재하나요?

2. **시각적 계층 구조 (Visual Hierarchy)**:
   - 반눈을 뜨고 흐리게 쳐다보는 테스트(Squint test)를 해보세요 — 눈을 살짝 찡그려 흐릿하게 보아도, 가장 중요한 요소, 두 번째로 중요한 요소, 그리고 각 그룹의 덩어리가 뚜렷하게 식별되나요?
   - 중요도가 올바르게 전달되고 있나요? (공간 여백과 글자의 무게감만으로도 충분히 전달할 수 있습니다 — 현재의 방식이 효과적인가요?)
   - 넓은 여백(Whitespace)이 사용자의 시선을 가장 중요한 곳으로 자연스럽게 안내하나요?

3. **그리드 및 구조 (Grid & Structure)**:
   - 기저에 깔린 명확한 구조가 눈에 보이나요, 아니면 무작위로 욱여넣은 듯 하나요?
   - 똑같이 생긴 카드 그리드가 무한정 남발되고 있지 않나요? (아이콘 + 제목 + 텍스트 덩어리의 무한 반복)
   - 모든 것이 중앙 정렬되어 있나요? (절대적인 규칙은 아니지만, 보통 왼쪽 정렬 및 비대칭 레이아웃이 훨씬 더 고안된 디자인처럼 느껴집니다)

4. **리듬 및 다양성 (Rhythm & Variety)**:
   - 레이아웃에 리듬감이 존재하나요? (타이트한 그룹과 넉넉한 공간이 번갈아 나타나는가)
   - 모든 섹션의 골격이 복붙된 듯이 똑같나요? (단조로운 반복 지양)
   - 강렬한 강조점이나 반전(Surprise)을 의도한 공간이 있나요?

5. **정보 밀도 (Density)**:
   - 위아래가 너무 심하게 꽉 막혀 있지 않나요? (숨이 막히는 답답한 공간)
   - 너무 비어 있어 허전하지 않나요? (의도 없는 과도한 여백 남발)
   - 밀도가 콘텐츠의 목적에 부합하나요? (데이터가 많은 대시보드 UI는 타이트하게, 시선을 끌어야 하는 마케팅 페이지는 넉넉하게 숨통을 틔워야 함)

**중요(CRITICAL)**: 아무리 폰트와 컬러를 완벽하게 조합해도 레이아웃이나 간격이 어설프면 인터페이스 전체가 저렴해 보입니다. 여백(Space)도 훌륭한 디자인 자재입니다 — 명확한 의도를 가지고 다루세요.

## 레이아웃 개선 계획

그리드 시스템, 시각적 리듬, 및 컨테이너 쿼리에 관한 구체적인 가이드는 `frontend-design` 스킬의 [공간 디자인 레퍼런스(reference/spatial-design.md)]를 참고하세요.

체계적인 계획을 수립하세요:

- **간격 시스템 (Spacing system)**: 일관된 스케일을 유지하세요 — 프레임워크가 제공하는 룰(예: Tailwind), rem 기반 토큰, 혹은 커스텀 시스템 등 어느 방식이든 상관없습니다. 구체적인 값 자체보다는 절대적인 일관성을 지키는 것이 핵심입니다.
- **계층 구조 전략 (Hierarchy strategy)**: 시각적인 여백을 도구로 써서 객체의 중요도를 어떻게 전달할 것인가?
- **레이아웃 접근 방식 (Layout approach)**: 현재 콘텐츠에 가장 적합한 뼈대 구조는 무엇인가? (1차원 배치엔 Flex, 2차원 배치엔 Grid, 복잡한 전역 페이지 레이아웃엔 Named Areas)
- **리듬 (Rhythm)**: 어디를 조여서 바짝 붙이고, 어디를 넉넉하게 띄워둘 것인가?

## 체계적인 레이아웃 개선

### 간격 시스템 확립

- 일관된 간격 스케일을 활용하세요 — 프레임워크 기본 제공 치수, rem 기반 토큰 스케일 등. 가장 중요한 건 임의의 난수 값이 아닌, 지정된 세트 안에서 설계값을 적용하는 것입니다.
- 커스텀 변수를 도입할 때는 직관적이고 시맨틱하게 이름을 부여하세요: `--spacing-8` 방식보다는 `--space-xs` 에서 `--space-xl` 등.
- 형제(Sibling) 요소 간의 간격을 벌릴 때는 margin이 아닌 `gap`을 사용하세요 — 귀찮은 margin 병합 버그를 원천 차단합니다.
- 큰 화면일수록 여백의 호흡이 유연하게 커질 수 있도록 `clamp()` 를 적극 활용하세요.

### 시각적 리듬 창조

- 관련된 속성끼리는 **타이트하게 그룹화** (형제 객체들 사이를 8-12px 간격으로 바짝 묶음)
- 연관성이 없는 다른 성격의 독립적인 섹션 간에는 **충분한 분리감 제공** (48-96px로 넉넉하게 단절)
- 섹션 내에서도 기계적인 동일한 간격 배제 — 행마다 맹목적으로 똑같은 `gap`이 들어갈 이유는 없습니다.
- **비대칭 (Asymmetric) 컴포지션** — 언제나 중앙에 배치하는 습관에서 벗어나세요. 명확한 의도가 있다면 과감하게 틀을 깨보는 것이 좋습니다.

### 적절한 레이아웃 도구 선택

- **1차원 배치에는 Flexbox 사용**: 가로 일렬 배치(Row), 내비게이션 바, 버튼 묶음, 카드 컴포넌트 내부 등, 대부분의 구조적 배치 작업에 Flex가 훨씬 빠르고 최적화에 어울립니다.
- **2차원 배치에는 Grid 사용**: 최상위 페이지 레이아웃 구조, 대시보드, 데이터 밀집 구조 형태, 열(Row)과 행(Column)의 동시 제어가 필수적인 곳에 적합.
- `flex-wrap`을 쓰면 단순하게 끝날 문제를 **기계적으로 Grid로 도배하지 마세요.**
- 분기점(Breakpoint) 선언 없이 유연하게 깨지지 않는 로우 밸런싱을 원한다면 `repeat(auto-fit, minmax(280px, 1fr))` 패턴을 활용하세요.
- 복잡한 페이지의 골격을 잡을 때는 `grid-template-areas`를 이용하여 Area를 네이밍하고 브레이크포인트에서 단위를 재정의하세요.

### 단조로운 카드 그리드 탈피

- 무조건 모든 요소 상위 컨테이너를 카드로 덮어씌울 필요는 없습니다 — 빈 공간과 적절한 정렬만으로도 시각적인 그룹핑은 충분합니다.
- 카드는 해당 객체의 영역이 고유하게 차별화되어 분리되고, 명확한 클릭 인터랙션을 지니고 있을 때만 활용하세요 — **절대 카드 안에 카드를 중첩시켜서 디자인하지 마세요.**
- 카드 규격에 변화를 주거나 비율을 스팬(Span)으로 늘리고, 혹은 카드 외의 일반 컴포넌트와 혼합 배치함으로써 기계적인 복제 나열을 막아보세요.

### 강력한 시각 계층 구조 확립

- 계층 구조를 명확하게 드러내기 위해 굳이 모든 폰트 굵기, 화려한 색상을 총동원할 필요가 없습니다. 공간(Space)만 창조해도 무방합니다 — 중요 객체 하나를 넉넉한 여백 한가운데에 배치해 보세요. 세상에서 제일 세련된 디자인은 대부분 단지 여백과 폰트의 무게감만으로 완벽한 리듬 제어를 해내는 결과물입니다. 이를 통해 1차 해결이 안 될 때만 색상이나 명도의 채널을 동원하세요.
- 사용자 시선 인식 방향을 감안하세요 — LTR 언어에선 좌상단에서 시작하여 우하단으로 향하지만, 다이얼로그의 주요 액션 버튼은 우측 하단, 네비게이션은 상단으로 가는 식으로 기능적 맥락을 반영해야 합니다.
- 게슈탈트 근접성의 원리(Proximity)를 극대화하여 거리에 따라 자연스럽게 컨텐츠 그룹을 구분하세요.

### 뎁스(Depth) 및 그림자 엘리베이션 관리

- 구조적인 z-index 시맨틱 스케일 확립 (드롭다운 → 스티키 → 모달 배경 → 모달 → 토스트 → 툴팁 순서로 상승)
- 단계를 지닌 그림자 스케일 설계 (sm → md → lg → xl) — 그림자 제어는 한쪽으로 쏠리지 않고 은은해야 합니다.
- Z축(Elevation)을 활용하는 이유는 단순한 스킨 장식이 아니고 최상위 요소 공간의 계층 구조 강조를 위함입니다.

### 시각적 광학(Optical) 미세 조정

- 만약 아이콘이 내부 픽셀 기준 수학적으로는 완벽하게 정중앙에 배치되어 있는데도 불구하고, 눈으로 보았을 때는 약간 기울어져 어색하게 느껴진다면, 오로지 이질적인 경우에 한해서만 시각 보정에 기대어 n 픽셀을 옮기십시오. 단, 이것이 맞다는 확신이 들 때만 진행하세요.

**절대 금지 (NEVER)**:
- 정해진 룰셋 치수에 포함되지 않는 뜬금없는 임의의 값 사용
- 모든 패딩과 마진에 똑같은 값 도배하기 — 차이가 있어야 위계가 성립됩니다.
- 화면 내에 던져 넣은 모든 컴포넌트에 액자처럼 테두리 박스 카드 사용하기
- 카드 컴포넌트 내부에 또다시 카드 디자인 넣기 — 선(Divider)을 긋거나 여백으로 분리하세요.
- 모두 똑같이 찍어낸 듯한 (아이콘 + 제목 + 문단 텍스트 조합의) 무한 카드 낭비
- 모든 요소를 강제 타이틀 센터(가운데 정렬)로 묶어두기 — 비대칭 레이아웃이 훨씬 자연스럽고 뛰어난 디자인 감각을 어필할 수 있습니다.
- (거대한 숫자 + 작은 요약 텍스트 + 우상향 통계 그래프 + 그라데이션)과 같은 식상한 히어로 메트릭 대시보드를 템플릿처럼 베껴 넣지 마세요. 사용자의 실제 리얼 데이터를 다룬다면 메트릭이 중요할 수 있으나, 단지 예뻐 보이는 숫자 그래픽 조각으로 소모했다간 역효과를 일으킵니다.
- Flexbox보다 오히려 코드가 훨씬 길어지고 불필요하게 복잡해지는 곳에 무작정 CSS Grid 때려 박기
- 체증을 막는다는 명목으로 999, 9999 처럼 아무 기준 없는 z-index 최상위 선언하기

## 레이아웃 개선 결과 확인

- **반눈 뜨기 테스트(Squint test)**: 흐릿하게 구도를 짚어봐도, 최우선 객체와 보조 객체의 구분, 그리고 그룹화 영역 식별이 가능한가요?
- **리듬감**: 페이지 여백에서 밀고 당기는 조임스런 텐션과 부드러운 환기 공간의 쾌감이 살아 숨 쉬나요?
- **명확한 계층구조**: 단 2초 만에 이 페이지가 말하고자 하는 중요한 컨텐츠가 무엇인지를 식별할 수 있나요?
- **충분한 숨통 공간**: 콘텐츠가 질식할 것처럼 좁지도 않고, 목적 잃은 우주처럼 허벌벌 낭비되는 공간 없이 잘 숨 쉬고 있나요?
- **기준의 통일성**: 여백 간격의 시스템 모듈 규격이 어느 지점에서든 일관되게 적용되었나요?
- **반응형 최적화**: 화면의 가로 너비가 비틀어지더라도 각 요소들이 찌그러지거나 겹치지 초자연스러운 형태로 우아하게 변형되나요?

기억하세요: 디자인에서 가장 다루기 힘들면서도 역설적으로 가장 활용도가 낮은 소외된 도구가 디자인 여백(Space)입니다. 강력한 시각적 리듬감과 계층의 밀도는 아주 사소하고 뻔한 요약 텍스트마저도 치밀하게 설계된 프리미엄 결과물처럼 체감되게 하는 위력을 발휘합니다.
