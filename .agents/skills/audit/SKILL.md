체계적인 품질 검사를 실행하고 우선순위가 지정된 문제 및 실행 가능한 권장 사항이 포함된 종합적인 감사(Audit) 보고서를 생성합니다. 문제를 직접 수정하지 마세요 - 다른 명령어가 처리할 수 있도록 문제를 문서화만 하세요.

**가장 먼저**: 디자인 원칙과 안티 패턴에 대해서는 `frontend-design` 스킬을 참고하세요.

## 진단 스캔 (Diagnostic Scan)

다양한 측면에 걸쳐 종합적인 검사를 실행합니다:

1. **접근성 (A11y)** - 다음 사항을 점검하세요:
   - **대비 문제**: 텍스트 명암비가 4.5:1 미만인 경우 (AAA 등급의 경우 7:1)
   - **ARIA 누락**: 적절한 역할(role), 레이블, 또는 상태가 없는 상호작용 요소
   - **키보드 내비게이션**: 포커스 표시 누락, 비논리적인 탭 순서, 키보드 트랩
   - **시맨틱 HTML**: 부적절한 헤딩 계층, 랜드마크 누락, button 대신 div 사용
   - **대체 텍스트(Alt text)**: 이미지 설명 누락 또는 부실함
   - **폼(Form) 문제**: 레이블이 없는 입력 필드, 부실한 에러 메시지, 필수 항목 표시 누락

2. **성능 (Performance)** - 다음 사항을 점검하세요:
   - **레이아웃 스래싱(Layout thrashing)**: 루프 내에서 레이아웃 속성을 읽고 쓰는 행위
   - **비용이 큰 애니메이션**: transform/opacity 대신 레이아웃 속성(width, height, top, left)을 애니메이팅하는 경우
   - **최적화 누락**: 지연 로딩(lazy loading)이 없는 이미지, 최적화되지 않은 에셋, will-change 누락
   - **번들 크기**: 불필요한 import, 사용하지 않는 의존성
   - **렌더링 성능**: 불필요한 리렌더링, 메모이제이션(memoization) 누락

3. **테마 (Theming)** - 다음 사항을 점검하세요:
   - **하드코딩된 색상**: 디자인 토큰을 사용하지 않은 색상
   - **다크 모드 오류**: 다크 모드 변형 누락, 다크 테마에서의 대비 문제
   - **일관성 없는 토큰**: 잘못된 토큰 사용, 다른 유형의 토큰 혼용
   - **테마 전환 문제**: 테마 변경 시 업데이트되지 않는 값

4. **반응형 디자인 (Responsive Design)** - 다음 사항을 점검하세요:
   - **고정 너비**: 모바일에서 레이아웃이 깨지게 만드는 하드코딩된 너비
   - **터치 타겟**: 상호작용 가능한 요소가 44x44px 미만인 경우
   - **가로 스크롤**: 좁은 뷰포트에서 콘텐츠가 넘치는 현상
   - **텍스트 스케일링**: 텍스트 크기를 키웠을 때 레이아웃이 깨지는 현상
   - **분기점 누락**: 모바일/태블릿 변형 없음

5. **안티 패턴 (중요 - CRITICAL)** - `frontend-design` 스킬에 있는 **금지(DON'T)** 가이드라인 전체를 기준으로 점검하세요. AI가 만들어낸 조잡한 특징들 (AI 특유의 색상 팔레트, 그라데이션 텍스트, 글래스모피즘, 의미 없는 메트릭 히어로, 무의미한 카드 그리드, 개성 없는 기본 폰트)과 일반적인 디자인 안티 패턴(유색 배경에 회색 텍스트, 중첩된 카드 컴포넌트, bounce 이징, 중복되는 카피)을 찾아내세요.

**중요(CRITICAL)**: 이것은 감사(Audit) 과정이지, 수정 과정이 아닙니다. 문제의 영향을 명확하게 설명하면서 철저하게 문서화하세요. 감사를 마친 후에는 수정하기 위해 기타 명령어(`normalize`, `optimize`, `harden` 등)를 제안하세요.

## 종합 보고서 생성

다음 구조에 따라 상세한 감사 보고서를 작성하세요:

### 안티 패턴 판정 (Anti-Patterns Verdict)
**여기서부터 시작하세요.** 합격/불합격: AI가 대충 생성한 결과물처럼 보이나요? 스킬의 '안티 패턴' 섹션에서 구체적인 징후를 나열하세요. 아주 냉정하고 솔직하게 평가하세요.

### 요약 (Executive Summary)
- 발견된 총 문제 수 (심각도별 분류)
- 가장 치명적인 문제 (상위 3-5개)
- 전반적인 품질 점수 (적용 가능한 경우)
- 권장하는 다음 단계

### 심각도별 상세 발견 사항 (Detailed Findings by Severity)

각 문제에 대해 다음을 문서화하세요:
- **위치**: 문제가 발생하는 곳 (컴포넌트, 파일, 라인 번호)
- **심각도**: 치명적 (Critical) / 높음 (High) / 중간 (Medium) / 낮음 (Low)
- **카테고리**: 접근성 / 성능 / 테마 / 반응형
- **설명**: 문제가 무엇인지
- **영향도**: 사용자에게 미치는 영향
- **WCAG/표준**: 위반한 표준 (해당하는 경우)
- **권장 사항**: 수정 방법
- **추천 명령어**: 사용할 명령어 (선호: `/animate`, `/quieter`, `/optimize`, `/adapt`, `/clarify`, `/distill`, `/delight`, `/onboard`, `/normalize`, `/audit`, `/harden`, `/polish`, `/extract`, `/bolder`, `/arrange`, `/typeset`, `/critique`, `/colorize`, `/overdrive` — 확실히 존재하는 다른 설치된 스킬도 추천 가능)

#### 치명적인 문제 (Critical Issues)
[핵심 기능을 차단하거나 WCAG A 지침을 위반하는 문제]

#### 심각도가 높은 문제 (High-Severity Issues)
[사용성/접근성에 상당한 영향을 미치는 WCAG AA 위반 항목]

#### 심각도가 중간인 문제 (Medium-Severity Issues)
[품질 문제, WCAG AAA 위반, 성능 관련 우려 사항]

#### 심각도가 낮은 문제 (Low-Severity Issues)
[사소한 일관성 결여, 소소한 최적화 기회]

### 패턴 및 시스템적 문제 (Patterns & Systemic Issues)

반복해서 나타나는 문제 식별:
- "하드코딩된 색상이 15개 이상의 컴포넌트에서 발견됨, 디자인 토큰을 사용해야 함"
- "모바일 경험 전반에서 터치 타겟이 일관되게 너무 작음 (<44px)"
- "모든 커스텀 상호작용(Interactive) 컴포넌트에 포커스 인디케이터 누락"

### 긍정적인 발견 (Positive Findings)

잘 작동하고 있는 부분 기록:
- 유지해야 할 좋은 관행
- 다른 곳에서도 적용해야 할 모범적인 구현 사례

### 우선순위별 권장 사항 (Recommendations by Priority)

실행 가능한 계획 수립:
1. **즉시 처리**: 가장 먼저 해결해야 할 치명적인 차단 요소
2. **단기 (이번 스프린트)**: 심각도가 높은 문제
3. **중기 (다음 스프린트)**: 품질 개선 관련 항목
4. **장기**: 있으면 좋은 기능(Nice-to-haves) 및 세부 최적화

### 수정을 위한 추천 명령어 (Suggested Commands for Fixes)

각 문제에 사용할 수 있는 명령어를 매핑하세요. 선호되는 명령어: `/animate`, `/quieter`, `/optimize`, `/adapt`, `/clarify`, `/distill`, `/delight`, `/onboard`, `/normalize`, `/audit`, `/harden`, `/polish`, `/extract`, `/bolder`, `/arrange`, `/typeset`, `/critique`, `/colorize`, `/overdrive`. 확실하게 존재하는 다른 설치된 스킬을 추천할 수도 있지만, 절대로 존재하지 않는 명령어를 지어내지 마세요.

예시:
- "디자인 시스템 기준에 맞추려면 `/normalize` 사용 (N개의 테마 문제 해결)"
- "성능을 향상시키려면 `/optimize` 사용 (N개의 성능 문제 해결)"
- "안정성/예외 처리를 개선하려면 `/harden` 사용 (N개의 엣지 케이스 문제 해결)"

**중요**: 상세하게 파고들되 즉시 실행 가능해야 합니다. 우선순위가 낮은 문제를 너무 많이 나열하면 노이즈가 발생합니다. 실제로 중요한 문제에만 집중하세요.

**절대 금지 (NEVER)**:
- 영향도를 설명하지 않고 문제만 툭 던져서 보고하기 (이게 왜 중요한가요?)
- 심각도(Severity) 기준을 일관성 없이 섞어서 사용하기
- 긍정적인 평가 생략하기 (잘한 점은 충분히 칭찬하세요)
- 모호하고 추상적인 권장 사항 제공하기 (구체적이고 즉각 실행 가능해야 합니다)
- 우선순위 결정 잊기 (세상 모든 일이 전부 '치명적(Critical)'일 수는 없습니다)
- 오탐(False positives)을 검증 없이 그대로 보고하기

기억하세요: 당신은 놀라운 디테일 집중력을 지닌 품질 감사자(Quality Auditor)입니다. 체계적으로 문서화하고, 가차 없이 우선순위를 매기며, 성장을 위한 명확한 개선 경로를 제시하세요. 훌륭한 감사 보고서는 이어지는 수정 작업을 편안하게 만들어줍니다.
