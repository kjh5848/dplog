# D-PLOG v13 모바일/태블릿 반응형 검토 보고서

> 검토일: 2026-02-14  
> 뷰포트: 모바일 375px / 390px, 태블릿 768px  
> 대상: `http://localhost:3000/dplog/v13`

---

## 📋 전체 요약

모바일(375px)에서는 대부분 세로 스택 레이아웃으로 잘 전환되지만, **태블릿(768px)이 가장 심각한 브레이크포인트**입니다. Tailwind의 `md:` 브레이크포인트(768px)에서 데스크톱 레이아웃이 적용되면서 좁은 화면에 폰트와 그리드가 빠듯하게 렌더링됩니다.

| 등급             | 항목 수 | 설명                      |
| ---------------- | ------- | ------------------------- |
| 🔴 **긴급**      | 3       | 가독성·사용성에 직접 영향 |
| 🟡 **개선 필요** | 4       | 시각적 완성도 저하        |

---

## 🔴 긴급 (Critical)

### 1. Navbar — 태블릿(768px) 로그인 텍스트 줄바꿈

|   모바일 (375px)    |   태블릿 (768px)   |
| :-----------------: | :----------------: |
| ✅ 햄버거 메뉴 정상 | ❌ "로그인" 줄바꿈 |

![태블릿 Problem 영역 - Navbar 줄바꿈 확인](./screenshots/tablet_problem.png)

- **문제**: `md:flex`로 768px부터 메뉴가 펼쳐지지만, 네비 항목 + 로그인 + CTA버튼이 768px에 다 들어가지 않아 "로그인" 텍스트가 2줄로 줄바꿈
- **수정**: 데스크톱 메뉴 전환점을 `lg:`(1024px)로 변경

### 2. Hero — 폰트 크기 모바일/태블릿 과대

![모바일 Hero](./screenshots/mobile_hero.png)
![태블릿 Hero](./screenshots/tablet_hero.png)

- **문제**: `text-4xl`(모바일)이 375px에서 화면 전체를 차지하며, 줄바꿈이 "법," 에서 끊어져 리듬감 저하
- **수정**: 모바일 `text-3xl`, 태블릿 중간 크기 `md:text-4xl`, 데스크톱 `lg:text-6xl`로 단계적 적용

### 3. FeatureExplorer — 태블릿 좌우배치 텍스트 좁음

![태블릿 Features](./screenshots/tablet_features.png)
![태블릿 Features2](./screenshots/tablet_features2.png)

- **문제**: `md:flex-row`로 768px부터 좌우배치하지만, 텍스트 영역(`flex-1`)이 ~300px밖에 안 되어 한글이 2-3글자씩 줄바꿈
- **수정**: `lg:flex-row`로 변경하여 1024px 이상에서만 좌우배치

---

## 🟡 개선 필요 (Important)

### 4. PainPoints — 태블릿 3열 카드 좁음

![태블릿 Problem](./screenshots/tablet_problem.png)

- **문제**: `md:grid-cols-3`으로 768px에서 3열이지만 카드 너비 ~220px로 매우 좁음
- **수정**: `lg:grid-cols-3`으로 변경하여 태블릿에서는 1열 유지

### 5. Solution — 태블릿 좌우배치 텍스트 압축

- **문제**: `md:flex-row`로 768px에서 텍스트(40%) + 이미지(60%) 배치, 텍스트 영역 좁음
- **수정**: `lg:flex-row`, `lg:w-[40%]`/`lg:w-[60%]`로 변경

### 6. 모바일 메뉴 — 버튼 텍스트 영문

- **문제**: 모바일 햄버거 메뉴의 하단 버튼이 "Sign In", "Get Started"로 **영문** 표기
- **수정**: "로그인", "무료로 시작하기"로 한글화

### 7. Hero 서브텍스트 — 모바일 줄바꿈 어색

- **문제**: 설명 텍스트가 "콘텐츠와" 이후 줄바꿈되어 읽기 흐름 불자연스러움
- **수정**: `break-keep` 유지하되 `max-w-xl`로 조정하여 줄바꿈 개선
