## 1. 목적
- “대표 5 + 희망 5” 키워드 입력을 일관되게 검증/정규화하여, 저장/순위 조회/리포트 생성 품질을 보장한다.

## 2. 범위
### 2.1 In scope
- 입력 제한(개수, 길이, 공백/특수문자)
- 정규화(normalization) 규칙(트리밍, 연속 공백, 대소문자 등)
- 중복 판단 기준(정규화 후 중복)
- 금칙어 정책(운영 가능하도록 목록/레벨 분리)
- 에러 메시지/코드(사용자 노출 문구 포함)

### 2.2 Out of scope
- 키워드 추천/자동 생성(향후 기능)

## 3. 정책(초안)
- 개수: 대표 5개, 희망 5개(필수 여부 결정 필요)
- 길이: (예) 2~30자 (확정 필요)
- 허용 문자: (예) 한글/영문/숫자/공백 + 일부 기호(확정 필요)
- 공백: 앞/뒤 트림, 중간 연속 공백 1개로 축약
- 중복: 정규화 후 중복 금지(대표↔희망 간 중복도 금지 여부 결정 필요)

## 4. 금칙어(운영)
- 레벨:
  - `BLOCK`: 저장/진단 불가
  - `WARN`: 저장 가능, 경고 노출
- 관리 방식:
  - 서버 설정(환경변수/DB) + 배포 없이 업데이트 가능한 구조 권장

## 5. 수용 기준(AC)
- [ ] 정규화/중복/길이/금칙어 규칙이 명확하다.
- [ ] 에러 코드는 FE/BE에서 동일하게 사용할 수 있다.
