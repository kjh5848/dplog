# D-PLOG: IP 우회 및 분산 스크래핑 아키텍처 전략

본 문서는 네이버 WAF(웹 방화벽) 차단을 우회하고 인프라 비용 0원을 달성하기 위한 **D-PLOG 분산 에이전트(로컬 프록시망) 구축 전략**과 배포 상의 보안/기술적 허들을 총정리한 기획 문서입니다.

---

## 1. 프록시 IP 확보 전략 (디바이스별 비교)

상용 프록시(Luminati 등 월 수백만 원)를 구매하지 않고, 무적의 우회 IP망을 구축하는 3가지 대안입니다.

| 전략 구분 | 방식 설명 | 브레인서버 (클라우드) | 장점 (Pros) | 단점 (Cons) | 최종 권장도 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **A. PC 배포 모델 (Thin Client)** | 가맹점주 PC에 파이썬 기반 `.exe` 에이전트를 설치하여 로컬 공유기 IP 우회 | Vercel (명령 하달) | **유지비 0원, 이론상 무한대 확장, 가장 강력한 P2P 프록시망** | 유저 PC가 켜져 있어야만 가동됨, 배포 시 백신 경고창 문제 | ⭐⭐⭐⭐⭐ (최강) |
| **B. 사무실 테더링 팜 (공기계)** | 안드로이드 공기계 5대를 사무실 PC에 물려, 비행기 모드로 자체 모바일 IP 세탁 | 로컬 메인 거치형 PC | 유저 설득 필요 없음, 중앙 통제 가능, 무적의 모바일 IP 풀 | 알뜰폰 유심 유지비(기기당 월 1.5만 원) 지속 발생 | ⭐⭐⭐ (대안) |
| **C. 모바일 앱 배포 (WebView)** | 가맹점주 폰에 Android 앱을 깔아 백그라운드 몰래 브라우저를 돌림 | Vercel (명령 하달) | 최고 등급 모바일 IP 확보 | iOS 심사 불가, 배터리 광탈로 유저 불만 폭주 (앱스토어 강퇴 위험) | ❌ (기각 권장) |

## 2. UI 및 로컬 에이전트 통합 아키텍처 대안 (UX 중심)

프론트엔드 뷰(Next.js)와 백엔드 엔진(Python)을 가맹점주에게 어떤 형태로 배포할 것인지에 대한 5가지 대안입니다.

### [대안 1] 웹 브라우저 팝업 (올인원 파일)
파이썬 `.exe` 실행 시, 내장된 Next.js 정적 파일을 읽어와 크롬 인터넷 창표준 탭(`localhost:8000`)에 자동으로 띄워주는 방식입니다.
- **장점**: 구현 난이도가 가장 낮고, 로컬 파이썬 통신이 매끄럽습니다.
- **단점**: 일반 웹사이트 탭처럼 보여 '독립된 프로그램'이라는 인상이 약합니다.

### [대안 2] PyWebView / Eel (네이티브 독립 팝업창)
웹 브라우저 탭이 아니라, 파이썬이 **주소창(URL)이 없는 깔끔한 독립적인 데스크탑 윈도우 창**을 직접 생성하는 방식입니다.
- **장점**: 파이썬 코드만으로도 완벽한 데스크탑 앱(B2B 설치형 프로그램)의 퀄리티와 사용자 경험(UX)을 제공합니다.
- **단점**: 구형 OS 모델(Windows 7 등)에서는 브라우저 엔진 파편화가 발생할 수 있습니다. (Windows 10+ 권장)

### [대안 3] 브레인-스텔스 분리형 (Cloud UI + Background Agent)
화면 전체(Next.js)는 Vercel(`dplog.com`)에 띄워두고, 로컬에는 화면이 없는 보이지 않는 코어 파이썬 에이전트(시스템 트레이 미니 아이콘)만 심어두는 방식입니다.
- **장점**: 대시보드 UI가 업데이트될 때마다 가맹점주가 프로그램을 다시 깔 필요 없이 클라우드에서 실시간 반영됩니다.
- **단점**: 클라우드 웹 ➔ 로컬 서버 통신을 위한 WebSocket(역방향 프록시) 같은 복잡한 터널이 뚫려 있어야 합니다.

### [대안 4] 크롬 확장 프로그램 (Chrome Extension) 배포 - 💡 **[초강력 대안]**
.exe 파일을 버리고, '크롬 웹스토어'에 확장 프로그램으로 배포합니다. 가맹점주의 브라우저 자체가 스크래핑 엔진이 됩니다.
- **장점**: 설치 클릭 한 번으로 끝납니다. **가장 치명적이었던 윈도우 디펜더/백신 경고창 문제가 0% 해결**됩니다.
- **단점**: 완성도 높은 현재의 파이썬 기반 데이터 추출 코드를 자바스크립트(JS)로 전면 재작성해야 합니다.

### [대안 5] Electron.js 플랫폼 결합
슬랙, VS Code처럼 크롬 엔진과 백엔드를 통째로 묶어 거대한 데스크탑 소프트웨어를 구축합니다.
- **장점**: OS 환경을 100% 제어하는 업계 최고 수준의 안정성과 활용도.
- **단점**: `.exe` 파일 용량이 200MB 이상으로 치솟으며, Node.js+Python 하이브리드 빌드 과정이 상당히 고통스럽습니다.

## 3. 백신 오진(SmartScreen) 대처 로드맵

**문제점:** 데스크탑 프로그램(.exe) 배포 시 윈도우 OS는 "공식 디지털 서명(인증서)"이 없으면 악성 바이러스로 오인하여 파란색 경고창을 띄웁니다.

- **극초기 대응 (자본금 0원)**: 수동 예외 가이드 배포 ("경고창에서 [추가 정보] -> [실행] 요망").
- **상용화 대응 (투자 필요)**: Nuitka C++ 언패킹 방지 적용 + 기업용 코드 서명 인증서(Code Signing Certificate) 구매/서명 (연 10~40만 원 수준).
*(유일한 예외: 2번의 [대안 4] 크롬 확장 프로그램 배포를 채택할 경우 이 경고창 문제를 통째로 스킵할 수 있습니다.)*
