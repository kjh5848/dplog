# 디플로그(D-PLOG) SaaS 모델을 위한 하이브리드 프록시 티어링 아키텍처 기획

본 기획서는 현재의 로컬(로컬 PC) 작동 수준을 넘어, 향후 다수의 B2B 고객(사장님들)이 동시에 접속하고 백그라운드 트래픽을 다발적으로 발생시키는 **SaaS(Software as a Service)** 환경으로 스케일업(Scale-up)할 때 발생하는 **'초대형 트래픽 과금 폭탄'** 을 원천 억제하고 스크래핑의 속도(동시성)를 확보하기 위한 **[이원화 프록시 라우팅 전략]** 입니다.

> [!IMPORTANT]
> **핵심 전략 (SaaS 최적화)**: 중앙 SaaS 서버에서 감당 불가능한 대규모 트래픽을 유발하는 지면(황금키워드)에는 무제한 요금제인 '데이터센터(ISP) 프록시'를 방패로 세워 트래픽(90%) 요금을 방어하고, 봇 탐지가 극도로 삼엄한 지그재그 패턴 구역(순위추적)에만 비싼 '주거용 프록시' 요금제를 핀셋 투입(10%)하여 수익률(Margin)을 극대화합니다.

## 기능별 프록시 분배 타겟팅 (SaaS 다중 유저 스케일)

### 1. 황금키워드 발굴 (Golden Keyword) - 트래픽 하마
- **타겟 지면**: 네이버 통합/블로그 검색 (`search.naver.com`), 연관 검색어
- **SaaS 리스크**: 유저 10명이 동시에 황금키워드 버튼을 누르면 순간 트래픽이 300MB(300원)씩 터지는 과금 붕괴 지점.
- **배정 프록시**: **데이터센터 프록시 (ISP / 무제한 IP)**
- **기획 이유**: 트래픽을 수십 GB 소모해도 월 정액 고정 금액만 내면 되므로 트래픽 한계 비용이 Zero(0)에 수렴합니다. 비용 걱정 없이 100명, 1000명의 동시 쿼리를 병렬로 돌릴 수 있는 SaaS 전용 아키텍처입니다.

### 2. 순위조회 트래킹 (Ranking Tracking) - 철통 보안 구역
- **타겟 지면**: 네이버 플레이스 모바일 (`m.place.naver.com`)
- **SaaS 리스크**: 다수 유저의 조회가 중앙 서버 단일 IP에 쏠리면 네이버 WAF에 의해 서비스 전체가 연쇄적으로 마비됨.
- **배정 프록시**: **주거용 프록시 (Residential IP - 발급 완료)**
- **기획 이유**: 이미지 차단(0.1~1.8MB) 최적화를 반드시 적용한 뒤, 주거용 프록시의 "세션 고정(Session ID)" 기능을 활용하여 '유저마다 각각 다른 일반인 모바일 환경' 인 것처럼 네이버를 완벽하게 속입니다.

### 3. 내 가게 (My Store) 홈 - 라이트웨이트
- **타겟 지면**: 네이버 플레이스 홈 단건 조회
- **배정 프록시**: **데이터센터(ISP) 풀 ➡️ (실패 시) 주거용 프록시 우회 (SaaS Failover)**
- **기획 이유**: SaaS 환경에서는 사장님의 로컬 컴퓨터 IP를 쓸 수 없으므로(클라우드 서버에서 파싱하기 때문), 트래픽이 무제한인 ISP 프록시로 가장 먼저 찌릅니다. 막힐 경우에만 브레이커가 작동해 주거용으로 자동 전환되어 체감 UX에 지장이 없도록 설계합니다.

### 4. 실시간 조회 (Real-time Search) - 고객 체감 영역
- **타겟 지면**: 즉각적인 실시간 랭킹 확인 (메인 화면)
- **배정 프록시**: **주거용 프록시 (Residential IP)**
- **기획 이유**: B2B SaaS 고객에게 가장 민감한 "속도와 즉각성" 구역입니다. 싼 프록시로 통과를 시도하다 튕기고 재시도(Retry)할 시간 단 2초조차 아깝습니다. 한 번 찌를 때 100% 한 방에 뚫리는 최고급 주거용 IP를 확정 배분합니다.

## 파이썬 SaaS 아키텍처 구현 방향 (Proxy Router)

향후 SaaS용 중앙 API 서버(FastAPI 등)에서 브라우저를 띄울 때, 백그라운드 태스크(Task) 종류에 따라 프록시와 세션을 다이나믹하게 할당(Dynamic Routing)하는 코드 디자인 패턴입니다.

    if task_type in ["GOLDEN_KEYWORD", "MY_STORE_INIT"]:
        proxy_url = "http://datacenter.proxy.io:port"
    
    # 2. 확실한 방어와 무한 스크롤이 필요한 플레이스는 주거용 핀셋 할당
    else:
        # SaaS 유저별로 세션 ID를 파티셔닝하여 네이버의 다중계정 쏠림 탐지 회피
        session_id = f"user_{user_id}_{random_string()}"
