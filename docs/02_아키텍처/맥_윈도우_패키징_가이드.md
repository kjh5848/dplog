# D-PLOG macOS / Windows 데스크톱 패키징 가이드

## 목적

D-PLOG 데스크톱 앱은 `frontend_dplog`의 정적 Next.js 산출물과 `backend_python`의 FastAPI 서버를 하나의 로컬 앱으로 묶어 배포한다. 앱은 실행 후 `127.0.0.1:45123`에서 백엔드를 띄우고, 설치된 Chrome 또는 Microsoft Edge를 앱 모드로 열어 UI를 보여준다.

## 기본 원칙

- macOS와 Windows는 같은 소스코드를 쓰되, 각 OS에서 따로 빌드한다.
- macOS에서 Windows `.exe`를 안정적으로 크로스 컴파일하지 않는다.
- Playwright 브라우저 바이너리는 앱에 포함하지 않는다.
- 사용자의 Chrome 또는 Microsoft Edge를 찾아 스크래핑과 앱 모드 실행에 사용한다.
- DB와 수집 이미지는 앱 내부가 아니라 OS별 사용자 데이터 폴더에 저장한다.

## 런타임 저장 위치

| OS | 저장 위치 |
| --- | --- |
| macOS | `~/Library/Application Support/dplog` |
| Windows | `%APPDATA%\dplog` |
| Linux/dev fallback | `~/.dplog` |

저장되는 항목:

- `dplog.db`
- `static/images/...`
- `chrome_profile`

기존 개발용 `~/.dplog/dplog.db`가 있으면 첫 실행 시 OS 표준 위치로 복사된다.
번들에 포함된 기본 미디어 파일은 사용자 데이터 폴더에 없는 파일만 시드한다. 사용자가 수집한 이미지와 DB가 앱 재실행 또는 앱 업데이트로 덮어써지면 안 되기 때문이다.

## macOS 빌드

```bash
./build_edge_mac.sh
```

결과물:

```text
backend_python/dist/macos/*.app
```

검증:

```bash
./test_desktop.sh
```

확인 항목:

- 앱 실행 후 브라우저 앱 모드가 열린다.
- `/settings`의 앱 상태가 정상이다.
- 상점 등록, 데이터 업데이트, 황금 키워드, 순위 조회가 동작한다.
- 재실행 후 DB와 이미지가 유지된다.

## Windows 빌드

Windows PC 또는 Windows CI에서 실행한다.

```powershell
Set-ExecutionPolicy -Scope Process Bypass
.\build_edge_windows.ps1
```

결과물:

```text
backend_python\dist\windows\D-PLOG.exe
```

패키징 전 빠른 시뮬레이션:

```powershell
Set-ExecutionPolicy -Scope Process Bypass
.\test_desktop_windows.ps1
```

Windows 사전 요구사항:

- Windows 10 이상
- Python 3.11 이상
- Node.js LTS
- Chrome 또는 Microsoft Edge
- Nuitka 컴파일용 C 컴파일러. `build_edge_windows.ps1` 실행 중 Nuitka가 요구하는 도구chain 안내가 뜨면 해당 안내에 따른다.

## 브라우저 탐색 규칙

환경변수 `DPLOG_BROWSER_PATH`가 있으면 그 경로를 우선 사용한다.

예:

```powershell
$env:DPLOG_BROWSER_PATH = "C:\Program Files\Google\Chrome\Application\chrome.exe"
```

자동 탐색 순서:

- macOS: Google Chrome, Microsoft Edge, Chromium
- Windows: Program Files / Program Files (x86) / LocalAppData의 Chrome, Edge, Chromium
- Linux: `google-chrome`, `chromium`, `microsoft-edge` 등 PATH 명령

## 배포 주의사항

- Windows `.exe`는 Windows에서 직접 빌드해야 한다.
- macOS 앱은 Apple notarization 전까지 Gatekeeper 경고가 뜰 수 있다.
- 코드서명/공증, Windows installer, 자동 업데이트는 1차 패키징 이후 별도 단계로 둔다.
- 고객용 배포 전에는 번들 DB가 샘플 데이터가 아닌 초기 상태인지 확인한다.
