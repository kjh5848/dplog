## 📋 전체 요약

## 🔴 긴급 (Critical)

### 1. FeatureExplorer 비디오 - 외부 의존성 문제

![FeatureExplorer 섹션](./screenshots/feature_explorer_section.png)

- **문제**: 비디오 소스가 외부 URL(Pexels)에 의존하고 있으며, 콘텐츠도 **D-PLOG 제품과 관련 없는 일반 스톡 영상**
- **SaaS 기준**: 기능 소개 영상은 **실제 제품 화면(Product Screenshot/Demo)** 이어야 신뢰도 확보
- **권장 조치**: 실제 D-PLOG 대시보드 녹화 또는 제품 스크린샷 GIF로 교체

### 2. Solution 섹션 이미지 - 스톡 이미지 의존

![Solution 섹션](./screenshots/solution_section.png)

- **문제**: "10분 내 진단 완료", "근거 기반 리포트", "즉시 실행 가능", "반복 개선 사이클" 4단계 모두 **일반 스톡 사진** 사용
- **SaaS 기준**: 각 단계에 **실제 제품 UI 스크린샷**이나 **목업(mockup)**을 넣어야 사용자가 "이렇게 작동하는구나"를 이해 가능
- **권장 조치**: 각 기능에 해당하는 제품 화면 캡처 또는 일러스트 제작

### 3. Footer 링크 "준비 중" 표시

- **문제**: 고객문의, FAQ, 이용가이드 등 주요 지원 링크가 "(준비 중)"으로 표시
- **SaaS 기준**: 유료 제품의 지원 체계가 미완성으로 보이면 **전환 저해 요인**
- **권장 조치**: 최소한 이메일 연락처나 카카오톡 채널 등 살아있는 지원 채널 연결

## 🟡 개선 필요 (Important)

### 4. Hero 섹션 - 제품 비주얼 부재

![Hero 섹션](./screenshots/hero_section.png)

- **문제**: Hero에 텍스트와 CTA 버튼만 있고, **제품 이미지/데모 영상이 없음**
- **SaaS 기준**: 최신 SaaS Hero는 반드시 **Product Screenshot + 그라데이션 그림자** 또는 **짧은 데모 영상**을 포함
- **참고**: Linear, Vercel, Notion 등 모두 Hero에 제품 비주얼 포함
- **권장 조치**: Hero 하단에 대시보드 스크린샷을 배치하되, 약간 기울인 3D perspective 적용

### 5. 신뢰 배지(Social Proof) 부재

- **문제**: Hero 아래에 "외식업 사장님 2,200명이 선택한 솔루션"이라는 수치만 있고, **로고 배너나 미디어 노출 배지가 없음**
- **SaaS 기준**: "Trusted by" 로고 배너 (파트너사, 미디어 노출, 인증 마크)는 전환율에 큰 영향
- **권장 조치**: Hero 아래에 파트너/미디어 로고 줄 추가 (SBA, 중소벤처기업부, 소상공인시장진흥공단 등)

### 6. Pricing 섹션 - 기능 비교표 부재

- **문제**: 각 요금제의 특징을 나열하고 있지만, **가격별 기능 비교 매트릭스(Feature Comparison Table)**가 없음
- **SaaS 기준**: Free → Standard 전환을 유도하려면 "뭐가 다른지" 한눈에 비교 가능해야 함
- **권장 조치**: 체크마크(✓) 기반 기능 비교표 추가

### 7. Action-First 섹션 - 메시지 전달력 부족

![Action-First 섹션](./screenshots/action_first_section.png)

- **문제**: 아이콘 그리드가 시각적으로 깔끔하지만, **각 아이콘이 무엇을 의미하는지 라벨이 없음**. 사용자가 아이콘만으로 기능을 유추하기 어려움
- **SaaS 기준**: Bento Grid 또는 Feature Card 형태로 아이콘 + 짧은 라벨(2~3단어)을 함께 표시
- **권장 조치**: 아이콘 아래에 "진단 리포트", "블로그 생성" 등 짧은 라벨 추가, 또는 Bento Grid 레이아웃으로 변경

### 8. CTA 버튼 일관성

- **문제**: 페이지 내 CTA가 "무료로 시작하기"(Navbar), "무료로 진단 시작하기"(Hero), "앱 스토어"(Footer CTA)로 분산되어, **최종 전환 동선이 불명확**
- **SaaS 기준**: Primary CTA는 페이지 전반에 걸쳐 **동일한 문구와 동작**으로 통일
- **권장 조치**: "무료로 진단 시작하기"를 주요 CTA로 통일하고 모든 곳에 동일하게 적용

## 🟢 권장 (Nice-to-have)

### 9. Testimonials - 별점/만족도 수치 추가

![Testimonials](./screenshots/testimonials_section.png)

- 현재 텍스트 후기만 있는데, ⭐ 별점 또는 "평균 만족도 4.8/5" 같은 **정량적 수치** 추가 시 신뢰도 상승

### 10. Problem 섹션 - 감정적 강화

![Problem 섹션](./screenshots/problem_section.png)

- 현재 잘 구성되어 있으나, **실제 사장님 인용("이달도 적자인데 뭘 먼저 해야 할지...")** 같은 감정적 요소를 추가하면 공감 극대화

### 11. 페이지 로딩 속도

- 외부 비디오 3개(UHD 해상도)가 페이지 로딩을 지연시킬 수 있음
- **권장 조치**: 비디오를 `lazy loading` 처리하고, 해상도를 720p로 축소하거나 WebM 포맷 사용

### 12. 모바일 반응형 검증

- 현재 데스크톱 기준으로만 확인. **모바일 뷰에서 레이아웃이 깨지는 부분이 없는지** 별도 검증 필요
- 특히 Action-First 아이콘 그리드, Solution 이미지+텍스트 섹션이 모바일에서 잘 쌓이는지 확인
