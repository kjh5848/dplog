# Threads — 상위노출 알고리즘 & 최적화 가이드

> **최종 업데이트:** 2026-04-19

---

## 1. 핵심 랭킹 시그널

| 시그널 | 가중치 | 설명 |
|--------|--------|------|
| **엔게이지먼트 벨로시티** | ⭐⭐⭐⭐⭐ | 게시 후 30~60분 내 반응 속도 |
| **리플라이 깊이** | ⭐⭐⭐⭐⭐ | 답글의 답글 (대화 체인) |
| **완독률** | ⭐⭐⭐⭐ | 멈춰서 읽는 비율 |
| **인스타 연동** | ⭐⭐⭐ | 인스타 활동이 Threads 추천에 영향 |

## 2. 피드 구분

- **"For You"** — AI 랭킹 추천 (발견 채널)
- **"Following"** — 순수 시간순 (알고리즘 미적용)

## 3. 셀프 리플라이 체이닝

500자 제한을 넘어 블로그급 콘텐츠 전달:

```
[PART 1 — 후킹] ≤500자 → 셀프 리플라이
[PART 2 — 본문] ≤500자 → 셀프 리플라이
[PART 3 — CTA]  ≤500자 + 토픽태그
```

## 4. 기술 사양

| 항목 | 값 |
|------|-----|
| 본문 | 파트당 **500자** |
| 첨부 텍스트 | 파트당 10,000자 |
| 이미지 | 파트당 **최대 20장** |
| 동영상 | 파트당 **5분** 이내 |
| 이미지 비율 | **4:5** (1080×1350) 권장 |

## 5. 체크리스트

### ✅ 해야 할 것
- 첫 줄 강력한 후킹
- 질문형/의견 유도 → 리플라이 깊이 극대화
- 게시 후 1시간 내 모든 댓글에 대화식 응답
- 토픽 태그 활용
- 텍스트 + 이미지 혼합

### ❌ 하지 말아야 할 것
- 타 플랫폼 콘텐츠 복붙
- 노골적 어그로/참여 유도
- 게시 후 방치

> **핵심:** 게시 후 1시간이 모든 것을 결정. "팔로워 수"보다 "대화의 깊이"가 도달을 결정하는 플랫폼.
