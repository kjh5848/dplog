# 네이버 블로그 에디터 한글 타이핑 꼬임 현상 리서치 및 해결 방안

## 1. 꼬임 현상(Scrambling)의 근본적 원인
Playwright나 Selenium을 이용해 네이버 블로그 스마트에디터(SmartEditor ONE)에 `type()`이나 `send_keys()`로 한글을 한 글자씩 타이핑할 때 발생하는 문자 섞임이나 커서 튐 현상은 다음 두 가지 이유로 발생합니다.

*   **IME Composition (한글 자소 결합) 이벤트 누락:** 브라우저 자동화 도구들은 기본적으로 영문 키보드를 모사하여 KeyDown/KeyUp 이벤트를 발생시킵니다. 한글의 특성상 초성, 중성, 종성이 결합되는 `CompositionStart`, `CompositionUpdate`, `CompositionEnd` 이벤트가 완벽히 재현되지 않아, 브라우저가 키 입력을 오해하게 됩니다.
*   **비동기 에디터 렌더링 간섭:** SmartEditor ONE은 React 기반으로 구현되어 있습니다. 실시간 맞춤법 검사, 임시 저장, 자동 태그 변환 등의 배경 스크립트가 실행되는 도중 단일 문자 단위의 강제 입력(`insert_text` 혹은 `type`)이 아주 짧은 딜레이로 계속해서 발생하면, 리액트의 에디터 DOM 상태(커서 위치 등) 계산이 실제 타이핑 타이밍을 따라가지 못해 커서가 과거의 위치로 튕겨버립니다. (위에서 경험하신 `"뉴일제히 상승장으로 마감했습니다.o.kr/news/..."` 현상이 전형적인 사례입니다.)

## 2. 해결 방법 및 선택지

웹 커뮤니티와 자동화 생태계에서 권장하는 네이버 블로그 한글 입력의 해결법은 아래 세 가지입니다.

### 방법 A: 단일 클립보드 전체 붙여넣기 (가장 안정적, 속도 최상)
타이핑해야 할 모든 텍스트를 `pyperclip`을 이용해 OS 클립보드에 담은 뒤, 에디터를 클릭하고 `Ctrl+V` (또는 `Cmd+V`) 단축키를 한 번에 보내는 방법입니다.
*   **장점:** 한글 자소 결합 이슈와 에디터 비동기 렌더링 딜레이를 완벽히 피할 수 있어 **100% 꼬임이 방지**됩니다.
*   **단점:** "사람이 직접 타자를 치는 듯한 시각적 효과(유도리 있는 타이핑 속도)"를 줄 수 없습니다.

### 방법 B: 단락(문장) 단위 분할 붙여넣기 + `Enter` 시뮬레이션 (절충안 / 현재 추천)
한 글자씩 치는 대신, **"단락(줄) 단위로 클립보드에 복사 -> 붙여넣기 -> 실제로 1~2초 쉬기 -> `Enter` 키 입력 -> 다음 줄 복사/붙여넣기"**를 파이프라인으로 구성합니다.
*   **장점:** 한글 꼬임을 방지하면서도, "문장을 쓰고 다음 줄로 넘어가서 (Enter) 다시 글을 쓰는" 사람과 매우 유사한 흐름을 보여줍니다.
*   **적용:** 사용자 분께서 원하셨던 **"실제로 글을 작성하듯이 하고 엔터를 치는 느낌"**을 가장 안전하게 구현할 수 있는 최적의 방식입니다.

### 방법 C: JavaScript DOM 직접 주입 (`innerHTML` / `execCommand`)
`page.evaluate()` 등을 통해 에디터 내부의 ContentEditable 컨테이너에 직접 `innerHTML`을 우겨넣거나 `document.execCommand('insertText', false, text)`를 실행시키는 방법입니다.
*   **단점:** 네이버 에디터는 단순한 DOM 조작을 감지하고 상태 업데이트를 무시할 수 있으며, 이 방식은 사람이 타이핑하는 척하는 매크로의 특성을 상실합니다.

## 3. 적용 제안 (결론)

현재처럼 `insert_text`를 `for char in text`로 한 글자씩 0.04초 간격으로 뿌리게 되면, 네이버 에디터가 이를 견디지 못하고 포커스를 잃습니다. 

**따라서 방법 B(단락별 분할 붙여넣기 및 명시적 Enter 제어)를 도입하여,**
사용자가 실제 블로그 캔버스 위에서 "한 줄을 적어내고(붙여넣기 찰나) ➜ 고민하듯 1초 쉬고 ➜ Enter를 쳐서 줄을 바꾸고 ➜ 다음 단락을 작성하는" 인간적인 페이스라인을 구축하는 것을 제안합니다.
