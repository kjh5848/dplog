# 인스타그램 자동화 업로드 및 봇 탐지 우회 전략

## 1. 인스타그램 봇 탐지(Bot Detection)의 특징
Playwright나 Selenium을 이용해 인스타그램 웹버전에서 게시물을 업로드할 때 겪게 되는 가장 큰 장벽은 '쉐도우밴(Shadowban)'과 '계정 블록'입니다. 인스타그램은 네이버보다 훨씬 고도화된 안티봇 시스템을 갖추고 있습니다.

*   **브라우저 핑거프린팅:** `webdriver` 속성, headless 브라우저 특유의 렌더링 방식, 비정상적인 User-Agent 등을 즉시 감지합니다.
*   **비인간적인 인터랙션:** 요소를 클릭하고 텍스트를 입력하는 속도가 기계적으로 균일하거나 지나치게 빠를 경우(`page.fill()` 등), 서버에서 봇으로 분류하여 동작을 차단합니다.
*   **세션 쿠키 및 IP 패턴:** 로그인 시도 시의 마우스 움직임 패턴과, 짧은 시간 내의 반복적인 로그인/로그아웃을 강력하게 제재합니다.

---

## 2. 해결 방법 및 우회 선택지

웹 자동화 생태계에서 인스타그램 블로킹을 피하기 위해 사용하는 핵심 우회 전략입니다.

### 방법 A: 공식 Graph API 활용 (가장 안전, 기능 제한)
가장 권장되며 유일하게 인스타그램 정책에 위배되지 않는 방식입니다. 비즈니스 계정과 연동하여 Facebook Graph API를 통해 포스팅을 전송합니다.
*   **장점:** 계정 정지 위험이 0%에 수렴합니다.
*   **단점:** 릴스, 다중 이미지(캐러셀) 업로드 등 일부 세부 기능 지원이 늦거나 복잡하며, API 엑세스 토큰 갱신 등 인프라 설정이 까다롭습니다.

### 방법 B: 수동 세션 스틸(Session Steal) + Stealth 플러그인 (현재 추천)
**현재 D-PLOG가 채택한 방식입니다.** 스크립트가 로그인 과정 전체를 처리하지 않습니다.
1. `headless=False`로 브라우저를 띄워 **사람이 직접 1회 로그인**합니다.
2. 봇 탐지 엔진이 '로그인 성공'으로 인가한 쿠키와 토큰을 `storage_state`로 로컬에 영구 저장합니다.
3. 이후 실행 시, `playwright-stealth` 패키지와 `--disable-blink-features=AutomationControlled` 속성을 부여하여 봇 속성을 숨긴 채 저장된 세션으로 곧바로 홈페이지에 진입합니다.

### 방법 C: 인간형 지연 입력 (Human-like Typing)
인스타그램 캡션(본문) 입력 창은 React 기반의 매우 복잡한 DOM 객체입니다. 한 번에 긴 글을 `innerHTML`로 찔러넣거나 `page.fill()`을 사용하면 이벤트가 막히거나 텍스트가 잘릴 위험이 큽니다.
*   **해결책 (`press_sequentially`):** 문자열 전체 복붙이 아닌, 사람처럼 타자를 치는 `page.locator().press_sequentially(text, delay=120)` 형태로 0.1~0.2초의 랜덤 딜레이를 주어 타이핑합니다.
*   **줄바꿈(Enter) 분리 제어:** 문자열 내의 `\n` 개행문자가 인스타 텍스트박스에서 씹히는 경우가 많습니다. 줄바꿈을 문장에서 제거하고, 코드로 명시적인 `page.keyboard.press("Enter")`를 호출하여 구분해야 합니다.

## 3. Playwright 캡션 타이핑 적용 제안 (결론)

네이버 블로그처럼 '통 복붙'은 지양해야 합니다. 인스타그램 자동화를 위해서는 **"랜덤성"**과 **"느림의 미학"**이 필요합니다.

1. **상호작용 전 지연:** 사진 업로드 및 캡션 창 클릭 전 `asyncio.sleep(random.uniform(2.0, 4.5))` 부여.
2. **이모지와 텍스트 타이핑 분리:** 캡션 텍스트는 `press_sequentially`를 사용하되, 딜레이를 변칙적으로 줍니다.
3. **명시적 Enter:** 문단이 끝날 때마다 실제 엔터 키 이벤트를 디스패치합니다.

이러한 인간과 유사한 마이크로 인터랙션을 모방해야만 계정을 오랫동안 안전하게 보호할 수 있습니다.
