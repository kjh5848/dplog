# 네이버블로그 미디어 레이아웃 글쓰기 규칙

> 목적: `blog_draft.md`만 보고도 HTML 미리보기에서 단일 이미지, 2열 그리드, 3열 격자, 슬라이드형 갤러리가 일관되게 나오도록 원고 작성 규칙을 고정한다.

---

## 1. 기본 원칙

- 미디어는 본문 흐름을 끊는 장식이 아니라, 바로 앞 문단의 증거로 배치한다.
- 이미지와 GIF의 `alt`는 실제 내용과 역할을 짧게 쓴다. 예: `야외 대기석 담요`, `생갈비 불판 GIF`.
- 키워드 반복을 노린 alt 텍스트는 금지한다. 예: `대연동맛집 대연동갈비 이조갈비 맛집`처럼 나열하지 않는다.
- 사람 얼굴, 차량번호, 개인정보가 보이면 모자이크 보정본을 먼저 사용한다.
- MP4/MOV/WEBM은 네이버 발행 안정성을 위해 GIF로 변환한 뒤 `.gif`를 우선 사용한다.

---

## 2. HTML 미리보기 레이아웃 규칙

`build-html-preview`는 연속된 이미지/GIF 개수로 레이아웃을 결정한다.

| 원고 작성 방식 | 미리보기 결과 | 사용 목적 |
|---|---|---|
| 이미지/GIF 1장 | 단일 이미지 블록 | 대표컷, 정보가 중요한 사진 |
| 이미지/GIF 2장 연속 | 2열 그리드 | 비교컷, 전후/좌우 맥락 |
| 이미지/GIF 3장 연속 | 3열 격자 | 메뉴판, 반찬, 소스처럼 같은 범주의 묶음 |
| 이미지/GIF 4장 연속 | 2열 그리드 2행 | 한입컷, 분위기컷, 디테일 묶음 |
| 이미지/GIF 5장 이상 연속 | 가로 스크롤 슬라이드 | 많은 사진을 한 번에 보여줄 때 |

본문 문장, 제목, 구분선(`---`), 지도, URL이 중간에 들어가면 이미지 묶음은 끊긴다.

---

## 3. 작성 예시

### 단일 이미지

```markdown
매장 입구는 과하게 화려하지 않고 깔끔했습니다.

![이조갈비 외부 간판](/절대경로/이미지/img_008.jpeg)
```

### 2열 그리드

```markdown
입구 쪽은 대기 동선과 외관을 같이 보면 분위기가 잘 잡힙니다.

![야간 간판](/절대경로/이미지/img_010_mosaic.jpeg)
![웨이팅 대기석](/절대경로/이미지/img_011.jpeg)
```

### 3열 격자

```markdown
소스와 반찬은 한 묶음으로 보면 구성이 더 잘 보입니다.

![장아찌 반찬](/절대경로/이미지/img_033.jpeg)
![유자 드레싱 샐러드](/절대경로/이미지/img_034.jpeg)
![단호박 샐러드](/절대경로/이미지/img_035.jpeg)
```

### 슬라이드형 갤러리

```markdown
한입 조합은 여러 장을 이어서 보여주는 편이 체류시간에 유리합니다.

![깻잎쌈 한입](/절대경로/이미지/img_015.jpeg)
![푸짐한 깻잎쌈](/절대경로/이미지/img_016.jpeg)
![냉면과 갈비](/절대경로/이미지/img_021.jpeg)
![상추 갈비쌈](/절대경로/이미지/img_022.jpeg)
![갈비 밥쌈](/절대경로/이미지/img_024.jpeg)
```

## 4. 글쓰기 배치 리듬

- 250~450자마다 미디어 묶음을 넣어 체류 시간을 늘린다.
- 정보성 컷은 단일 이미지로 크게 보여준다.
- 비슷한 성격의 컷은 2~4장 그리드로 묶어 반복감을 줄인다.
- 많은 디테일 컷은 5장 이상 슬라이드로 묶어 본문 길이를 과하게 늘리지 않는다.
- GIF는 움직임이 의미 있는 구간에만 쓴다. 예: 불판 위 생갈비, 갈비 굽는 장면, 냉면 비비는 장면.

---

## 5. 네이버 발행 주의

HTML 미리보기의 2열/3열/슬라이드 레이아웃은 검수용 시각화다. 실제 네이버 스마트에디터는 업로드 방식과 에디터 정책에 따라 동일한 CSS 레이아웃을 보장하지 않을 수 있다.

따라서 발행 기준은 다음 순서로 본다.

1. 원고에서 미디어 순서와 alt 텍스트가 맞는가.
2. HTML 미리보기에서 묶음 의도가 맞는가.
3. 네이버 에디터에서 실제 업로드 후 깨지는 구간이 없는가.

실제 발행 직전에는 `build-html-preview`로 먼저 검수하고, 네이버 에디터에서는 최종 화면을 수동 확인한다.
