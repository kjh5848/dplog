## 1. 목적
- 사용자가 과거 리포트를 쉽게 다시 보고, 기간/키워드 기준으로 비교하여 “반복 개선” 사이클을 돌리게 한다.

## 2. 범위
### 2.1 In scope
- 리포트 목록(스토어별) 조회/필터
- 리포트 상세 조회
- 비교(2개 리포트 선택 → 차이 요약)
- 보존 정책(무료/유료, 기간, 삭제 정책)
- 권한(본인/직원/대행) 모델(향후 확장 가능)

### 2.2 Out of scope
- 팀/조직 기능(권한 관리 UI) — 필요 시 별도 Spec

## 3. 사용자 시나리오
- Happy path: 매장 선택 → 리포트 목록 → 최근 리포트 열람 → 이전 리포트와 비교
- 예외: 리포트 없음(empty), 권한 없음(403), 오래된 데이터 보존 만료

## 4. API(초안)
- `GET /v1/stores/{storeId}/reports`
- `GET /v1/reports/{reportId}`
- (비교) `GET /v1/reports/compare?left=...&right=...` 또는 FE에서 diff

## 5. FE UX(초안)
- 리스트: 날짜/진단 상태/Top10 개수 같은 요약 칩
- 비교: 요약 변화 + 키워드별 순위 변화(가능하면)
