# D-PLOG 2.0: 멀티모달 AI 자동화 파이프라인 기획안

사용자께서 요청하신 1~5번 요건(미디어 무작위 첨부, 자동 분류, 동영상 GIF 변환, AI 자유도에 따른 컨텐츠 배치)을 완벽하게 충족하기 위한 고도화 아키텍처 모델입니다.

## 1. 아키텍처 및 시스템 워크플로우

본 기능을 구현하기 위해 D-PLOG 데스크탑 시스템은 4가지 핵심 엔진을 탑재합니다.

### A. 미디어 팩토리 (Media Factory) 엔진
*   **동기화 및 분류:** PyQt6 GUI에 무작위로 떨어진 이미지와 동영상을 확장자별로 분류합니다.
*   **비디오 ➔ GIF 하이라이트 변환기 (moviepy 활용):** 동영상의 핵심 부분(예: 첫 3~5초 혹은 하이라이트)을 자동으로 프레임 추출하여 `gif` 움짤 파일로 압축 생성합니다. 네이버 블로그 생태계에서 체류시간을 높이는 핵심 기술인 '생동감 있는 움짤'을 자동으로 양산합니다.

### B. 멀티모달 AI 뇌 (LLM Director)
*   **요구사항:** 이미지 픽셀 정보 시야각을 처리할 수 있는 GPT-4o 또는 Gemini 1.5 Pro API를 연동합니다.
*   **자유도 위임 (Autonomy):** 미디어 팩토리가 정제한 이미지 목록(예: img1, img2, gif_v1)을 AI에게 전송하고, AI가 알아서 매장 정보와 맥락을 파악해 글을 작성합니다.
*   **적재적소 배치:** AI가 *"여기서 고기 굽는 이미지가 들어가면 좋겠다"* 고 판단하면, 마크다운 텍스트 사이에 `[MEDIA: img1.jpg]` 와 같이 미디어 치환자표를 자율적으로 삽입합니다.

### C. 에디터 렌더링 엔진 (SmartEditor Publisher)
*   기존의 `open_naver.py`를 대대적으로 업그레이드합니다.
*   파서(Parser)가 글을 쓰다가 `[MEDIA: ...]` 태그를 발견하면, 타이핑을 멈추고 네이버 스마트에디터의 상단 **[사진📝]** 버튼 하부의 숨겨진 `<input type="file">`에 접근하여 해당 파일을 강제 업로드시킵니다. 이미지가 본문 중앙에 삽입되어 로딩될 때까지 기다린 후 다시 텍스트 타이핑을 이어갑니다.

## 2. 세부 개발 단계 (Action Plan)

1.  **동영상 처리 모듈 구축:** `moviepy` 라이브러리를 통해 동영상을 가져오고 짧은 GIF로 Export하는 파이썬 사이드 스크립트를 작성합니다.
2.  **LLM Prompt & API 세팅:** GUI에 **[AI 원고 자동 생성]** 버튼을 추가하고, LLM API와 통신하여 미디어 배치형 원고 텍스트를 돌려받는 코드를 작성합니다.
3.  **마크다운-미디어 믹스 파싱 보완:** `manuscript.md` 포맷에 들어있는 미디어 치환 태그를 Playwright가 읽고 사진 업로드 로직으로 변환하는 처리를 덧붙입니다.

## 3. 🚨 사용자 리뷰 필요 (오픈 퀘스천)

이 야심 찬 파이프라인을 구축하기 전, 기술적 방향을 확정하기 위해 다음 두 가지를 질문드립니다!

1.  **AI 모델 및 API 키 문제:** 이 모든 시각 분석과 글쓰기 판단 지능을 수행하기 위해선 백엔드에서 **구글 Gemini(추천)**나 **OpenAI GPT-4o**의 API Key 결제가 필요합니다. 연동을 원하시는 API 종류가 무엇이며, 키 값이 준비되어 있으신가요? (키는 보안 파일인 `.env`에 저장하게 될 것입니다.)
2.  **네이버 동영상 원본 업로드 여부:** 동영상을 짜깁기해서 **GIF 움짤로 변환해 본문에 자유롭게 배치**하는 것은 말씀하신대로 진행합니다. 이때, 변환을 마친 묵직한 오리지널 **원본 동영상 자체**도 네이버 에디터 내장 플레이어로 업로드되기를 희망하시나요? (네이버 내장 동영상 업로더는 업로드 창이 무겁고 처리 시간이 10~20초 이상 더 걸릴 수 있습니다.)

승인과 함께 질문에 대한 답변을 주시면 즉각 엔진 개발을 시작합니다!
